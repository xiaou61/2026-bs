# 校园图书漂流借阅与读书打卡平台 PRD

## 项目概述
本系统面向校园图书漂流、学生借阅、读书打卡、笔记沉淀、积分勋章和阅读活动场景，提供漂流书架、漂流图书、读者档案、图书捐赠、借阅记录、归还流转、读书打卡、读书笔记、积分勋章、阅读活动和消息通知的一体化后台管理能力。

## 用户角色
- 系统管理员：账号权限、基础配置和全部数据维护
- 图书馆管理员：维护漂流书架、漂流图书、借阅记录和统计规则
- 漂流负责人：维护图书捐赠、借阅记录、归还流转和消息通知
- 学生读者：参与图书捐赠、图书借阅、归还流转、读书打卡和读书笔记
- 读书社团：组织阅读活动、协同打卡运营和积分勋章评定
- 阅读指导老师：审核读书笔记、指导阅读活动、评定积分勋章和读者成长

## 功能模块
- 漂流书架：书架编号、书架名称、所在校区、维护人员、启用时间和书架状态维护
- 漂流图书：图书编号、图书名称、图书类型、当前持有人、入库时间和图书状态维护
- 读者档案：读者编号、读者姓名、所在班级、负责老师、建档时间和读者状态维护
- 图书捐赠：捐赠编号、捐赠图书、捐赠类型、接收人员、捐赠时间和捐赠状态维护
- 借阅记录：借阅编号、借阅图书、借阅类型、借阅读者、借阅时间和借阅状态维护
- 归还流转：流转编号、归还图书、流转类型、交接人员、归还时间和流转状态维护
- 读书打卡：打卡编号、打卡图书、打卡类型、打卡读者、打卡时间和打卡状态维护
- 读书笔记：笔记编号、阅读图书、笔记类型、提交读者、提交时间和笔记状态维护
- 积分勋章：勋章编号、获奖读者、积分类型、评定人员、获得时间和勋章状态维护
- 阅读活动：活动编号、活动名称、活动类型、组织人员、活动时间和活动状态维护
- 消息通知：通知编号、通知标题、通知类型、发送人员、发送时间和通知状态维护
- 操作日志：日志编号、操作模块、操作类型、操作人、操作时间和执行结果维护

## 技术栈
- 后端：Spring Boot 2.7.18、MyBatis 注解 SQL、PageHelper、MySQL、Redis、JWT
- 前端：Vue3、Element Plus、Pinia、Axios、ECharts、Vite
- 数据库：campus_book_drift_175
