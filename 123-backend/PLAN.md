# 实现计划

## 问题陈述

构建一个围绕水位监测、雨量预警和应急调度的前后端分离系统，覆盖水位点位、雨量站点、排涝泵站、内涝预警、应急预案、物资保障和处置任务。

## 当前状态

`123-backend` 与 `123-frontend` 已完成城市内涝调度平台的基础实现，当前进入精修阶段，重点核对监测口径、预警闭环、认证安全和文档准确性。

## 实施方案

### 第一阶段：后端开发
1. Spring Boot 2.7.18 + MyBatis-Plus 3.5.5 + Redis + JWT
2. 创建 `urban_flood_dispatch_123` 数据库和 14 张业务表
3. 实现统一返回、异常处理、JWT 拦截和 Redis Token 校验
4. 实现 Entity、Mapper、Service 和 Controller
5. 实现预警规则、应急调度、任务分派、物资保障和处置跟踪
6. 实现首页统计和操作日志

### 第二阶段：前端开发
1. Vue3 + Vite + Element Plus + Pinia + Axios + ECharts
2. 登录页、布局页、首页看板
3. 水位点位、雨量站点、排涝泵站、水位数据、雨量数据、预警规则、内涝预警、应急预案、物资保障和日志页面
4. 业务按钮支持预警、调度、派发、反馈和查看态势

### 第三阶段：文档更新
1. 更新 `readme.md`
2. 更新 `readme_simple.md`
3. 更新候选清单

## 完成标准

1. 后端结构完整。
2. 前端结构完整。
3. SQL 包含演示数据。
4. 认证返回不暴露敏感字段。
5. 水位、雨量、预警、调度和物资保障口径一致。
6. 文档描述与实际实现保持一致。
7. 不执行编译验证。
