# 城市内涝监测与应急调度平台 PRD

## 1. 项目背景
城市极端降雨和低洼点积水风险提升，平台需要统一管理水位点位、雨量站、排涝设备、预警规则、应急任务和物资调度闭环。

## 2. 用户角色
系统提供四类默认演示角色：ADMIN（admin / 123456）、MONITOR（monitor / 123456）、DISPATCHER（dispatcher / 123456）、MANAGER（manager / 123456）。

## 3. 功能模块
1. 账号权限：管理员、监测员、调度员、应急主管四角色账号、JWT 鉴权和 Redis 登录态。
2. 水位点位：点位编号、名称、行政区、河道/道路、警戒水位和状态管理。
3. 雨量站点：站点编号、名称、区域、设备类型、采样频率和负责人管理。
4. 排涝泵站：泵站编号、名称、区域、排水能力、在线泵数和运行状态管理。
5. 水位数据：数据编号、点位、采集时间、水位值、上涨速度和数据状态管理。
6. 雨量数据：数据编号、雨量站、采集时间、小时雨量、累计雨量和雨强等级管理。
7. 预警规则：规则编号、规则名称、指标类型、阈值、预警等级和启用状态管理。
8. 内涝预警：预警编号、点位、等级、触发原因、处理人和触发时间管理。
9. 应急预案：预案编号、名称、适用等级、责任部门、启动条件和预案状态管理。
10. 调度任务：任务编号、预警编号、任务类型、责任单位、截止时间和执行状态管理。
11. 救援队伍：队伍编号、名称、类型、人数、联系电话和驻点位置管理。
12. 物资储备：物资编号、名称、类型、库存、安全库存和仓库位置管理。
13. 避险点位：点位编号、名称、地址、容量、负责人和开放状态管理。
14. 操作日志：操作人、模块、动作、对象、详情和IP地址审计。

## 4. 核心流程
1. 管理员维护账号、基础档案和业务主数据。
2. 业务人员登记监测、预约、故障或任务记录并推进状态流转。
3. 管理人员查看预警、调度、收益和运维指标。
4. 系统通过首页看板展示趋势、分布和关键数量。
5. 操作日志沉淀关键动作，方便答辩时展示闭环。

## 5. 非功能要求
- 后端采用 Spring Boot，前端采用 Vue3。
- 数据库使用 MySQL，默认账号密码为 root / 1234。
- 登录态使用 JWT 与 Redis 校验。
- 数据页支持分页查询、关键词检索、状态筛选、新增、编辑、删除和状态动作。
