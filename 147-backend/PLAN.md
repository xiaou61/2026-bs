# 校园心理咨询预约与危机干预管理系统 开发计划

## 实施范围
- 后端工程：实体、Mapper、Service、Controller、统一响应、JWT 登录、Redis Token 校验、统计接口
- 前端工程：登录页、布局菜单、通用数据页、模块页面、统计看板、接口封装和路由守卫
- 数据脚本：MySQL 建库建表、初始化账号和演示业务数据

## 模块拆分
- 咨询个案：个案编号、咨询主题、问题类型、所属学院、提交时间和状态维护
- 咨询室管理：咨询室编号、咨询室名称、咨询类型、所属校区、接待上限和状态维护
- 学生档案：学生编号、学生姓名、所在班级、重点标签、联系电话和状态维护
- 值班安排：排班编号、值班老师、值班日期、值班时段、生效时间和状态维护
- 预约申请：预约编号、咨询主题、申请人、预约时间、预约状态和状态维护
- 咨询记录：记录编号、咨询主题、咨询老师、咨询时间、咨询结论和状态维护
- 测评问卷：问卷编号、咨询主题、问卷名称、提交时间、适用对象和状态维护
- 风险评估：评估编号、咨询主题、评估老师、风险等级、评估时间和状态维护
- 危机干预：干预编号、咨询主题、干预类型、干预对象、干预结果和状态维护
- 家校沟通：沟通编号、咨询主题、沟通主题、沟通方式、沟通时间和状态维护
- 随访计划：随访编号、咨询主题、随访阶段、随访内容、计划时间和状态维护
- 通知公告：公告编号、咨询主题、公告类型、公告内容、发布时间和状态维护
- 操作日志：关键模块操作人、动作、对象、详情和结果记录

## 验收标准
- 具备 PRD、PLAN、后端、前端和 SQL 初始化脚本
- 后端端口、前端端口、数据库名、Redis DB 与项目编号一致
- 静态结构达到 14 张表、14 个实体、14 个 Mapper、16 个 Controller、17 个前端页面
- 无旧项目包名、标题、端口、数据库名残留
