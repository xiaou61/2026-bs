# 实现计划

## 问题陈述

构建一个围绕充电桩预约、充电服务和运维管理的前后端分离系统，覆盖充电站点、充电桩位、用户车辆、预约订单、充电记录、故障报修、维修工单和保养计划。

## 当前状态

`124-backend` 与 `124-frontend` 已完成充电桩预约平台的基础实现，当前进入精修阶段，重点核对预约履约、运维闭环、认证安全和文档准确性。

## 实施方案

### 第一阶段：后端开发
1. Spring Boot 2.7.18 + MyBatis 2.3.1 + PageHelper + Redis + JWT
2. 创建 `ev_charging_124` 数据库和 14 张业务表
3. 实现统一返回、异常处理、JWT 拦截、Redis 配置和分页查询
4. 实现 Entity、Mapper、Service 和 Controller
5. 实现预约下单、充电记录、故障报修、维修处理和收益统计
6. 实现首页统计和操作日志

### 第二阶段：前端开发
1. Vue3 + Vite + Element Plus + Pinia + Axios + ECharts
2. 登录页、布局页、首页看板
3. 充电站点、充电桩位、用户车辆、预约订单、充电记录、故障报修、维修工单、保养计划和日志页面
4. 业务按钮支持预约、充电、报修、维修和查看能耗

### 第三阶段：文档更新
1. 更新 `readme.md`
2. 更新 `readme_simple.md`
3. 更新候选清单

## 完成标准

1. 后端结构完整。
2. 前端结构完整。
3. SQL 包含演示数据。
4. 认证返回不暴露敏感字段。
5. 预约、充电、报修、维修和收益统计口径一致。
6. 文档描述与实际实现保持一致。
7. 不执行编译验证。
