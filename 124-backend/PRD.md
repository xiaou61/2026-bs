# 电动车充电桩预约与运维管理系统 PRD

## 1. 项目背景
电动车充电基础设施快速增长，平台需要统一管理充电站、充电桩、用户车辆、预约订单、充电记录、故障报修和运维收益。

## 2. 用户角色
系统提供四类默认演示角色：ADMIN（admin / 123456）、OPERATOR（operator / 123456）、MAINTAINER（maintainer / 123456）、OWNER（owner / 123456）。

## 3. 功能模块
1. 账号权限：管理员、运营员、维修员、车主四角色账号、JWT 鉴权和 Redis 登录态。
2. 充电站点：站点编号、名称、城市、地址、运营商和开放状态管理。
3. 充电桩位：桩位编号、所属站点、功率、接口类型、费率和桩位状态管理。
4. 用户车辆：车辆编号、车主、车牌、电池容量、车型和绑定状态管理。
5. 预约订单：预约单号、车主、站点、桩位、预约时间和预计时长管理。
6. 充电记录：充电编号、预约单、桩位、开始时间、充电电量和充电金额管理。
7. 故障报修：报修编号、桩位、故障类型、严重等级、上报人和上报时间管理。
8. 维修工单：工单编号、报修编号、维修员、处理方案、截止时间和维修结果管理。
9. 保养计划：计划编号、站点、桩位、保养类型、负责人和计划时间管理。
10. 电价策略：策略编号、站点、时段类型、电价、服务费和生效日期管理。
11. 支付记录：支付编号、充电编号、车主、支付金额、支付方式和支付时间管理。
12. 收益统计：统计编号、站点、日期、订单数、充电电量和收益金额管理。
13. 能耗监测：监测编号、站点、采集时间、用电量、负载率和异常类型管理。
14. 操作日志：操作人、模块、动作、对象、详情和IP地址审计。

## 4. 核心流程
1. 管理员维护账号、基础档案和业务主数据。
2. 业务人员登记监测、预约、故障或任务记录并推进状态流转。
3. 管理人员查看预警、调度、收益和运维指标。
4. 系统通过首页看板展示趋势、分布和关键数量。
5. 操作日志沉淀关键动作，方便答辩时展示闭环。

## 5. 非功能要求
- 后端采用 Spring Boot，前端采用 Vue3。
- 数据库使用 MySQL，默认账号密码为 root / 1234。
- 登录态使用 JWT 与 Redis 校验。
- 数据页支持分页查询、关键词检索、状态筛选、新增、编辑、删除和状态动作。
