# 企业碳排放核算与减排任务管理系统 开发计划

## 实施范围
- 后端工程：实体、Mapper、Service、Controller、统一响应、JWT 登录、Redis Token 校验、统计接口
- 前端工程：登录页、布局菜单、通用数据页、模块页面、统计看板、接口封装和路由守卫
- 数据脚本：MySQL 建库建表、初始化账号和演示业务数据

## 模块拆分
- 企业档案：企业编号、名称、行业、区域、联系人和状态维护
- 排放因子：排放因子编号、名称、能源类型、因子值和单位维护
- 核算周期：核算周期、企业边界、核算月份和复核人维护
- 能源消耗：能源消耗记录、企业、能源类型、消耗量和单位维护
- 排放记录：排放编号、企业、范围类型、碳排放量和来源维护
- 减排任务：减排任务、目标减排量、责任人和执行状态维护
- 减排措施：减排措施、投资金额、预期减排量和执行状态维护
- 碳配额：碳配额编号、企业、年度、配额量和已使用量维护
- 核查报告：核查报告、企业、报告月份、核查员和结论维护
- 佐证附件：附件编号、企业、文件名、文件类型、上传人和状态维护
- 预警规则：预警规则、指标名称、阈值、等级和启用状态维护
- 碳排预警：碳排预警、企业、预警等级、触发原因和处理人维护
- 操作日志：关键模块操作人、动作、对象、详情和结果记录

## 验收标准
- 具备 PRD、PLAN、后端、前端和 SQL 初始化脚本
- 后端端口、前端端口、数据库名、Redis DB 与项目编号一致
- 静态结构达到 14 张表、14 个实体、14 个 Mapper、16 个 Controller、17 个前端页面
- 无旧项目包名、标题、端口、数据库名残留
