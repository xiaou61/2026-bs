# ESG 数据填报与可视化报告系统 开发计划

## 实施范围
- 后端工程：实体、Mapper、Service、Controller、统一响应、JWT 登录、Redis Token 校验、统计接口
- 前端工程：登录页、布局菜单、通用数据页、模块页面、统计看板、接口封装和路由守卫
- 数据脚本：MySQL 建库建表、初始化账号和演示业务数据

## 模块拆分
- 指标库：ESG 指标编号、名称、维度、口径、权重和状态维护
- 披露模板：披露模板编号、名称、适用行业、版本和负责人维护
- 报告周期：报告周期、公司、报告月份、模板和填报状态维护
- 企业填报：企业填报单、公司、填报人、提交时间和状态维护
- 指标数据：指标数据、指标名称、填报值、单位和数据来源维护
- 佐证材料：佐证材料、文件名称、文件类型、上传人和审核状态维护
- 审核任务：审核任务、填报编号、审核员、截止时间和审核状态维护
- 评分模型：评分模型、维度、评分方法、权重和启用状态维护
- ESG评分：ESG 评分、公司、环境分、社会分、治理分和等级维护
- 改进任务：改进任务、责任部门、整改事项、截止时间和执行状态维护
- 利益相关方反馈：反馈编号、相关方、反馈类型、反馈内容和处理状态维护
- 报告导出：报告导出、公司、报告月份、导出格式和导出状态维护
- 操作日志：关键模块操作人、动作、对象、详情和结果记录

## 验收标准
- 具备 PRD、PLAN、后端、前端和 SQL 初始化脚本
- 后端端口、前端端口、数据库名、Redis DB 与项目编号一致
- 静态结构达到 14 张表、14 个实体、14 个 Mapper、16 个 Controller、17 个前端页面
- 无旧项目包名、标题、端口、数据库名残留
