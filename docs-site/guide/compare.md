# 对比选择指南

> 面对 200 个毕设项目，选择哪个项目做可能让人头疼。使用下面的筛选和对比工具，根据不同维度快速找到最适合的项目。

## 如何选择毕设项目

### 按复杂度选择

| 复杂度 | 说明 | 适合人群 |
|--------|------|----------|
| **简单** | API 少、模块少、业务逻辑清晰 | 基础较弱、时间紧张 |
| **中等** | 有一定规模、含多个业务模块 | 有一定基础、正常准备 |
| **复杂** | 模块多、业务闭环长、数据量大 | 基础扎实、准备充分 |

### 按需求场景选择

| 场景 | 建议分类 | 关键词 |
|------|----------|--------|
| 想做校园相关 | 校园生活类 | 校园、学生、选课、请假 |
| 想做电商交易 | 电商交易类 | 购物、订单、商城、团购 |
| 想做医疗健康 | 医疗健康类 | 医院、预约、药品、养老 |
| 想做企业管理 | 企业管理类 | ERP、OA、招聘、面试 |
| 想做社区服务 | 社区服务类 | 物业、停车、家政、缴费 |
| 想做文化娱乐 | 文化娱乐类 | 非遗、短视频、艺术品 |

### 答辩建议

1. **功能闭环** — 选择功能模块完整、流程闭环的项目，答辩时可以展示完整业务流程
2. **技术栈匹配** — 选择与你熟悉的技术栈匹配的项目，降低开发难度
3. **代码量适中** — 不要选太简单的（没什么可讲），也不要选太复杂的（来不及做完）
4. **有特色亮点** — 选择有独特功能或特色的项目，答辩时更有差异化优势
