## 项目简介

本系统面向直播基地运营、主播排班、商品选品、样品借还、直播计划、直播场次和直播复盘场景，提供直播间档案、主播档案、商家档案、商品选品、样品台账、样品借还、主播排班、选品评测、直播计划、直播场次和直播复盘的一体化后台管理能力。

## 角色权限

系统支持 **6** 种角色，各角色权限如下：

### - 系统管理员

- 账号权限
- 基础配置和全部数据维护

### - 基地运营

- 维护直播间档案
- 主播档案
- 主播排班
- 直播计划和统计规则

### - 主播

- 查看排班
- 参与样品借还
- 执行直播场次并提交直播复盘

### - 选品专员

- 维护商品选品
- 选品评测
- 直播计划和样品协同

### - 样品管理员

- 维护样品台账
- 样品借还
- 样品归还和状态流转

### - 商家代表

- 维护商家档案
- 商品提报
- 样品寄送和直播复盘反馈

- 使用 MyBatis 注解 SQL + PageHelper 分页查询
- 各业务模块支持列表查询、新增编辑、删除、状态处理和状态完成
- 首页看板展示核心指标、近7日趋势和状态分布
- Spring Boot + Vue3 前后端分离
- MyBatis 注解 SQL + PageHelper 分页查询
- JWT 登录态拦截与 Redis Token 校验

- ECharts 展示业务趋势和状态分布
- Element Plus 通用数据页复用，统一搜索、表格、分页、弹窗和状态动作交互
- 直播基地主播排班与选品样品管理系统类毕设项目
- 直播电商、业务审批、台账管理和统计分析课程设计
- 多角色后台管理系统综合实战
- 需要展示完整业务闭环和可视化看板能力的答辩项目

## 快速启动

### 环境要求

| 工具 | 版本 |
|------|------|
| JDK | 8+ / 11+ |
| Maven | 3.8+ |
| MySQL | 8.0+ |
| Node.js | 18+ |

### 后端启动

```bash
cd 177-backend
# 1. 创建数据库并执行 init.sql
# 2. 修改 application.yml 数据库配置
# spring.datasource.url=jdbc:mysql://localhost:3306/live_base_177
# spring.datasource.username=root
# spring.datasource.password=你的密码
mvn spring-boot:run
```

### 前端启动

```bash
cd 177-frontend
npm install
# 修改后端接口地址（通常在 .env.development 或 src/utils/request.js 中）
npm run dev
```
