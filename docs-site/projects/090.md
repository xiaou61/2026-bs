### - 管理员

- admin
- 负责基础组织
- 剧目
- 排期
- 公告与统计

### - 艺术家

- artist
- 负责数字资源
- 签到
- 研学评分与赏析互动结果查看

### - 会员

- member
- 负责预约
- 查看行程
- 查看资源
- 查看研学评分
- 提交赏析互动

## 特色亮点

- 戏曲文化主题鲜明，适合做非遗数字化传承与文化运营类毕设
- 管理员、艺术家、会员三角色视角清晰，方便答辩时演示差异化流程
- 形成“剧目维护-排期预约-现场签到-研学评分-赏析互动”完整业务链路
- MyBatis原生XML + PageHelper 便于展示 SQL 能力与分页实现
- 看板聚合剧目、排期、预约与公告数据，演示效果直观

## 演示流程

1. 管理员维护剧种分类、名家档案、场馆和文化专题
2. 管理员发布经典剧目与展演排期，设置开放预约名额
3. 会员登录后浏览剧目详情，完成预约并查看个人行程
4. 艺术家发布数字资源，现场登记签到并录入研学评分
5. 会员参与活动后提交赏析互动，后台同步查看统计看板

## 快速启动

### 环境要求

| 工具 | 版本 |
|------|------|
| JDK | 8+ / 11+ |
| Maven | 3.8+ |
| MySQL | 8.0+ |
| Node.js | 18+ |

### 后端启动

```bash
cd 090-backend
# 1. 创建数据库并执行 init.sql
# 2. 修改 application.yml 数据库配置
# spring.datasource.url=jdbc:mysql://localhost:3306/Shanghai
# spring.datasource.username=root
# spring.datasource.password=你的密码
mvn spring-boot:run
```

### 前端启动

```bash
cd 090-frontend
npm install
# 修改后端接口地址（通常在 .env.development 或 src/utils/request.js 中）
npm run dev
```
