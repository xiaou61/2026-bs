## 项目简介

本系统面向博物馆、非遗馆、展览馆等文旅场馆，提供票务预约、讲解排期、扫码核销、客流统计、游客评价和公告发布的一体化后台管理能力。

## 技术栈

### 后端

- Spring Boot 2.7.18
- MyBatis 2.3.1 注解 SQL
- PageHelper 1.4.7
- MySQL 8.0
- Redis 登录态缓存
- JWT 认证

### 前端

- Vue 3.4.0
- Element Plus 2.4.4
- Vite 5
- Pinia 2.1.7
- Axios 1.6.2
- ECharts 5.4.3

## 角色权限

系统支持 **5** 种角色，各角色权限如下：

### - 系统管理员

- 账号权限
- 基础配置和全部数据维护

### - 场馆主管

- 场馆档案
- 票种
- 排期
- 活动
- 公告和统计看板维护

### - 讲解员

- 讲解员档案
- 讲解排期
- 讲解预约和游客评价处理

### - 票务核销员

- 票务预约
- 扫码核销和客流统计维护

### - 预约游客

- 票务预约
- 讲解预约和评价反馈提交

## 特色亮点

- 选题贴合文旅服务方向，覆盖票务预约、讲解排期、扫码核销、客流统计
- 形成“票种配置-票务预约-讲解排期-讲解预约-扫码核销-客流统计-游客评价-公告发布”完整业务闭环
- 使用 MyBatis 注解 SQL + PageHelper 分页查询
- 各业务模块支持列表查询、新增编辑、删除、状态处理和状态完成，并按角色收口操作按钮
- 首页看板展示票务预约、讲解排期、扫码核销、客流统计指标和状态分布
- Spring Boot + Vue3 前后端分离
- MyBatis 注解 SQL + PageHelper 分页查询
- JWT 登录态拦截与 Redis Token 校验
- 通用业务台账、状态流转和统计看板联动
- ECharts 展示业务趋势和状态分布
- Element Plus 通用数据页复用，统一搜索、表格、分页、弹窗和状态动作交互
- 文旅场馆讲解预约与票务核销管理平台类毕设项目
- 文旅服务、业务审批、台账管理和统计分析课程设计
- 多角色后台管理系统综合实战
- 需要展示完整业务闭环和可视化看板能力的答辩项目

## 快速启动

### 环境要求

| 工具 | 版本 |
|------|------|
| JDK | 8+ / 11+ |
| Maven | 3.8+ |
| MySQL | 8.0+ |
| Node.js | 18+ |

### 后端启动

```bash
cd 151-backend
# 1. 创建数据库并执行 init.sql
# 2. 修改 application.yml 数据库配置
# spring.datasource.url=jdbc:mysql://localhost:3306/Shanghai
# spring.datasource.username=root
# spring.datasource.password=你的密码
mvn spring-boot:run
```

### 前端启动

```bash
cd 151-frontend
npm install
# 修改后端接口地址（通常在 .env.development 或 src/utils/request.js 中）
npm run dev
```
