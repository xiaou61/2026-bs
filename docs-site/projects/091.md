## 项目简介

### 项目简介
电影院会员管理系统面向影院会员运营、门店服务和观影消费场景，覆盖会员档案、影片与门店资源、会员储值、优惠券、消费订单、电子观影码、评价反馈和数据看板等核心流程，帮助影院完成会员拉新、留存、复购和服务管理的一体化运营。

## 特色亮点

- 聚焦“电影院会员管理”而不是单纯售票，业务主题更适合会员运营类毕设答辩
- 会员编号、等级、积分、储值、消费等关键资产字段完整，展示会员体系更直观
- 保留影片、门店、影厅、场次、订单、电子观影码真实影院业务场景
- 管理员、门店员工、会员三角色职责清晰，便于演示权限差异和业务流转
- MyBatis-Plus + Redis + JWT 组合实现速度快，前后端结构统一清晰

## 演示流程

1. 管理员维护影片、门店、影厅和会员场次，设置推荐影片与售卖状态
2. 管理员或门店员工查看会员档案，维护等级、积分、余额和会员状态
3. 会员登录后完成储值、领取优惠券，在会员订票中心锁座并生成消费订单
4. 会员使用余额支付后系统生成电子观影码，门店员工在后台完成核销
5. 会员提交观影评价，管理端审核并通过运营看板查看消费与储值趋势

## 快速启动

### 环境要求

| 工具 | 版本 |
|------|------|
| JDK | 8+ / 11+ |
| Maven | 3.8+ |
| MySQL | 8.0+ |
| Node.js | 18+ |

### 后端启动

```bash
cd 091-backend
# 1. 创建数据库并执行 init.sql
# 2. 修改 application.yml 数据库配置
# spring.datasource.url=jdbc:mysql://localhost:3306/Shanghai
# spring.datasource.username=root
# spring.datasource.password=你的密码
mvn spring-boot:run
```

### 前端启动

```bash
cd 091-frontend
npm install
# 修改后端接口地址（通常在 .env.development 或 src/utils/request.js 中）
npm run dev
```
