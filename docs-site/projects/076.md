## 演示流程

1. 管理员登录系统，查看平台看板与公告概览
2. 管理员维护信息分类，设置分类排序和启停状态
3. 维护方发布企业信息，填写企业简称、所属行业、企业规模和处理方式
4. 申请人进入信息大厅，按关键词和分类筛选目标企业信息
5. 申请人创建处理单，填写处理日期/时间、联系电话、联系人和处理说明
6. 申请人支付处理单，维护方在处理工单中确认处理
7. 处理完成后，申请人确认完成并提交评价
8. 若存在争议，申请人可发起申诉，管理员介入回复处理
9. 用户可收藏高频企业信息，便于后续快速发起处理
10. 管理员持续查看趋势与排行数据，优化平台运营策略

## 快速启动

### 环境要求

| 工具 | 版本 |
|------|------|
| JDK | 8+ / 11+ |
| Maven | 3.8+ |
| MySQL | 8.0+ |
| Node.js | 18+ |

### 后端启动

```bash
cd 076-backend
# 1. 创建数据库并执行 init.sql
# 2. 修改 application.yml 数据库配置
# spring.datasource.url=jdbc:mysql://localhost:3306/Shanghai
# spring.datasource.username=root
# spring.datasource.password=你的密码
mvn spring-boot:run
```

### 前端启动

```bash
cd 076-frontend
npm install
# 修改后端接口地址（通常在 .env.development 或 src/utils/request.js 中）
npm run dev
```
