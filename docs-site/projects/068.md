## 项目简介

本项目是周边游平台个人管理模块后端服务，配套 `068-frontend` 使用。系统覆盖统一登录、用户资料、景点浏览、出行人管理、景点收藏、旅游订单、评价投诉和运营看板等流程。

## 角色权限

系统支持 **2** 种角色，各角色权限如下：

### - 管理员

- 用户管理
- 景点管理
- 订单全局查看与导出
- 评价审核回复
- 投诉处理
- 运营统计与趋势分析

### - 普通用户

- 个人资料
- 出行人
- 收藏
- 订单
- 评价
- 投诉（支持附件）

## 功能模块

| 模块 | 说明 |
|------|------|
| **用户与认证** | 登录注册、JWT鉴权、退出登录、密码修改 |
| **用户管理** | 管理员分页维护账号、角色、状态与资料信息 |
| **景点管理** | 管理员维护景点信息、上下架与删除 |
| **景点图片上传** | 管理端支持上传封面图片并预览 |
| **个人中心** | 个人资料维护、头像与简介编辑 |
| **常用出行人** | 出行人新增编辑、默认出行人设置 |
| **景点收藏** | 景点查询、收藏与取消收藏 |
| **订单管理** | 订单创建、支付、取消、完成、管理员全局查看 |
| **订单导出** | 用户和管理员按筛选条件导出CSV订单数据 |
| **评价管理** | 用户提交订单评价、敏感词待审核、管理员审核回复 |
| **投诉管理** | 用户提交投诉、附件上传、管理员处理闭环 |
| **数据看板** | 用户个人指标、近7日订单趋势、状态分布图展示 |

## 演示流程

1. 用户注册或使用 user/123456 登录进入个人中心
2. 在常用出行人维护联系人信息并设置默认出行人
3. 浏览景点并收藏感兴趣的目的地
4. 创建订单并完成支付、完结流程
5. 对已完成订单提交评价并查看审核与管理员回复
6. 发起投诉并上传附件，由管理员账号处理形成闭环
7. 在看板查看近7日订单趋势与状态分布图

## 快速启动

### 环境要求

| 工具 | 版本 |
|------|------|
| JDK | 8+ / 11+ |
| Maven | 3.8+ |
| MySQL | 8.0+ |
| Node.js | 18+ |

### 后端启动

```bash
cd 068-backend
# 1. 创建数据库并执行 init.sql
# 2. 修改 application.yml 数据库配置
# spring.datasource.url=jdbc:mysql://localhost:3306/Shanghai
# spring.datasource.username=root
# spring.datasource.password=你的密码
mvn spring-boot:run
```

### 前端启动

```bash
cd 068-frontend
npm install
# 修改后端接口地址（通常在 .env.development 或 src/utils/request.js 中）
npm run dev
```
