## 特色亮点

- 以铁路场景重构订票闭环，演示路线比普通票务系统更贴近日常生活
- 使用 Redis 做座位锁定，避免多人同时下单造成超卖
- 一个系统覆盖后台资料维护、调度审核、乘客购票三端角色
- 电子票、乘车人、退改签等业务链路完整，适合答辩展示
- 统计看板聚合销售趋势、订单总量与热门车次排行

## 演示流程

1. 管理员维护车站、列车和车厢模板基础数据
2. 调度员创建班次并配置票价、发车时间和售卖状态
3. 乘客维护常用乘车人后进入余票选座页提交订单
4. 乘客完成余额支付，系统自动生成电子车票
5. 乘客提交退改签申请，调度员在后台审核处理并查看统计看板

### 后端启动

```bash
cd 089-backend
# 1. 创建数据库并执行 init.sql
# 2. 修改 application.yml 数据库配置
# spring.datasource.url=jdbc:mysql://localhost:3306/railway_ticket_089
# spring.datasource.username=root
# spring.datasource.password=你的密码
mvn spring-boot:run
```

### 前端启动

```bash
cd 089-frontend
npm install
# 修改后端接口地址（通常在 .env.development 或 src/utils/request.js 中）
npm run dev
```
