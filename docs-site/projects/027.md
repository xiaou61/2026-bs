## 项目简介

一个完整的在线理发预约系统，采用前后端一体化架构（Spring Boot + Vue 3），支持用户预约、会员积分、订单支付、评价等核心功能。

## 角色权限

系统支持 **2** 种角色，各角色权限如下：

### 用户

- 1. **用户注册登录** - 手机号注册、JWT认证
- 2. **门店管理** - 门店列表、详情、搜索、排序
- 3. **理发师管理** - 理发师列表、详情、评分展示

- 5. **在线预约** - 4步预约流程、时段冲突检测、7天提前预约
- 6. **预约管理** - 我的预约、取消预约、状态查询
- 7. **会员体系** - 四级会员（普通/银卡/金卡/钻石）、自动升级、专享折扣
- 8. **积分系统** - 消费积分、签到积分、评价积分
- ...还有 2 项权限

### 管理端（核心后端接口）

- 预约确认/取消
- 预约状态流转（6种状态）
- 订单管理
- 评价管理

## 特色亮点

- 智能预约系统：4步预约流程、时段冲突自动检测、最多提前7天预约
- 四级会员体系：普通(无折扣)→银卡(9.5折)→金卡(9折)→钻石(8.5折)，积分自动升级
- 预约限制规则：单用户最多3个未完成预约、超时15分钟自动取消
- 6种状态流转：待确认→已确认→服务中→已完成/已取消/超时取消
- 订单自动生成：服务完成后自动创建订单、应用会员折扣、支付后添加积分
- 评价联动更新：用户评价后自动更新理发师和门店平均评分
- 前后端一体化：Vite打包输出到Spring Boot的static目录，单JAR部署
- 完整积分体系：消费1元=1积分、签到积分、评价积分、积分历史记录

- 1. 前后端一体化部署 - Vite打包输出到Spring Boot的static目录，单JAR部署
- 2. JWT无状态认证 - Token认证，支持用户登录状态保持
- 3. 智能预约系统 - 时段冲突检测、自动取消超时预约
- 4. 会员积分体系 - 消费积分、自动升级、会员折扣
- 5. 订单支付流程 - 余额支付、会员折扣、积分奖励
- 6. 评价系统 - 自动更新理发师和门店评分
- 7. 响应式设计 - Element Plus组件适配各种屏幕
- 8. RESTful API - 统一的API设计规范

## 演示流程

1. 用户手机号注册登录 → 进入首页
2. 浏览门店列表 → 选择心仪门店 → 查看门店详情
3. 查看门店理发师列表 → 选择理发师 → 查看评分和评价
4. 选择服务项目 → 选择预约日期和时段
5. 系统检查时段冲突 → 确认预约 → 生成预约记录
6. 管理员确认预约 → 用户到店服务
7. 服务完成 → 自动生成订单 → 应用会员折扣
8. 用户余额支付 → 获得消费积分 → 积分自动升级会员等级
9. 用户发表评价 → 上传图片 → 选择标签
10. 系统自动更新理发师和门店评分

### 后端启动

```bash
cd 027-backend
# 1. 创建数据库并执行 init.sql
# 2. 修改 application.yml 数据库配置
# spring.datasource.url=jdbc:mysql://localhost:3306/db_027
# spring.datasource.username=root
# spring.datasource.password=你的密码
mvn spring-boot:run
```

### 前端启动

```bash
cd 027-frontend
npm install
# 修改后端接口地址（通常在 .env.development 或 src/utils/request.js 中）
npm run dev
```
