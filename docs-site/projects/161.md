## 项目简介

本系统面向景区游客失物登记、拾物上报、身份核验与寻回协同场景，提供景区区域、失物登记、拾物上报、游客认领、身份核验、位置追踪、暂存保管、通知协同、归还交接、寻回任务和回访评价的一体化后台管理能力。

- 通知协同

- 失物登记
- 拾物上报
- 游客认领和寻回进度查看

## 特色亮点

- 选题贴合景区游客服务方向，覆盖失物登记、拾物上报、身份核验、暂存保管、归还交接和回访评价
- 形成“景区区域-失物登记-拾物上报-游客认领-身份核验-位置追踪-暂存保管-通知协同-归还交接-寻回任务-回访评价”完整业务闭环
- 使用 MyBatis 注解 SQL + PageHelper 分页查询
- 各业务模块支持列表查询、新增编辑、删除、状态处理和寻回完成
- 首页看板展示登记寻回趋势、认领状态分布和游客服务核心指标
- Spring Boot + Vue3 前后端分离
- MyBatis 注解 SQL + PageHelper 分页查询
- JWT 登录态拦截与 Redis Token 校验
- 按游客服务员、安保巡查员、景区运营员、通知协同员、游客用户进行菜单、路由和操作权限收口
- ECharts 展示失物登记趋势、认领核验状态和寻回任务概览
- Element Plus 通用数据页复用，统一搜索、表格、分页、弹窗和寻回状态动作交互
- 景区失物招领与游客寻回协同平台类毕设项目
- 旅游服务、失物招领、游客认领、位置追踪和服务回访课程设计
- 多角色后台管理系统综合实战
- 需要展示完整业务闭环和可视化看板能力的答辩项目

## 快速启动

### 环境要求

| 工具 | 版本 |
|------|------|
| JDK | 8+ / 11+ |
| Maven | 3.8+ |
| MySQL | 8.0+ |
| Node.js | 18+ |

### 后端启动

```bash
cd 161-backend
# 1. 创建数据库并执行 init.sql
# 2. 修改 application.yml 数据库配置
# spring.datasource.url=jdbc:mysql://localhost:3306/Shanghai
# spring.datasource.username=root
# spring.datasource.password=你的密码
mvn spring-boot:run
```

### 前端启动

```bash
cd 161-frontend
npm install
# 修改后端接口地址（通常在 .env.development 或 src/utils/request.js 中）
npm run dev
```
