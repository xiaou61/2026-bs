## 特色亮点

- 选题贴合智慧农业、水产养殖、水质监测、投喂预警、病害处置和产量统计方向
- 形成“养殖池塘-传感设备-水质读数-投喂计划-投喂记录-鱼苗批次-生长采样-病害预警-用药记录-养殖设备-水质规则-产量统计”完整闭环
- 使用 MyBatis-Plus 分页查询和通用 CRUD 服务复用，与 128 的 MyBatis 注解 SQL 路线继续交替
- 水质读数、投喂记录、生长采样、病害预警、用药记录和产量统计都有状态流转动作
- 首页看板展示养殖池塘、水质读数、投喂记录、病害预警指标，以及水质监测趋势和养殖品类分布

## 演示流程

1. 管理员维护账号权限、养殖池塘、传感设备和水质规则
2. 水质技术员采集水质读数并查看异常状态
3. 养殖员维护投喂计划并登记投喂记录
4. 养殖经理跟踪鱼苗批次、生长采样和产量统计
5. 系统根据水质规则生成病害预警
6. 养殖员登记用药记录并处理养殖设备状态
7. 首页看板查看水质趋势、养殖品类和投喂预警指标

## 快速启动

### 环境要求

| 工具 | 版本 |
|------|------|
| JDK | 8+ / 11+ |
| Maven | 3.8+ |
| MySQL | 8.0+ |
| Node.js | 18+ |

### 后端启动

```bash
cd 129-backend
# 1. 创建数据库并执行 init.sql
# 2. 修改 application.yml 数据库配置
# spring.datasource.url=jdbc:mysql://localhost:3306/Shanghai
# spring.datasource.username=root
# spring.datasource.password=你的密码
mvn spring-boot:run
```

### 前端启动

```bash
cd 129-frontend
npm install
# 修改后端接口地址（通常在 .env.development 或 src/utils/request.js 中）
npm run dev
```
