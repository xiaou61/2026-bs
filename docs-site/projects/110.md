## 角色权限

系统支持 **4** 种角色，各角色权限如下：

### - 管理员

- 维护账号
- 数据主体
- 数据项和操作日志

### - 隐私官

- 维护授权目的
- 授权策略
- 风险预警和审计报告

### - 数据使用人

- 提交访问申请并在授权范围内访问数据

### - 审计员

- 复核访问日志
- 撤销申请和审计报告

## 功能模块

| 模块 | 说明 |
|------|------|
| **账号权限** | 管理员/隐私官/数据使用人/审计员四角色登录、JWT鉴权和Redis登录态 |
| **数据主体** | 管理个人数据主体身份、联系方式、地区和认证状态 |
| **个人数据项** | 维护数据项分类、敏感等级、保存期限和负责人 |
| **授权目的** | 管理数据处理目的、业务场景、目的说明和有效期 |
| **授权策略** | 配置授权目的、数据范围、策略版本并支持提交、通过、归档 |
| **授权记录** | 记录个人授权编号、主体、渠道、授权时间、到期时间并支持生效、过期、撤销 |
| **授权范围** | 管理授权记录关联的数据项、范围类型、必要性和说明 |
| **访问申请** | 管理个人数据访问申请、申请人、访问目的并支持提交、通过、驳回 |
| **访问授权** | 管理审批后的访问令牌、被授权人、数据范围和有效期 |
| **访问日志** | 记录访问人、数据主体、数据项、访问结果和访问时间 |
| **撤销申请** | 管理个人授权撤销申请、撤销原因并支持通过、驳回、完成 |
| **风险预警** | 记录过期访问、无授权访问、高频访问、导出风险并支持确认、解决、关闭 |
| **审计报告** | 生成隐私访问审计、专项审计报告，支持发布和归档 |
| **操作日志** | 记录关键操作对象、动作、详情和IP地址 |

## 特色亮点

- 选题贴合个人信息保护、隐私合规和访问审计方向，适合数据治理类答辩
- 形成“数据主体-授权目的-授权记录-访问申请-访问日志-撤销申请-风险预警-审计报告”完整闭环
- 使用 MyBatis 注解 SQL + PageHelper，与 109 的 MyBatis-Plus 路线继续交替
- 授权策略、授权记录、访问申请、撤销申请、风险预警和审计报告都有清晰状态流转
- 首页看板展示数据主体、授权记录、访问日志、风险预警、授权趋势和风险类型分布

## 演示流程

1. 管理员维护账号权限、数据主体和个人数据项
2. 隐私官配置授权目的和授权策略并提交审批
3. 数据主体形成授权记录和授权范围
4. 数据使用人提交访问申请并生成访问授权
5. 系统沉淀访问日志并触发风险预警
6. 审计员处理撤销申请并发布审计报告
7. 首页看板查看授权趋势、风险分布和核心指标

## 快速启动

### 环境要求

| 工具 | 版本 |
|------|------|
| JDK | 8+ / 11+ |
| Maven | 3.8+ |
| MySQL | 8.0+ |
| Node.js | 18+ |

### 后端启动

```bash
cd 110-backend
# 1. 创建数据库并执行 init.sql
# 2. 修改 application.yml 数据库配置
# spring.datasource.url=jdbc:mysql://localhost:3306/Shanghai
# spring.datasource.username=root
# spring.datasource.password=你的密码
mvn spring-boot:run
```

### 前端启动

```bash
cd 110-frontend
npm install
# 修改后端接口地址（通常在 .env.development 或 src/utils/request.js 中）
npm run dev
```
