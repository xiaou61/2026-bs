## 功能模块

| 模块 | 说明 |
|------|------|
| **账号权限** | 管理员/合规专员/研发负责人/审计员四角色登录、JWT鉴权和Redis登录态 |
| **组织团队** | 维护研发团队、负责人、联系方式和团队说明 |
| **项目台账** | 登记项目名称、编码、仓库地址、技术语言、风险等级和最近扫描时间 |
| **分支版本** | 记录仓库分支、提交哈希、默认分支和启用状态 |
| **依赖台账** | 维护依赖名称、版本、包管理器、许可证、依赖范围和来源地址 |
| **许可证策略** | 配置许可证风险等级、商用兼容性和处置策略 |
| **合规基线** | 按项目维护允许许可证范围、要求等级和生效日期 |
| **扫描任务** | 创建依赖扫描任务，支持启动、完成和失败标记 |
| **扫描结果** | 沉淀依赖许可证识别结果、命中规则、证据说明和风险状态 |
| **风险问题** | 记录高危许可证、声明缺失、未知许可证等问题并支持确认或忽略 |
| **整改任务** | 分配整改责任人、方案、进度、截止日期并支持分配、完成和重开 |
| **审计报告** | 汇总扫描风险、整改结论，支持发布和归档 |
| **审批记录** | 管理临时豁免、报告发布和整改关闭审批记录 |
| **操作日志** | 记录关键操作对象、动作、详情和IP地址 |

## 特色亮点

- 选题贴合开源治理、软件供应链安全和企业研发合规方向，热门且实用
- 形成“项目登记-依赖识别-策略匹配-风险确认-整改跟踪-报告发布”完整闭环
- 使用 MyBatis 注解 SQL + PageHelper，与 103 的 MyBatis-Plus 路线继续交替
- 扫描任务、风险问题、整改任务、审计报告和审批记录都有清晰状态流转
- 首页看板展示项目、依赖、扫描、风险、报告和许可证分布，答辩演示链路清楚

## 演示流程

1. 管理员登录维护账号权限、组织团队和操作日志
2. 合规专员维护许可证策略和项目合规基线
3. 研发负责人登记项目仓库、分支版本和依赖台账
4. 合规专员创建扫描任务并启动，系统沉淀扫描结果和风险问题
5. 研发负责人确认风险问题并创建整改任务，更新整改进展
6. 审计员查看整改结果，审批豁免或报告发布申请
7. 合规专员发布审计报告，首页查看综合统计和许可证分布

| MySQL | 8.0+ |

### 后端启动

```bash
cd 104-backend
# 1. 创建数据库并执行 init.sql
# 2. 修改 application.yml 数据库配置
# spring.datasource.url=jdbc:mysql://localhost:3306/Shanghai
# spring.datasource.username=root
# spring.datasource.password=你的密码
mvn spring-boot:run
```

### 前端启动

```bash
cd 104-frontend
npm install
# 修改后端接口地址（通常在 .env.development 或 src/utils/request.js 中）
npm run dev
```
