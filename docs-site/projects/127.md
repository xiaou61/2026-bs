## 特色亮点

- 选题贴合双碳管理、企业碳核算、排放因子、减排任务、核查报告和预警治理方向
- 形成“企业档案-排放因子-核算周期-能源消耗-排放记录-减排任务-减排措施-碳配额-核查报告-佐证附件-预警规则-碳排预警”完整闭环
- 使用 MyBatis-Plus 分页查询和通用 CRUD 服务复用，与 126 的 MyBatis 注解 SQL 路线继续交替
- 核算周期、能源消耗、排放记录、减排任务、减排措施、核查报告和碳排预警都有状态流转动作
- 首页看板展示企业、排放记录、减排任务、碳排预警指标，以及碳排核算趋势和排放范围分布

## 演示流程

1. 管理员维护账号权限、企业档案、排放因子和预警规则
2. 碳核算员建立核算周期并登记能源消耗
3. 系统根据排放因子沉淀排放记录和范围分布
4. 核查员审核能源消耗、排放记录、佐证附件和核查报告
5. 减排经理制定减排任务、拆分减排措施并跟踪执行
6. 系统根据碳配额和规则生成碳排预警
7. 首页看板查看碳排趋势、范围分布和减排治理指标

## 快速启动

### 环境要求

| 工具 | 版本 |
|------|------|
| JDK | 8+ / 11+ |
| Maven | 3.8+ |
| MySQL | 8.0+ |
| Node.js | 18+ |

### 后端启动

```bash
cd 127-backend
# 1. 创建数据库并执行 init.sql
# 2. 修改 application.yml 数据库配置
# spring.datasource.url=jdbc:mysql://localhost:3306/Shanghai
# spring.datasource.username=root
# spring.datasource.password=你的密码
mvn spring-boot:run
```

### 前端启动

```bash
cd 127-frontend
npm install
# 修改后端接口地址（通常在 .env.development 或 src/utils/request.js 中）
npm run dev
```
