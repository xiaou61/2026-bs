## 项目简介

医院门诊检查预约与报告回传管理系统面向毕业设计答辩场景，提供多角色登录、业务台账维护、审批状态流转、统计看板和操作留痕能力。

## 功能模块

| 模块 | 说明 |
|------|------|
| **账号权限** | 系统管理员/门诊医生/检查技师/患者本人四角色登录、JWT鉴权和Redis登录态 |
| **检查项目** | 项目编号、项目名称、检查类别、适用科室、检查时长和状态维护 |
| **患者档案** | 患者编号、患者姓名、身份证号、联系电话、就诊卡号和状态维护 |
| **医生档案** | 医生编号、医生姓名、联系电话、所属科室、职称等级和状态维护 |
| **检查预约** | 预约编号、检查项目、预约日期、检查时段、开单医生和状态维护 |
| **检查科室** | 科室编号、科室名称、检查设备、开放时间、候诊区域和状态维护 |
| **签到记录** | 签到编号、患者姓名、签到方式、签到窗口、签到时间和状态维护 |
| **检查报告** | 报告编号、患者姓名、检查结论、完成时间、检查技师和状态维护 |
| **异常预警** | 预警编号、患者姓名、预警类型、预警时间、处理人和状态维护 |
| **报告回传** | 回传编号、患者姓名、回传方式、回传时间、接收人和状态维护 |
| **复诊建议** | 建议编号、患者姓名、建议主题、建议类型、建议时间和状态维护 |
| **队列叫号** | 叫号编号、患者姓名、叫号时间、候诊状态、诊室编号和状态维护 |
| **通知公告** | 通知编号、患者姓名、通知类型、通知内容、接收人和状态维护 |
| **操作日志** | 关键模块操作人、动作、对象、详情和结果记录 |

## 特色亮点

- 选题贴合门诊检查管理、预约登记、队列叫号、报告回传和异常提醒方向
- 形成“检查项目-患者档案-医生档案-检查预约-检查科室-签到记录-检查报告-异常预警-报告回传-复诊建议-队列叫号-通知公告”完整闭环
- 使用 MyBatis 注解 SQL + PageHelper 分页查询，与 149 的 MyBatis-Plus 路线继续交替
- 检查预约、签到记录、检查报告、异常预警、报告回传和队列叫号都有状态流转动作
- 首页看板展示项目、科室、报告、叫号指标，以及检查预约趋势和检查状态分布
- Spring Boot + Vue3 前后端分离
- MyBatis 注解 SQL + PageHelper 分页查询
- JWT 登录态拦截与 Redis Token 校验
- 检查预约、报告回传、异常预警和队列叫号状态机联动
- ECharts 展示检查预约趋势和检查状态分布
- Element Plus 通用数据页复用，统一搜索、表格、分页、弹窗和状态动作交互
- 医院门诊检查预约与报告回传管理系统类毕设项目
- 医疗信息化、门诊检查、报告流转、排队叫号和患者服务课程设计
- 多角色后台管理系统综合实战
- 需要展示门诊检查闭环、报告回传和队列管理能力的答辩项目

## 快速启动

### 环境要求

| 工具 | 版本 |
|------|------|
| JDK | 8+ / 11+ |
| Maven | 3.8+ |
| MySQL | 8.0+ |
| Node.js | 18+ |

### 后端启动

```bash
cd 150-backend
# 1. 创建数据库并执行 init.sql
# 2. 修改 application.yml 数据库配置
# spring.datasource.url=jdbc:mysql://localhost:3306/Shanghai
# spring.datasource.username=root
# spring.datasource.password=你的密码
mvn spring-boot:run
```

### 前端启动

```bash
cd 150-frontend
npm install
# 修改后端接口地址（通常在 .env.development 或 src/utils/request.js 中）
npm run dev
```
