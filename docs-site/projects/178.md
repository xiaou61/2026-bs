## 项目简介

本系统面向医院手术室器械包追踪、供应中心回收清点、灭菌批次记录、放行审核、手术使用和异常召回场景，提供手术室档案、器械包档案、器械明细、器械包追踪、灭菌批次、灭菌记录、放行审核、手术使用、回收清点、缺损上报和异常召回的一体化后台管理能力。

- PageHelper 1.4.7

### 前端

- Pinia 2.1.7

- ECharts 5.4.3

## 角色权限

系统支持 **6** 种角色，各角色权限如下：

### - 系统管理员

- 账号权限
- 基础配置和全部数据维护

### - 手术室护士

- 维护手术室档案
- 手术使用
- 回收清点和器械包追踪

### - 供应中心

- 维护器械包档案
- 器械明细
- 回收清点和灭菌批次

### - 灭菌人员

- 登记灭菌批次
- 灭菌记录
- 器械状态和缺损上报

### - 质控审核

- 执行放行审核
- 异常召回
- 缺损复核和统计规则维护

### - 手术医生

- 查看器械包追踪
- 手术使用记录和召回提醒

| **缺损上报** | 上报编号、缺损器械、缺损类型、上报人员、上报时间和缺损状态维护 |
| **异常召回** | 召回编号、召回器械包、召回类型、质控人员、召回时间和召回状态维护 |
| **操作日志** | 关键模块操作人、动作、对象、详情和结果记录 |

## 快速启动

### 环境要求

| 工具 | 版本 |
|------|------|
| JDK | 8+ / 11+ |
| Maven | 3.8+ |
| MySQL | 8.0+ |
| Node.js | 18+ |

### 后端启动

```bash
cd 178-backend
# 1. 创建数据库并执行 init.sql
# 2. 修改 application.yml 数据库配置
# spring.datasource.url=jdbc:mysql://localhost:3306/Shanghai
# spring.datasource.username=root
# spring.datasource.password=你的密码
mvn spring-boot:run
```

### 前端启动

```bash
cd 178-frontend
npm install
# 修改后端接口地址（通常在 .env.development 或 src/utils/request.js 中）
npm run dev
```
