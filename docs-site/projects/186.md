## 项目简介

本系统面向校园餐厅后厨留样、食品安全巡检、食材验收、消毒记录、问题整改和风险预警场景，提供餐厅档案、后厨区域、菜品档案、留样登记、留样存储、食材验收、消毒记录、卫生巡检、问题整改、风险提醒和厨余处置的一体化后台管理能力。

## 角色权限

系统支持 **6** 种角色，各角色权限如下：

### - 系统管理员

- 账号权限
- 基础配置和全部数据维护

### - 食堂管理员

- 维护餐厅档案
- 后厨区域
- 菜品档案和台账规则

### - 后厨负责人

- 登记菜品留样
- 留样存储
- 消毒记录和厨余处置

### - 卫生巡检员

- 执行卫生巡检
- 发现问题并跟踪整改闭环

### - 仓储管理员

- 维护食材验收
- 库存流转和风险提醒

### - 校方监管

- 查看留样
- 巡检
- 整改和风险统计

## 快速启动

### 环境要求

| 工具 | 版本 |
|------|------|
| JDK | 8+ / 11+ |
| Maven | 3.8+ |
| MySQL | 8.0+ |
| Node.js | 18+ |

### 后端启动

```bash
cd 186-backend
# 1. 创建数据库并执行 init.sql
# 2. 修改 application.yml 数据库配置
# spring.datasource.url=jdbc:mysql://localhost:3306/Shanghai
# spring.datasource.username=root
# spring.datasource.password=你的密码
mvn spring-boot:run
```

### 前端启动

```bash
cd 186-frontend
npm install
# 修改后端接口地址（通常在 .env.development 或 src/utils/request.js 中）
npm run dev
```
