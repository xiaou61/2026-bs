## 特色亮点

- 智能预约规则：提前最多2天预约、单次最长4小时、防止恶意占座
- 完善信用体系：正常使用+2分、违约扣分、低信用限制预约、四级等级制度
- 二维码签到：每个预约生成唯一二维码、15分钟签到有效、防作弊机制
- 自动化管理：超时自动释放、定时任务检测、智能通知推送
- 座位可视化：平面图选座、实时状态展示、座位类型标识、悬浮信息
- 数据可视化：ECharts图表展示、使用率趋势、热门座位排行、实时统计
- 移动端适配：响应式设计、触控优化、扫码便捷、随时随地预约
- 权限分离：学生端预约、管理端监控、系统管理员配置三端独立

## 演示流程

1. 学生注册登录 → 完善个人信息 → 获得初始100信用分
2. 浏览自习室列表 → 查看实时座位状态 → 选择目标自习室
3. 进入座位选择页 → 平面图选座 → 选择合适的座位和时间段
4. 确认预约信息 → 提交预约 → 生成二维码取件凭证
5. 预约开始前到达 → 扫码签到 → 15分钟内完成签到验证
6. 正常使用结束 → 主动签退或自动签退 → 获得+2信用分
7. 查看预约记录 → 个人中心统计 → 信用分历史查看
8. 管理员监控 → 实时数据看板 → 处理异常预约和违约情况

## 快速启动

### 环境要求

| 工具 | 版本 |
|------|------|
| JDK | 17+ |
| Maven | 3.8+ |
| MySQL | 8.0+ |
| Node.js | 18+ |

### 后端启动

```bash
cd 022-backend
# 1. 创建数据库并执行 init.sql
# 2. 修改 application.yml 数据库配置
# spring.datasource.url=jdbc:mysql://localhost:3306/study_room
# spring.datasource.username=root
# spring.datasource.password=你的密码
mvn spring-boot:run
```

### 前端启动

```bash
cd 022-frontend
npm install
# 修改后端接口地址（通常在 .env.development 或 src/utils/request.js 中）
npm run dev
```
