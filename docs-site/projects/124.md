## 特色亮点

- 选题贴合新能源充电、充电桩预约、故障运维、收益统计和能耗监测方向
- 形成“充电站点-充电桩位-用户车辆-预约订单-充电记录-故障报修-维修工单-保养计划-电价策略-支付记录-收益统计-能耗监测”完整闭环
- 使用 MyBatis 注解 SQL + PageHelper 分页查询，与 123 的 MyBatis-Plus 路线继续交替
- 预约订单、充电记录、故障报修、维修工单、保养计划、支付记录和能耗监测都有状态流转动作
- 首页看板展示站点、桩位、预约、故障指标，以及预约充电趋势和桩位状态分布

## 演示流程

1. 管理员维护账号权限、充电站点、充电桩位和电价策略
2. 车主绑定用户车辆并提交预约订单
3. 运营员跟踪充电记录、支付记录和收益统计
4. 车主或运营员提交故障报修
5. 维修员接收维修工单并执行保养计划
6. 系统沉淀能耗监测记录和异常预警
7. 首页看板查看预约趋势、桩位状态和收益运维指标

## 快速启动

### 环境要求

| 工具 | 版本 |
|------|------|
| JDK | 8+ / 11+ |
| Maven | 3.8+ |
| MySQL | 8.0+ |
| Node.js | 18+ |

### 后端启动

```bash
cd 124-backend
# 1. 创建数据库并执行 init.sql
# 2. 修改 application.yml 数据库配置
# spring.datasource.url=jdbc:mysql://localhost:3306/Shanghai
# spring.datasource.username=root
# spring.datasource.password=你的密码
mvn spring-boot:run
```

### 前端启动

```bash
cd 124-frontend
npm install
# 修改后端接口地址（通常在 .env.development 或 src/utils/request.js 中）
npm run dev
```
