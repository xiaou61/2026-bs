## 项目简介

电子合同签署与印章审批管理系统面向毕业设计答辩场景，提供多角色登录、业务台账维护、审批状态流转、统计看板和操作留痕能力。

### 前端

## 特色亮点

- 选题贴合电子合同流转、用印审批、在线签署、归档检索和风险条款管理方向
- 形成“合同模板-对方单位-签署方档案-合同草稿-用印申请-审批流程-合同签署-用印记录-归档记录-到期提醒-风险条款-合同通知”完整闭环
- 使用 MyBatis 注解 SQL + PageHelper 分页查询，与 139 的 MyBatis-Plus 路线继续交替
- 用印申请、审批流程、合同签署、归档记录、到期提醒、风险条款和合同通知都有状态流转动作
- 首页看板展示合同、审批、签署、归档指标，以及审批趋势和合同类型分布
- Spring Boot + Vue3 前后端分离
- MyBatis 注解 SQL + PageHelper 分页查询
- JWT 登录态拦截与 Redis Token 校验
- 用印审批、合同签署、到期提醒和风险处置状态机联动
- ECharts 展示审批趋势和合同类型分布
- Element Plus 通用数据页复用，统一搜索、表格、分页、弹窗和状态动作交互
- 电子合同签署与印章审批管理系统类毕设项目
- 合同管理、法务流程、审批流、档案管理和风险控制课程设计
- 多角色后台管理系统综合实战
- 需要展示合同签署闭环、审批轨迹、归档检索和到期提醒能力的答辩项目

## 快速启动

### 环境要求

| 工具 | 版本 |
|------|------|
| JDK | 8+ / 11+ |
| Maven | 3.8+ |
| MySQL | 8.0+ |
| Node.js | 18+ |

### 后端启动

```bash
cd 140-backend
# 1. 创建数据库并执行 init.sql
# 2. 修改 application.yml 数据库配置
# spring.datasource.url=jdbc:mysql://localhost:3306/Shanghai
# spring.datasource.username=root
# spring.datasource.password=你的密码
mvn spring-boot:run
```

### 前端启动

```bash
cd 140-frontend
npm install
# 修改后端接口地址（通常在 .env.development 或 src/utils/request.js 中）
npm run dev
```
