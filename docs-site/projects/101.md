## 项目简介

`101-backend` 是 101 号项目的 Spring Boot 后端，提供认证鉴权、候选人档案、简历与证书、岗位与岗位要求、解析任务、匹配任务、面试排期、面试反馈、统计看板和操作日志接口。

**前端：** `101-frontend` 是 101 号项目的 Vue 3 管理端，按角色提供管理员、HR、候选人和面试官四类入口。

## 功能模块

| 模块 | 说明 |
|------|------|
| **账号权限** | 管理员/HR/候选人/面试官四角色登录、JWT鉴权和Redis登录态 |
| **候选人档案** | 维护姓名、学历、工作年限、目标岗位、技能标签和期望薪资 |
| **简历材料** | 管理简历文件地址、简历文本、学历、技能、经验和解析状态 |
| **证书材料** | 管理证书名称、类型、发证机构、材料地址和HR核验结果 |
| **岗位管理** | 维护岗位名称、部门、类型、地点、薪资、描述和发布状态 |
| **岗位要求** | 按岗位配置技能、学历、经验、证书等关键词与权重 |
| **解析任务** | 创建并启动简历解析任务，生成结构化解析结果 |
| **解析结果** | 查看学历、技能、经验、分数、结论并支持人工复核 |
| **匹配任务** | 为候选人和岗位创建匹配任务，启动后生成推荐结果 |
| **匹配结果** | 展示匹配技能、缺失技能、匹配分、推荐等级和复核意见 |
| **面试排期** | 安排面试官、时间、方式、地址并支持确认、取消、完成 |
| **面试反馈** | 面试官填写评分、优势、不足、建议和面试结果 |
| **统计审计** | 首页指标、图表看板和关键操作日志 |

## 特色亮点

- 选题贴合AI招聘、简历解析、人才画像和智能岗位推荐热点，适合人工智能应用方向答辩
- 形成“候选人建档-材料上传-证书核验-简历解析-岗位匹配-面试安排-反馈归档”完整闭环
- 解析任务会根据工作年限、技能数量和已核验证书生成材料完整度评分
- 匹配任务会按岗位要求权重计算匹配技能、缺失技能、匹配分和推荐等级
- 四角色演示路径清晰，既能展示HR工作台，也能展示候选人材料和面试官反馈

## 演示流程

1. 管理员登录维护用户账号并查看操作日志
2. 候选人维护个人档案、简历材料和证书材料
3. HR核验证书，通过或驳回证书并填写核验意见
4. HR维护岗位和岗位要求，发布或关闭招聘岗位
5. HR创建解析任务并启动，系统生成解析结果和材料评分
6. HR创建匹配任务并启动，系统生成岗位匹配结果和推荐等级
7. HR安排面试计划，面试官填写面试反馈，首页查看整体招聘指标

## 快速启动

### 环境要求

| 工具 | 版本 |
|------|------|
| JDK | 8+ / 11+ |
| Maven | 3.8+ |
| MySQL | 8.0+ |
| Node.js | 18+ |

### 后端启动

```bash
cd 101-backend
# 1. 创建数据库并执行 init.sql
# 2. 修改 application.yml 数据库配置
# spring.datasource.url=jdbc:mysql://localhost:3306/Shanghai
# spring.datasource.username=root
# spring.datasource.password=你的密码
mvn spring-boot:run
```

### 前端启动

```bash
cd 101-frontend
npm install
# 修改后端接口地址（通常在 .env.development 或 src/utils/request.js 中）
npm run dev
```
