## 项目简介

零信任设备准入与访问控制管理系统面向企业终端安全和访问控制场景，围绕设备资产、员工账号、身份源、风险评估、访问策略、准入申请、访问会话、设备证书和审计事件构建闭环，确保每一次访问都经过身份、设备和风险校验。

## 角色权限

系统支持 **4** 种角色，各角色权限如下：

### - 管理员

- 维护账号
- 设备资产
- 员工账号和操作日志

### - 安全管理员

- 维护风险模型
- 风险评估
- 访问策略和审计事件

### - 网络管理员

- 维护网络分区
- 设备证书
- 访问会话和准入申请

### - 审计员

- 查看策略命中
- 阻断记录和审计事件

## 功能模块

| 模块 | 说明 |
|------|------|
| **账号权限** | 管理员/安全管理员/网络管理员/审计员四角色登录、JWT鉴权和Redis登录态 |
| **设备资产** | 管理终端设备、设备编号、系统版本、归属人和准入状态 |
| **员工账号** | 管理员工身份、账号名、部门、MFA状态和账号状态 |
| **身份源** | 管理企业AD、统一身份平台、同步方式和连接状态 |
| **风险模型** | 配置设备健康、身份可信、位置和行为风险模型并支持发布归档 |
| **风险评估** | 记录访问前风险评分、风险等级、处置建议并支持复核放行阻断 |
| **访问策略** | 管理资源访问策略、访问条件、负责人并支持启用停用 |
| **策略规则** | 管理策略下的条件类型、动作类型、优先级和状态 |
| **准入申请** | 管理设备访问资源的申请编号、原因和审批状态 |
| **访问会话** | 记录零信任访问会话、来源IP、登录时间和会话状态 |
| **网络分区** | 管理网络分区、CIDR网段、安全域和运行状态 |
| **设备证书** | 管理设备证书编号、签发方、有效期并支持吊销续期 |
| **审计事件** | 记录异常访问、策略命中、设备阻断等事件并支持确认关闭 |
| **操作日志** | 记录关键操作对象、动作、详情和IP地址 |

## 特色亮点

- 选题贴合零信任架构、终端安全、网络准入和企业安全运营方向
- 形成“设备资产-员工身份-风险模型-访问策略-准入申请-访问会话-审计事件”完整闭环
- 使用 MyBatis 注解 SQL + PageHelper，与 111 的 MyBatis-Plus 路线继续交替
- 设备资产、员工账号、身份源、风险模型、评估、策略、申请、会话、证书和审计事件都有状态流转动作
- 首页看板展示设备资产、风险评估、访问会话、审计事件、准入趋势和风险等级分布

## 演示流程

1. 管理员维护账号权限、身份源和员工账号
2. 网络管理员登记设备资产、网络分区和设备证书
3. 安全管理员配置风险模型、访问策略和策略规则
4. 员工设备发起准入申请并进入风险评估
5. 系统根据风险评分执行放行、复核或阻断
6. 系统沉淀访问会话和审计事件
7. 审计员查看准入趋势、风险分布和操作日志

## 快速启动

### 环境要求

| 工具 | 版本 |
|------|------|
| JDK | 8+ / 11+ |
| Maven | 3.8+ |
| MySQL | 8.0+ |
| Node.js | 18+ |

### 后端启动

```bash
cd 112-backend
# 1. 创建数据库并执行 init.sql
# 2. 修改 application.yml 数据库配置
# spring.datasource.url=jdbc:mysql://localhost:3306/Shanghai
# spring.datasource.username=root
# spring.datasource.password=你的密码
mvn spring-boot:run
```

### 前端启动

```bash
cd 112-frontend
npm install
# 修改后端接口地址（通常在 .env.development 或 src/utils/request.js 中）
npm run dev
```
