## 项目简介

企业知识库智能问答与文档权限管理系统面向企业内部知识沉淀、文档检索和问答追溯场景，提供知识空间、文档入库、文本分段、权限组授权、问答会话、命中来源、反馈复核和访问审计能力。系统帮助企业把制度、产品、运营和项目文档转化为可查询、可授权、可追溯的知识资产。

## 角色权限

系统支持 **3** 种角色，各角色权限如下：

### - 管理员

- 用户
- 空间
- 权限组
- 授权
- 日志和全部业务管理

### - 编辑员

- 文档入库
- 分段维护
- 发布归档和知识索引管理

### - 员工

- 发起问答
- 查看结果
- 提交反馈和查看统计

## 功能模块

| 模块 | 说明 |
|------|------|
| **账号权限** | 管理员/编辑员/员工三角色登录、JWT鉴权和访问控制 |
| **知识空间** | 空间名称、负责人、可见级别、说明和启停状态维护 |
| **文档分类** | 按知识空间维护分类编码、排序、说明和启用状态 |
| **知识文档** | 文档标题、摘要、来源、标签、正文、发布和归档管理 |
| **文档分段** | 分段标题、内容、关键词、排序和模拟向量索引状态 |
| **权限授权** | 权限组、组成员、空间/文档授权和访问级别管理 |
| **问答会话** | 创建知识库问答会话，按空间发起问题并关闭会话 |
| **问答记录** | 生成模拟答案、保存命中分段、置信度和解决状态 |
| **反馈审计** | 问答反馈、处理回复、访问日志和统计看板 |

## 特色亮点

- 选题贴合 RAG 知识库和企业文档权限管理热点，适合 AI 应用类毕设答辩
- 形成“知识空间-文档分类-文档入库-分段索引-权限授权-问答追踪-反馈处理”完整闭环
- 使用 MyBatis 注解 SQL + PageHelper 分页，与前一个项目的 MyBatis-Plus 技术路线区分明显
- 问答记录保存命中分段编号和置信度，能展示答案来源追溯能力
- 权限组、成员和授权表覆盖空间级与文档级访问控制，业务深度较强

## 演示流程

1. 管理员维护用户、知识空间、权限组、组成员和文档授权
2. 编辑员创建文档分类，录入知识文档并发布
3. 编辑员拆分文档内容为多个分段，点击生成索引
4. 员工创建问答会话，选择已授权知识空间并提交问题
5. 系统基于已索引分段生成参考答案，保存来源分段编号和置信度
6. 员工标记问答是否解决，并提交质量反馈
7. 编辑员处理反馈，管理员查看访问日志和首页统计看板

## 快速启动

### 环境要求

| 工具 | 版本 |
|------|------|
| JDK | 8+ / 11+ |
| Maven | 3.8+ |
| MySQL | 8.0+ |
| Node.js | 18+ |

### 后端启动

```bash
cd 098-backend
# 1. 创建数据库并执行 init.sql
# 2. 修改 application.yml 数据库配置
# spring.datasource.url=jdbc:mysql://localhost:3306/Shanghai
# spring.datasource.username=root
# spring.datasource.password=你的密码
mvn spring-boot:run
```

### 前端启动

```bash
cd 098-frontend
npm install
# 修改后端接口地址（通常在 .env.development 或 src/utils/request.js 中）
npm run dev
```
