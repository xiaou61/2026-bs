## 功能模块

| 模块 | 说明 |
|------|------|
| **账号权限** | 管理员/发布经理/运维工程师/审计员四角色登录、JWT鉴权和Redis登录态 |
| **发布环境** | 维护生产、预发、测试环境、集群、命名空间和负责人 |
| **应用服务** | 登记服务编码、代码仓库、技术语言、负责人和部署类型 |
| **流水线** | 维护默认分支、构建命令、镜像仓库和流水线状态 |
| **发布计划** | 管理计划编号、发布窗口、风险等级和发布说明 |
| **发布单** | 管理发布单号、版本号、申请人、发布范围并支持提交、通过、驳回、排期 |
| **审批流程** | 配置发布审批节点、审批角色、排序和启用状态 |
| **审批记录** | 记录发布单审批节点、审批人、结果和意见 |
| **版本制品** | 维护制品名称、镜像标签、校验值和构建时间 |
| **部署任务** | 管理部署任务、执行人、开始结束时间和日志摘要 |
| **回滚预案** | 维护回滚版本、策略名称、触发条件和负责人 |
| **回滚记录** | 记录回滚编号、原因、操作人、开始结束时间和状态 |
| **变更检查** | 管理发布前必检项、检查结果和证据地址 |
| **操作日志** | 记录关键操作对象、动作、详情和IP地址 |

## 特色亮点

- 选题贴合DevOps、云原生发布治理和运维流程数字化方向，实战感强
- 形成“发布计划-发布单-审批流-制品确认-部署执行-回滚预案-变更检查”完整闭环
- 使用 MyBatis 注解 SQL + PageHelper，与 105 的 MyBatis-Plus 路线继续交替
- 发布单、审批记录、部署任务、回滚预案、回滚记录均有清晰状态流转动作
- 首页看板展示服务、发布单、部署任务、回滚记录、制品数量和发布趋势

## 演示流程

1. 管理员登录维护账号权限、发布环境和操作日志
2. 发布经理登记应用服务、流水线和版本制品
3. 发布经理创建发布计划和发布单，提交审批并安排发布窗口
4. 审计员维护审批流程并处理审批记录
5. 运维工程师启动部署任务，记录完成或失败状态
6. 运维工程师维护回滚预案，必要时创建回滚记录并启动回滚
7. 首页看板查看发布趋势、状态分布和核心指标

## 快速启动

### 环境要求

| 工具 | 版本 |
|------|------|
| JDK | 8+ / 11+ |
| Maven | 3.8+ |
| MySQL | 8.0+ |
| Node.js | 18+ |

### 后端启动

```bash
cd 106-backend
# 1. 创建数据库并执行 init.sql
# 2. 修改 application.yml 数据库配置
# spring.datasource.url=jdbc:mysql://localhost:3306/Shanghai
# spring.datasource.username=root
# spring.datasource.password=你的密码
mvn spring-boot:run
```

### 前端启动

```bash
cd 106-frontend
npm install
# 修改后端接口地址（通常在 .env.development 或 src/utils/request.js 中）
npm run dev
```
