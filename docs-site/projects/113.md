区块链农产品质量溯源与监管平台面向农产品从生产、质检、上链、仓储、物流、销售到监管抽查的全过程管理，围绕产品批次、种植记录、农资投入、质检报告、区块存证、流通节点、召回事件和监管检查形成可追溯闭环。

## 功能模块

| 模块 | 说明 |
|------|------|
| **账号权限** | 管理员/监管员/农户/质检员四角色登录、JWT鉴权和Redis登录态 |
| **种植基地** | 管理基地备案、区域、地块面积、负责人和认证状态 |
| **农户档案** | 管理农户实名、合作社、联系方式和信用等级 |
| **产品分类** | 管理农产品分类、执行标准、检测要求和风险等级 |
| **产品批次** | 管理批次编号、产品、基地、农户、采收日期、溯源码和上链状态 |
| **种植记录** | 记录播种、施肥、灌溉、采收等生产过程并支持确认上链 |
| **农资投入** | 管理种子、肥料、农药等投入品来源、用量和审核状态 |
| **质检报告** | 管理农残、重金属、抽检结论、报告地址并支持发布和不合格处理 |
| **区块存证** | 管理交易哈希、数据哈希、区块高度和上链确认状态 |
| **流通节点** | 管理仓储、加工、批发、零售等节点备案和启停状态 |
| **物流记录** | 记录批次流转、承运方、温控记录和签收状态 |
| **召回事件** | 管理问题批次召回原因、风险等级、处理人和处置进度 |
| **监管检查** | 管理监管抽查、检查结论、整改要求和闭环状态 |
| **操作日志** | 记录关键操作对象、动作、详情和IP地址 |

## 特色亮点

- 选题贴合区块链溯源、农产品质量安全、监管数字化和供应链可信存证方向
- 形成“基地-农户-产品批次-种植记录-质检报告-区块存证-物流节点-召回监管”完整闭环
- 使用 MyBatis-Plus BaseMapper + 通用 CRUD Service，与 112 的 MyBatis 路线继续交替
- 批次、种植、农资、质检、上链、物流、召回和监管检查都有状态流转动作
- 首页看板展示产品批次、质检报告、区块存证、流通节点、上链趋势和产品风险分布

## 演示流程

1. 管理员维护账号权限、产品分类和基础资料
2. 农户登记种植基地、农户档案和产品批次
3. 农户补充种植记录和农资投入并提交审核
4. 质检员发布质检报告并生成区块存证
5. 流通企业维护仓储、批发、物流和签收记录
6. 监管员发起监管检查或召回事件
7. 首页看板查看上链趋势、风险分布和核心统计指标

| MySQL | 8.0+ |

### 后端启动

```bash
cd 113-backend
# 1. 创建数据库并执行 init.sql
# 2. 修改 application.yml 数据库配置
# spring.datasource.url=jdbc:mysql://localhost:3306/Shanghai
# spring.datasource.username=root
# spring.datasource.password=你的密码
mvn spring-boot:run
```

### 前端启动

```bash
cd 113-frontend
npm install
# 修改后端接口地址（通常在 .env.development 或 src/utils/request.js 中）
npm run dev
```
