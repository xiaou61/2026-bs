## 功能模块

| 模块 | 说明 |
|------|------|
| **账号权限** | 系统管理员/会务管理员/审稿专家/议程秘书四角色登录、JWT鉴权和Redis登录态 |
| **会议信息** | 会议编号、会议名称、会议主题、主办单位、会场数量和状态维护 |
| **征稿通知** | 通知编号、通知标题、发布人、截稿日期、投稿指引和状态维护 |
| **作者档案** | 作者编号、作者姓名、所属单位、联系邮箱、联系电话和状态维护 |
| **论文投稿** | 投稿编号、论文题目、作者姓名、提交版本、附件数量和状态维护 |
| **审稿专家** | 专家编号、专家姓名、研究方向、所在单位、邀请时间和状态维护 |
| **审稿分配** | 分配编号、投稿编号、审稿专家、分配意见、分配时间和状态维护 |
| **盲审记录** | 评审编号、稿件编号、论文题目、评审得分、提交日期和状态维护 |
| **录用通知** | 通知编号、稿件编号、论文题目、录用结果、通知人和状态维护 |
| **参会报名** | 报名编号、参会主题、参会单位、报名人数、联系人和状态维护 |
| **会场安排** | 会场编号、会场名称、对应议题、座位数量、签到数量和状态维护 |
| **会议日程** | 日程编号、日程主题、开始顺序、日程类型、主持人和状态维护 |
| **签到记录** | 签到编号、参会人、签到次数、签到方式、会务人员和状态维护 |
| **操作日志** | 关键模块操作人、动作、对象、详情和结果记录 |

## 特色亮点

- 选题贴合学术会议投稿、专家评审、录用通知、议程编排和签到服务方向
- 形成“会议信息-征稿通知-作者档案-论文投稿-审稿专家-审稿分配-盲审记录-录用通知-参会报名-会场安排-会议日程-签到记录”完整闭环
- 使用 MyBatis-Plus 分页查询与通用 CRUD 服务复用，与 134 的 MyBatis 注解 SQL 路线继续交替
- 论文投稿、审稿分配、盲审记录、录用通知、参会报名和签到记录都有状态流转动作
- 首页看板展示会议、投稿、评审、日程指标，以及投稿趋势和议程分布
- Spring Boot + Vue3 前后端分离
- MyBatis-Plus 分页查询与通用 CRUD 服务复用
- JWT 登录态拦截与 Redis Token 校验
- 投稿、评审、录用、议程、签到和通知状态机闭环
- ECharts 展示投稿趋势和会议议程分布
- Element Plus 通用数据页复用，统一搜索、表格、分页、弹窗和状态动作交互
- 学术会议投稿评审与日程管理系统类毕设项目
- 会务管理、学术服务、盲审流程、录用通知和日程编排课程设计
- 多角色后台管理系统综合实战
- 需要展示投稿评审、录用通知、会场日程和签到管理闭环的答辩项目

## 快速启动

### 环境要求

| 工具 | 版本 |
|------|------|
| JDK | 8+ / 11+ |
| Maven | 3.8+ |
| MySQL | 8.0+ |
| Node.js | 18+ |

### 后端启动

```bash
cd 135-backend
# 1. 创建数据库并执行 init.sql
# 2. 修改 application.yml 数据库配置
# spring.datasource.url=jdbc:mysql://localhost:3306/Shanghai
# spring.datasource.username=root
# spring.datasource.password=你的密码
mvn spring-boot:run
```

### 前端启动

```bash
cd 135-frontend
npm install
# 修改后端接口地址（通常在 .env.development 或 src/utils/request.js 中）
npm run dev
```
