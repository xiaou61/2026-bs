## 角色权限

系统支持 **3** 种角色，各角色权限如下：

### - 管理员

- admin
- 负责用户
- 区域
- 门店
- 公告和统计

### - 店长

- staff
- 负责门店
- 店宠
- 菜单
- 座位
- 预约和评价回复

### - 顾客

- customer
- 负责预约
- 下单
- 支付
- 评价和查看店宠门店

## 特色亮点

- 完整宠物咖啡业务：覆盖店宠、菜单、座位、预约、消费全链路
- 余额支付模拟：顾客充值、余额扣款、支付记录
- 互动预约体验：选座预约、状态流转、到店核销
- 角色权限隔离：管理员/店长/顾客权限边界清晰

## 演示流程

1. 管理员登录 → 管理区域、门店、公告
2. 店长登录 → 管理店宠、菜单、座位
3. 顾客浏览 → 查看店宠、菜单、公告
4. 顾客预约 → 选择日期和座位 → 创建预约
5. 顾客点单 → 选择菜品 → 余额支付 → 生成订单
6. 订单完成后 → 评价打分 → 店长回复

## 快速启动

### 环境要求

| 工具 | 版本 |
|------|------|
| JDK | 8+ / 11+ |
| Maven | 3.8+ |
| MySQL | 8.0+ |
| Node.js | 18+ |

### 后端启动

```bash
cd 094-backend
# 1. 创建数据库并执行 init.sql
# 2. 修改 application.yml 数据库配置
# spring.datasource.url=jdbc:mysql://localhost:3306/Shanghai
# spring.datasource.username=root
# spring.datasource.password=你的密码
mvn spring-boot:run
```

### 前端启动

```bash
cd 094-frontend
npm install
# 修改后端接口地址（通常在 .env.development 或 src/utils/request.js 中）
npm run dev
```
