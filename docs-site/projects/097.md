## 项目简介

大模型提示词资产管理与效果评测平台面向企业内部 AI 应用团队，提供 Prompt 资产沉淀、版本迭代、测试集维护、模型配置、自动评测、人工复核和效果对比能力。系统帮助团队把零散提示词变成可管理、可追踪、可复用、可评估的工程化资产。

## 特色亮点

- 选题贴合大模型应用工程化趋势，适合 AI 应用、Prompt 工程和企业知识生产类答辩
- 形成“资产创建-版本迭代-测试用例-模型评测-结果复核-反馈优化”完整闭环
- 管理员、工程师、评审员三角色权限差异清晰，演示路径层次完整
- 后端包含 12 张业务表和完整 REST API，覆盖资产、版本、测试、评测和审计全链路
- 前端提供通用数据管理组件、ECharts 看板、评测动作按钮和复核处理弹窗

## 演示流程

1. 管理员登录后维护用户、团队空间、Prompt分类、模型配置和评分规则
2. 工程师创建 Prompt 资产，补充场景、标签和团队归属信息
3. 工程师为资产新增多个版本，设置基准版本并发布可评测版本
4. 工程师维护测试用例，录入输入、期望输出、评分要点和难度
5. 工程师创建评测任务并启动评测，系统自动生成评测结果、平均分和通过率
6. 评审员进入评测结果页复核输出质量，填写复核意见并处理用户反馈
7. 管理员在首页看板查看资产数量、版本数量、任务数量、平均分和待处理反馈

## 快速启动

### 环境要求

| 工具 | 版本 |
|------|------|
| JDK | 8+ / 11+ |
| Maven | 3.8+ |
| MySQL | 8.0+ |
| Node.js | 18+ |

### 后端启动

```bash
cd 097-backend
# 1. 创建数据库并执行 init.sql
# 2. 修改 application.yml 数据库配置
# spring.datasource.url=jdbc:mysql://localhost:3306/Shanghai
# spring.datasource.username=root
# spring.datasource.password=你的密码
mvn spring-boot:run
```

### 前端启动

```bash
cd 097-frontend
npm install
# 修改后端接口地址（通常在 .env.development 或 src/utils/request.js 中）
npm run dev
```
