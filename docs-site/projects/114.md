## 特色亮点

- 选题贴合冷链物流、物联网温控、供应链追踪和异常预警方向
- 形成“仓点-承运企业-温控设备-运输订单-温控记录-异常告警-处置任务-责任追溯”完整闭环
- 使用 MyBatis 注解 SQL + PageHelper，与 113 的 MyBatis-Plus 路线继续交替
- 运输订单、温控记录、异常告警、处置任务、责任追溯和设备维护都有状态流转动作
- 首页看板展示运输订单、温控记录、异常告警、处置任务、采集趋势和异常类型分布

## 演示流程

1. 管理员维护账号权限、冷链仓点和承运企业
2. 承运员登记温控设备和冷链货品
3. 调度员创建运输订单并派发车辆
4. 系统持续沉淀温控记录和运输轨迹
5. 告警规则触发异常告警并生成处置任务
6. 监管员复核处置结果并形成责任追溯
7. 首页看板查看温控采集趋势、异常类型分布和核心指标

## 快速启动

### 环境要求

| 工具 | 版本 |
|------|------|
| JDK | 8+ / 11+ |
| Maven | 3.8+ |
| MySQL | 8.0+ |
| Node.js | 18+ |

### 后端启动

```bash
cd 114-backend
# 1. 创建数据库并执行 init.sql
# 2. 修改 application.yml 数据库配置
# spring.datasource.url=jdbc:mysql://localhost:3306/Shanghai
# spring.datasource.username=root
# spring.datasource.password=你的密码
mvn spring-boot:run
```

### 前端启动

```bash
cd 114-frontend
npm install
# 修改后端接口地址（通常在 .env.development 或 src/utils/request.js 中）
npm run dev
```
