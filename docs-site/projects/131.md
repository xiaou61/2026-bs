## 项目简介

药品不良反应上报与随访管理系统面向毕业设计答辩场景，提供多角色登录、业务台账维护、状态流转、统计看板和操作留痕能力。

## 功能模块

| 模块 | 说明 |
|------|------|
| **账号权限** | 管理员/上报员/复核员/随访医生四角色登录、JWT鉴权和Redis登录态 |
| **患者档案** | 管理患者编号、姓名、性别、年龄、联系电话和状态 |
| **药品目录** | 管理药品编号、名称、剂型、生产企业和批准文号 |
| **上报人档案** | 管理上报人编号、姓名、机构、联系电话和角色 |
| **不良反应上报** | 管理上报编号、患者、药品、上报时间、严重等级和处理状态 |
| **反应症状** | 管理症状编号、上报编号、症状名称、出现时间和症状等级 |
| **风险评估** | 管理评估编号、上报编号、风险等级、评分和评估人 |
| **随访计划** | 管理计划编号、上报编号、随访日期、随访方式和负责人 |
| **随访记录** | 管理记录编号、计划编号、随访时间、患者情况和记录人 |
| **病例复核** | 管理复核编号、上报编号、复核人、复核意见和复核时间 |
| **处置建议** | 管理建议编号、上报编号、建议类型、建议内容和医生 |
| **科室信息** | 管理科室编号、科室名称、医院名称、联系人和联系电话 |
| **统计报表** | 管理统计编号、科室、统计月份、上报数量和严重例数 |
| **操作日志** | 记录关键模块操作人、动作、对象、详情和IP地址 |

## 特色亮点

- 选题贴合医药监管、药品不良反应上报、风险分级、病例复核和随访闭环方向
- 形成“患者档案-药品目录-上报人档案-不良反应上报-反应症状-风险评估-随访计划-随访记录-病例复核-处置建议-统计报表”完整闭环
- 使用 MyBatis-Plus 分页查询和通用 CRUD 服务复用，与 130 的 MyBatis 注解 SQL 路线继续交替
- 不良反应上报、反应症状、风险评估、随访计划、随访记录和病例复核都有状态流转动作
- 首页看板展示患者、上报、评估、随访指标，以及不良反应上报趋势和严重等级分布
- Spring Boot + Vue3 前后端分离
- MyBatis-Plus 分页查询与通用 CRUD 服务复用
- JWT 登录态拦截与 Redis Token 校验
- 上报、评估、随访、复核和处置建议状态机闭环
- ECharts 展示不良反应上报趋势和严重等级分布
- Element Plus 通用数据页复用，统一搜索、表格、分页、弹窗和状态动作交互
- 药品不良反应上报与随访管理类毕设项目
- 医药监管、医院药学、病例复核、随访管理和风险评估课程设计
- 多角色后台管理系统综合实战
- 需要展示药品上报、风险评估、随访复核和统计报表闭环的答辩项目

## 快速启动

### 环境要求

| 工具 | 版本 |
|------|------|
| JDK | 8+ / 11+ |
| Maven | 3.8+ |
| MySQL | 8.0+ |
| Node.js | 18+ |

### 后端启动

```bash
cd 131-backend
# 1. 创建数据库并执行 init.sql
# 2. 修改 application.yml 数据库配置
# spring.datasource.url=jdbc:mysql://localhost:3306/Shanghai
# spring.datasource.username=root
# spring.datasource.password=你的密码
mvn spring-boot:run
```

### 前端启动

```bash
cd 131-frontend
npm install
# 修改后端接口地址（通常在 .env.development 或 src/utils/request.js 中）
npm run dev
```
