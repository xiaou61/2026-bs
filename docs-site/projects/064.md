## 项目简介

本项目是经方药食两用服务平台后端服务，配套 `064-frontend` 使用。系统覆盖登录注册、用户管理、食材分类、药食同源食材、经方信息、药膳方案、调理服务预约、体质记录、收藏、公告和数据看板等流程。

## 角色权限

系统支持 **2** 种角色，各角色权限如下：

### - ADMIN

- 平台管理员
- 维护基础数据
- 审核服务单
- 发布公告与查看看板

### - USER

- 普通用户
- 浏览经方和药膳方案
- 提交体质记录
- 发起服务预约与收藏

## 功能模块

| 模块 | 说明 |
|------|------|
| **用户与认证** | 登录注册、JWT鉴权、个人资料维护、密码修改 |
| **用户管理** | 管理员维护账号、角色、状态与账号禁用 |
| **食材分类管理** | 药食同源分类维护、排序与启停控制 |
| **药食食材管理** | 性味归经、功效、适宜/禁忌人群维护 |
| **经方库管理** | 经方出处、主治、组成、用法维护与查询 |
| **药膳方案管理** | 关联经方、适用体质、食用时段、步骤配置 |
| **调理服务单** | 用户预约提交、管理员状态处理与回复 |
| **体质记录管理** | 用户提交体质与症状，管理员回复建议 |
| **收藏中心** | 用户收藏经方/方案与收藏列表管理 |
| **公告中心** | 管理员发布公告，用户查看公开公告 |
| **数据看板** | 核心指标统计、7日服务单趋势、状态分布 |

## 演示流程

1. 管理员登录后维护分类、食材、经方、药膳方案与公告
2. 用户浏览经方库和药膳方案，收藏常用内容
3. 用户提交体质记录与症状描述，管理员给出调理建议
4. 用户发起调理服务预约，管理员处理状态并回复结果
5. 管理员在看板查看服务单趋势和待处理任务数量

## 快速启动

### 环境要求

| 工具 | 版本 |
|------|------|
| JDK | 8+ / 11+ |
| Maven | 3.8+ |
| MySQL | 8.0+ |
| Node.js | 18+ |

### 后端启动

```bash
cd 064-backend
# 1. 创建数据库并执行 init.sql
# 2. 修改 application.yml 数据库配置
# spring.datasource.url=jdbc:mysql://localhost:3306/Shanghai
# spring.datasource.username=root
# spring.datasource.password=你的密码
mvn spring-boot:run
```

### 前端启动

```bash
cd 064-frontend
npm install
# 修改后端接口地址（通常在 .env.development 或 src/utils/request.js 中）
npm run dev
```
