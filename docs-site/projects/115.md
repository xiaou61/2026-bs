## 项目简介

面向跨境电商运营、关务、财务和平台管理员，覆盖商家店铺、海外客户、商品SKU、跨境订单、清关申报、税费缴纳、汇率换算、结算对账和国际物流跟踪。

## 角色权限

系统支持 **4** 种角色，各角色权限如下：

### 管理员

- 维护账号权限
- 基础资料和全局配置。

### 关务专员

- 处理清关申报
- 单证审核
- 税费记录和放行状态。

### 财务结算员

- 维护汇率牌价
- 结算账单
- 支付流水和订单对账。

### 运营人员

- 维护店铺商品
- 客户档案
- 订单履约和物流轨迹。

## 功能模块

| 模块 | 说明 |
|------|------|
| **账号权限** | 管理员/关务专员/财务结算员/运营人员四角色登录、JWT鉴权和Redis登录态 |
| **商家店铺** | 管理跨境商家、平台店铺、主体公司、国家地区和合作状态 |
| **客户档案** | 管理海外客户身份、联系方式、地区和合规核验状态 |
| **商品SKU** | 管理跨境商品SKU、品类、HS编码、原产国和申报价值 |
| **跨境订单** | 管理店铺订单、客户、目的国、币种金额和履约状态 |
| **清关申报** | 管理清关申报单、口岸、申报类型、金额、风险等级和放行状态 |
| **清关单证** | 管理发票、箱单、报关委托、物流凭证和审核结果 |
| **税费记录** | 管理关税、增值税、消费税、缴纳金额、币种和支付状态 |
| **汇率牌价** | 管理币种对、汇率值、生效日期、来源机构和启用状态 |
| **结算账单** | 管理商家结算账单、原币金额、折算人民币、结算状态和对账管理 |
| **支付记录** | 管理结算支付流水、支付渠道、金额、币种、时间和支付状态 |
| **物流跟踪** | 管理国际物流单号、承运商、当前节点、位置、轨迹时间和签收状态 |
| **订单对账** | 管理账单对账任务、订单数量、差异金额、核对结果和异常闭环 |
| **操作日志** | 记录关键模块操作人、动作、对象、详情和IP地址 |

## 特色亮点

- 选题贴合跨境电商、关务合规、汇率结算、国际物流和财务对账方向
- 形成“店铺商品-跨境订单-清关申报-单证税费-汇率结算-支付对账-物流跟踪”完整业务闭环
- 使用 MyBatis-Plus 统一 CRUD 和分页查询，与 114 的 MyBatis 路线继续交替
- 跨境订单、清关申报、清关单证、税费记录、汇率牌价、结算账单、支付记录、物流跟踪和订单对账都有状态流转动作
- 首页看板展示订单、清关、税费、汇率、结算、物流、对账指标，以及汇率趋势和业务状态分布

## 演示流程

1. 管理员维护账号权限、商家店铺和客户档案
2. 运营人员维护商品SKU并创建跨境订单
3. 关务专员提交清关申报、审核清关单证并确认税费
4. 财务结算员维护汇率牌价并核算结算账单
5. 系统记录支付流水并执行订单对账
6. 运营人员维护国际物流轨迹并完成签收
7. 首页看板查看汇率趋势、业务状态分布和核心指标

## 快速启动

### 环境要求

| 工具 | 版本 |
|------|------|
| JDK | 8+ / 11+ |
| Maven | 3.8+ |
| MySQL | 8.0+ |
| Node.js | 18+ |

### 后端启动

```bash
cd 115-backend
# 1. 创建数据库并执行 init.sql
# 2. 修改 application.yml 数据库配置
# spring.datasource.url=jdbc:mysql://localhost:3306/Shanghai
# spring.datasource.username=root
# spring.datasource.password=你的密码
mvn spring-boot:run
```

### 前端启动

```bash
cd 115-frontend
npm install
# 修改后端接口地址（通常在 .env.development 或 src/utils/request.js 中）
npm run dev
```
