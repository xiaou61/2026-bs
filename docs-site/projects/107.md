## 项目简介

本系统面向云服务器运维监控场景，围绕云区域、主机资产、指标定义、指标采样、告警规则、告警事件、通知记录、处置工单、维护窗口和容量规划构建监控闭环。

| 模块 | 说明 |
|------|------|
| **账号权限** | 管理员/运维工程师/SRE工程师/运维主管四角色登录、JWT鉴权和Redis登录态 |
| **云区域** | 维护云厂商、地域、可用区、网络类型和状态 |

| **资源分组** | 管理业务线、环境、分组负责人和资源说明 |
| **指标定义** | 配置CPU、内存、磁盘、网络等监控指标、单位、采样周期和阈值 |
| **指标采样** | 记录主机指标值、采样时间、趋势状态和异常说明 |
| **告警规则** | 维护告警级别、比较符、阈值、持续时间和启用状态 |
| **告警事件** | 记录告警编号、触发值、确认人、恢复时间和事件状态 |
| **通知记录** | 管理短信、邮件、企业微信等通知渠道、接收人和发送结果 |
| **处置工单** | 记录告警处理、责任人、优先级、解决方案和关闭状态 |
| **维护窗口** | 管理计划维护时间、影响范围、审批人和执行状态 |
| **容量规划** | 维护资源增长预测、扩容建议、预算金额和执行结果 |
| **看板组件** | 配置首页组件类型、刷新周期、排序和展示状态 |
| **操作日志** | 记录关键操作对象、动作、详情和IP地址 |

## 特色亮点

- 选题贴合云运维、SRE监控和资源治理方向，适合展示平台化运维能力
- 形成“主机资产-指标采样-规则触发-告警通知-工单处置-容量规划”完整闭环
- 使用 MyBatis-Plus BaseMapper + 通用 CRUD Service，与 106 的 MyBatis 路线继续交替
- 告警事件支持确认、恢复、关闭，处置工单支持分配、解决、关闭，维护窗口支持审批和执行
- 首页看板展示服务器数量、告警趋势、工单状态、容量预测和资源分布

## 演示流程

1. 管理员维护账号、云区域、资源分组和操作日志
2. 运维工程师登记主机资产并配置指标定义
3. 系统沉淀指标采样数据并根据告警规则触发告警事件
4. SRE工程师维护告警规则和通知记录
5. 运维工程师确认、恢复或关闭告警事件
6. 运维工程师创建处置工单并执行分配、解决、关闭
7. 运维主管查看容量规划、维护窗口和首页监控趋势

## 快速启动

### 环境要求

| 工具 | 版本 |
|------|------|
| JDK | 8+ / 11+ |
| Maven | 3.8+ |
| MySQL | 8.0+ |
| Node.js | 18+ |

### 后端启动

```bash
cd 107-backend
# 1. 创建数据库并执行 init.sql
# 2. 修改 application.yml 数据库配置
# spring.datasource.url=jdbc:mysql://localhost:3306/Shanghai
# spring.datasource.username=root
# spring.datasource.password=你的密码
mvn spring-boot:run
```

### 前端启动

```bash
cd 107-frontend
npm install
# 修改后端接口地址（通常在 .env.development 或 src/utils/request.js 中）
npm run dev
```
