## 项目简介

本系统面向水务管线巡查、阀门台账维护、故障上报、工单派发、检修回执和风险预警场景，提供水务站点、管线区段、巡线路线、阀门台账、巡线任务、巡线记录、故障上报、故障派工、检修回执、备件台账和风险预警的一体化后台管理能力。

## 角色权限

系统支持 **6** 种角色，各角色权限如下：

### - 系统管理员

- 账号权限
- 基础配置和全部数据维护

### - 调度管理员

- 维护水务站点
- 管线区段
- 巡线任务
- 故障派工和统计规则

### - 巡线班组

- 执行巡线路线
- 填写巡线记录
- 上报故障和风险隐患

### - 检修人员

- 接收故障派工
- 登记检修过程
- 提交检修回执和备件消耗

### - 备件管理员

- 维护备件台账
- 跟踪备件状态并协同检修领用

### - 监管负责人

- 查看巡线
- 故障
- 检修和风险预警统计并复核闭环结果

- 水务巡线工单与阀门检修协同管理系统类毕设项目
- 市政运维、水务管线巡检、阀门检修、工单派发和风险预警课程设计
- 多角色后台管理系统综合实战

## 快速启动

### 环境要求

| 工具 | 版本 |
|------|------|
| JDK | 8+ / 11+ |
| Maven | 3.8+ |
| MySQL | 8.0+ |
| Node.js | 18+ |

### 后端启动

```bash
cd 176-backend
# 1. 创建数据库并执行 init.sql
# 2. 修改 application.yml 数据库配置
# spring.datasource.url=jdbc:mysql://localhost:3306/Shanghai
# spring.datasource.username=root
# spring.datasource.password=你的密码
mvn spring-boot:run
```

### 前端启动

```bash
cd 176-frontend
npm install
# 修改后端接口地址（通常在 .env.development 或 src/utils/request.js 中）
npm run dev
```
