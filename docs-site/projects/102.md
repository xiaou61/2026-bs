## 项目简介

本项目面向律师事务所、法律咨询机构和高校法律援助中心，提供委托人档案、律师档案、案件台账、咨询记录、案件进度、证据材料、文书模板、智能文书、版本留痕、预约咨询、费用记录和审计统计能力。系统采用 Spring Boot + MyBatis + MySQL + Redis + Vue3 技术栈，围绕“咨询建档-案件分配-进度跟踪-证据归档-文书生成-预约沟通-费用结算-审计留痕”形成闭环。

## 角色权限

系统支持 **4** 种角色，各角色权限如下：

### - 管理员

- 账号
- 律师档案
- 模板和审计管理

### - 律师

- 案件处理
- 咨询答复
- 文书复核和预约管理

### - 助理

- 案件资料
- 进度节点
- 证据材料和费用协同维护

### - 委托人

- 查看案件进度
- 咨询预约和材料状态

## 功能模块

| 模块 | 说明 |
|------|------|
| **账号权限** | 管理员/律师/助理/委托人四角色登录、JWT鉴权和Redis登录态 |
| **委托人档案** | 维护委托人姓名、证件号、联系方式、地址和偏好案件类型 |
| **律师档案** | 维护执业证号、擅长领域、经验年限和服务状态 |
| **案件台账** | 管理案件编号、委托人、承办律师、类型、优先级、状态和下一步动作 |
| **进度节点** | 维护案件阶段、计划时间、完成时间和节点备注 |
| **咨询记录** | 记录咨询方式、问题、律师答复、风险等级和跟进动作 |
| **文书模板** | 管理仲裁申请、律师函、合同审查等模板和变量说明 |
| **法律文书** | 按案件和模板生成文书，支持复核通过或驳回 |
| **文书版本** | 记录文书版本号、正文快照、变更说明和操作人 |
| **咨询预约** | 安排咨询时间、方式、地点并支持确认、取消、完成 |
| **证据材料** | 管理证据名称、类型、文件地址和核验意见 |
| **费用记录** | 维护咨询费、代理费、文书费并支持支付和退款 |
| **统计审计** | 首页指标、图表看板和关键操作日志 |

## 特色亮点

- 选题贴合法律服务数字化、智能文书和案件过程管理方向，适合法律科技类答辩
- 形成“咨询建档-案件分配-进度跟踪-证据归档-文书生成-预约沟通-费用结算”完整闭环
- 使用 MyBatis 注解 SQL + PageHelper，与 101 的 MyBatis-Plus 技术路线区分明显
- 文书模块支持模板生成、复核意见和版本留痕，便于展示智能文书管理价值
- 多角色演示路径清晰，律师、助理、委托人和管理员均有明确业务入口

## 演示流程

1. 管理员登录维护账号、律师档案、文书模板和操作日志
2. 助理维护委托人档案并创建案件台账，分配承办律师
3. 律师记录咨询问题和答复，设置风险等级和跟进动作
4. 助理维护案件进度节点，点击完成或重开节点
5. 律师按模板创建法律文书并点击生成，随后复核通过或驳回
6. 委托人补充证据材料，律师或助理核验通过或驳回
7. 助理安排咨询预约并维护费用支付状态，首页查看综合统计

## 快速启动

### 环境要求

| 工具 | 版本 |
|------|------|
| JDK | 8+ / 11+ |
| Maven | 3.8+ |
| MySQL | 8.0+ |
| Node.js | 18+ |

### 后端启动

```bash
cd 102-backend
# 1. 创建数据库并执行 init.sql
# 2. 修改 application.yml 数据库配置
# spring.datasource.url=jdbc:mysql://localhost:3306/Shanghai
# spring.datasource.username=root
# spring.datasource.password=你的密码
mvn spring-boot:run
```

### 前端启动

```bash
cd 102-frontend
npm install
# 修改后端接口地址（通常在 .env.development 或 src/utils/request.js 中）
npm run dev
```
