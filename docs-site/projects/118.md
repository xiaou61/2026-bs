## 角色权限

系统支持 **4** 种角色，各角色权限如下：

### 管理员

- 维护账号权限
- 仓库基础资料和系统配置。

### 调度员

- 派发AGV任务
- 选择路径并处理异常告警。

### 仓管员

- 维护库存物料
- 入库订单
- 出库订单和库位状态。

### 维保员

- 处理AGV车辆
- 充电站和传感器维保任务。

## 特色亮点

- 选题贴合智能仓储、AGV调度、库位优化、自动化物流和设备运维方向
- 形成“仓库区域-库位档案-库存物料-入库出库-AGV任务-路径规划-库位推荐-异常告警-设备维保”完整闭环
- 使用 MyBatis 注解 SQL + PageHelper 分页查询，与 117 的 MyBatis-Plus 路线继续交替
- 库位、AGV车辆、入库单、出库单、AGV任务、路径规划、库位推荐、设备维保和异常告警都有状态流转动作
- 首页看板展示库位、AGV车辆、任务、异常告警指标，以及近7日任务趋势和库区任务分布

## 演示流程

1. 管理员维护账号权限、仓库区域、库位档案和AGV车辆
2. 仓管员维护库存物料并创建入库或出库订单
3. 调度员基于订单派发AGV任务并确认路径规划
4. 系统依据物料周转、温区、承重和热度生成库位推荐
5. AGV执行搬运任务，异常告警进入调度处理
6. 维保员处理设备维保并恢复车辆或站点状态
7. 首页看板查看AGV任务趋势、库区分布和核心运营指标

## 快速启动

### 环境要求

| 工具 | 版本 |
|------|------|
| JDK | 8+ / 11+ |
| Maven | 3.8+ |
| MySQL | 8.0+ |
| Node.js | 18+ |

### 后端启动

```bash
cd 118-backend
# 1. 创建数据库并执行 init.sql
# 2. 修改 application.yml 数据库配置
# spring.datasource.url=jdbc:mysql://localhost:3306/Shanghai
# spring.datasource.username=root
# spring.datasource.password=你的密码
mvn spring-boot:run
```

### 前端启动

```bash
cd 118-frontend
npm install
# 修改后端接口地址（通常在 .env.development 或 src/utils/request.js 中）
npm run dev
```
