## 项目简介

车辆保险理赔材料审核与进度跟踪系统面向毕业设计答辩场景，提供多角色登录、业务台账维护、审批状态流转、统计看板和操作留痕能力。

## 特色亮点

- 选题贴合车辆保险理赔、事故报案、材料审核、定损赔付和进度跟踪方向
- 形成“保险保单-车辆档案-客户档案-理赔申请-事故报案-材料清单-材料审核-定损记录-赔付记录-进度跟踪-回访记录-消息通知”完整闭环
- 使用 MyBatis 注解 SQL + PageHelper 分页查询，与 141 的 MyBatis-Plus 路线继续交替
- 理赔申请、材料审核、定损记录、赔付记录、进度跟踪和消息通知都有状态流转动作
- 首页看板展示保单、报案、审核、进度指标，以及理赔进度趋势和状态分布
- Spring Boot + Vue3 前后端分离
- MyBatis 注解 SQL + PageHelper 分页查询
- JWT 登录态拦截与 Redis Token 校验
- 报案受理、材料审核、定损赔付和进度跟踪状态机联动
- ECharts 展示理赔进度趋势和理赔状态分布
- Element Plus 通用数据页复用，统一搜索、表格、分页、弹窗和状态动作交互
- 车辆保险理赔材料审核与进度跟踪系统类毕设项目
- 保险业务、理赔流程、审核协同、赔付管理和客户回访课程设计
- 多角色后台管理系统综合实战
- 需要展示理赔处理闭环、材料审核轨迹、定损赔付和进度通知能力的答辩项目

## 快速启动

### 环境要求

| 工具 | 版本 |
|------|------|
| JDK | 8+ / 11+ |
| Maven | 3.8+ |
| MySQL | 8.0+ |
| Node.js | 18+ |

### 后端启动

```bash
cd 142-backend
# 1. 创建数据库并执行 init.sql
# 2. 修改 application.yml 数据库配置
# spring.datasource.url=jdbc:mysql://localhost:3306/Shanghai
# spring.datasource.username=root
# spring.datasource.password=你的密码
mvn spring-boot:run
```

### 前端启动

```bash
cd 142-frontend
npm install
# 修改后端接口地址（通常在 .env.development 或 src/utils/request.js 中）
npm run dev
```
