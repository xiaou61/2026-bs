| 属性 | 内容 |

## 项目简介

本系统面向社区垃圾分类督导与居民积分激励场景，提供社区区域、楼栋档案、居民档案、分类规则、分类打卡、督导记录、整改任务、积分记录、兑换商品、积分兑换和楼栋排名的一体化后台管理能力。

### - 系统管理员

- 分类打卡

- 整改任务
- 积分记录和楼栋排名维护

### - 居民用户

- 分类打卡
- 积分记录

### - 志愿协管员

- 楼栋协管
- 督导记录
- 整改任务和居民协同维护

### - 积分兑换员

- 兑换商品
- 积分兑换
- 兑换核销和库存状态维护

## 功能模块

| 模块 | 说明 |
|------|------|
| **账号权限** | 系统管理员、社区管理员、分类督导员、居民用户、志愿协管员、积分兑换员六类账号登录、JWT鉴权和Redis登录态 |
| **社区区域** | 区域编号、区域名称、覆盖楼栋、区域负责人、启用时间和区域状态维护 |
| **楼栋档案** | 楼栋编号、楼栋名称、所属网格、楼栋管家、建档时间和楼栋状态维护 |
| **居民档案** | 居民编号、居民姓名、所属楼栋、联系方式、登记时间和居民状态维护 |
| **分类规则** | 规则编号、垃圾类别、投放要求、责任人员、生效时间和规则状态维护 |
| **分类打卡** | 打卡编号、打卡居民、垃圾类别、督导人员、打卡时间和打卡状态维护 |
| **督导记录** | 督导编号、督导楼栋、督导类型、督导人员、督导时间和督导状态维护 |
| **整改任务** | 整改编号、整改对象、问题类型、责任人员、整改时间和整改状态维护 |
| **积分记录** | 积分编号、积分居民、积分来源、记录人员、积分时间和积分状态维护 |
| **兑换商品** | 商品编号、商品名称、商品类型、兑换负责人、上架时间和商品状态维护 |
| **积分兑换** | 兑换编号、兑换居民、兑换类型、兑换人员、兑换时间和兑换状态维护 |
| **楼栋排名** | 排名编号、排名楼栋、排名周期、统计人员、统计时间和排名状态维护 |
| **操作日志** | 关键模块操作人、动作、对象、详情和结果记录 |

- 形成“社区区域-楼栋档案-居民档案-分类规则-分类打卡-督导记录-整改任务-积分记录-兑换商品-积分兑换-楼栋排名”完整业务闭环
- 使用 MyBatis 注解 SQL + PageHelper 分页查询

- 首页看板展示核心指标、近7日趋势和状态分布

- 社区垃圾分类督导与积分兑换平台类毕设项目
- 社区治理、业务审批、台账管理和统计分析课程设计
- 多角色后台管理系统综合实战

## 快速启动

### 环境要求

| 工具 | 版本 |
|------|------|
| JDK | 8+ / 11+ |
| Maven | 3.8+ |
| MySQL | 8.0+ |
| Node.js | 18+ |

### 后端启动

```bash
cd 167-backend
# 1. 创建数据库并执行 init.sql
# 2. 修改 application.yml 数据库配置
# spring.datasource.url=jdbc:mysql://localhost:3306/Shanghai
# spring.datasource.username=root
# spring.datasource.password=你的密码
mvn spring-boot:run
```

### 前端启动

```bash
cd 167-frontend
npm install
# 修改后端接口地址（通常在 .env.development 或 src/utils/request.js 中）
npm run dev
```
