## 特色亮点

- 选题贴合智慧城市、城市内涝、水位雨量监测、预警规则和应急调度方向
- 形成“水位点位-雨量站点-排涝泵站-监测数据-预警规则-内涝预警-应急预案-调度任务-救援队伍-物资储备-避险点位”完整闭环
- 使用 MyBatis-Plus 分页查询和通用 CRUD 服务复用，与 122 的 MyBatis 注解 SQL 路线继续交替
- 水位数据、雨量数据、内涝预警、应急预案、调度任务、物资储备和避险点位都有状态流转动作
- 首页看板展示监测点、预警、任务、物资指标，以及内涝预警趋势和预警等级分布

## 演示流程

1. 管理员维护账号权限、水位点位、雨量站点和排涝泵站
2. 监测员登记水位数据和雨量数据
3. 应急主管配置预警规则和应急预案
4. 系统触发内涝预警并派发调度任务
5. 调度员协调救援队伍、物资储备和避险点位
6. 任务完成后关闭预警并沉淀操作日志
7. 首页看板查看预警趋势、等级分布和应急资源指标

## 快速启动

### 环境要求

| 工具 | 版本 |
|------|------|
| JDK | 8+ / 11+ |
| Maven | 3.8+ |
| MySQL | 8.0+ |
| Node.js | 18+ |

### 后端启动

```bash
cd 123-backend
# 1. 创建数据库并执行 init.sql
# 2. 修改 application.yml 数据库配置
# spring.datasource.url=jdbc:mysql://localhost:3306/Shanghai
# spring.datasource.username=root
# spring.datasource.password=你的密码
mvn spring-boot:run
```

### 前端启动

```bash
cd 123-frontend
npm install
# 修改后端接口地址（通常在 .env.development 或 src/utils/request.js 中）
npm run dev
```
