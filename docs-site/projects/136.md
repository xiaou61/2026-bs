## 功能模块

| 模块 | 说明 |
|------|------|
| **账号权限** | 系统管理员/导师/学生/教务管理员四角色登录、JWT鉴权和Redis登录态 |
| **课题发布** | 课题编号、课题名称、导师姓名、所属方向、计划容量和状态维护 |
| **导师档案** | 导师编号、导师姓名、所属学院、研究方向、可带人数和状态维护 |
| **学生档案** | 学号、学生姓名、专业名称、班级名称、志愿数量和状态维护 |
| **课题申请** | 申请编号、课题编号、申请学生、志愿顺序、申请时间和状态维护 |
| **双选审核** | 审核编号、申请编号、审核节点、审核意见、审核人和状态维护 |
| **双选确认** | 确认编号、申请编号、确认结果、导师姓名、确认意见和状态维护 |
| **任务书下达** | 任务书编号、课题编号、阶段任务、下达时间、指导教师和状态维护 |
| **开题材料** | 材料编号、课题编号、材料名称、材料类型、提交学生和状态维护 |
| **开题答辩** | 答辩编号、课题编号、答辩主题、答辩小组、答辩时间和状态维护 |
| **中期检查** | 检查编号、课题编号、检查节点、检查教师、检查时间和状态维护 |
| **指导记录** | 记录编号、课题编号、指导主题、指导时间、完成情况和状态维护 |
| **节点通知** | 通知编号、课题编号、节点类型、提醒内容、接收人和状态维护 |
| **操作日志** | 关键模块操作人、动作、对象、详情和结果记录 |

## 特色亮点

- 选题贴合毕业设计过程管理、课题双选、任务书下达、开题答辩和阶段提醒方向
- 形成“课题发布-导师档案-学生档案-课题申请-双选审核-双选确认-任务书下达-开题材料-开题答辩-中期检查-指导记录-节点通知”完整闭环
- 使用 MyBatis 注解 SQL + PageHelper 分页查询，与 135 的 MyBatis-Plus 路线继续交替
- 课题申请、双选审核、双选确认、任务书下达、开题材料、开题答辩和节点通知都有状态流转动作
- 首页看板展示课题、申请、材料、通知指标，以及双选申请趋势和过程节点分布
- Spring Boot + Vue3 前后端分离
- MyBatis 注解 SQL + PageHelper 分页查询
- JWT 登录态拦截与 Redis Token 校验
- 课题申请、审核确认、任务书、开题答辩和中期检查状态机闭环
- ECharts 展示双选申请趋势和过程节点分布
- Element Plus 通用数据页复用，统一搜索、表格、分页、弹窗和状态动作交互
- 导师课题双选与开题过程管理系统类毕设项目
- 毕业设计管理、课题双选、开题答辩、指导记录和节点提醒课程设计
- 多角色后台管理系统综合实战
- 需要展示课题双选、任务书下达、开题过程和阶段督导闭环的答辩项目

## 快速启动

### 环境要求

| 工具 | 版本 |
|------|------|
| JDK | 8+ / 11+ |
| Maven | 3.8+ |
| MySQL | 8.0+ |
| Node.js | 18+ |

### 后端启动

```bash
cd 136-backend
# 1. 创建数据库并执行 init.sql
# 2. 修改 application.yml 数据库配置
# spring.datasource.url=jdbc:mysql://localhost:3306/Shanghai
# spring.datasource.username=root
# spring.datasource.password=你的密码
mvn spring-boot:run
```

### 前端启动

```bash
cd 136-frontend
npm install
# 修改后端接口地址（通常在 .env.development 或 src/utils/request.js 中）
npm run dev
```
