## 功能模块

| 模块 | 说明 |
|------|------|
| **账号权限** | 管理员/产品经理/测试工程师/开发人员四角色登录、JWT鉴权和Redis登录态 |
| **接口项目** | 管理项目编码、基础地址、鉴权方式、版本号和状态 |
| **接口分组** | 按项目维护接口目录、父级分组、排序和说明 |
| **接口定义** | 维护接口名称、请求方式、路径、内容类型、维护人和发布状态 |
| **请求参数** | 管理参数位置、数据类型、是否必填、示例值和说明 |
| **响应字段** | 管理响应字段层级、类型、示例和说明 |
| **Mock规则** | 配置匹配表达式、响应码、响应内容、延迟和启用状态 |
| **测试用例** | 维护正向、异常、边界等用例类型、优先级和预期结果 |
| **用例步骤** | 配置请求数据、断言表达式、变量提取和步骤顺序 |
| **环境配置** | 维护开发、测试、预发环境地址、请求头和变量 |
| **执行记录** | 记录执行批次、环境、执行人、开始结束时间和摘要 |
| **结果明细** | 记录用例执行状态、实际状态码、耗时和断言信息 |
| **文档快照** | 生成接口文档版本，支持发布和归档 |
| **操作日志** | 记录关键操作对象、动作、详情和IP地址 |

## 特色亮点

- 选题贴合接口协作、自动化测试和研发效能方向，适合软件测试类答辩
- 形成“接口定义-参数字段-Mock规则-测试用例-执行结果-文档快照”完整闭环
- 使用 MyBatis-Plus BaseMapper + 通用 CRUD Service，与 104 的 MyBatis 路线继续交替
- 接口发布下线、用例启停、执行启动完成失败、结果确认驳回、文档发布归档都有状态流转
- 首页看板展示接口、用例、执行、文档、通过率趋势和请求方法分布

## 演示流程

1. 管理员登录维护账号权限和操作日志
2. 产品经理创建接口项目、接口分组和接口定义
3. 开发人员补充请求参数、响应字段和Mock规则
4. 测试工程师编写测试用例和用例步骤，配置测试环境
5. 测试工程师启动执行记录，系统沉淀结果明细和断言信息
6. 产品经理发布接口文档快照，开发人员下线过期接口
7. 首页看板查看接口数量、用例数量、执行趋势和方法分布

## 快速启动

### 环境要求

| 工具 | 版本 |
|------|------|
| JDK | 8+ / 11+ |
| Maven | 3.8+ |
| MySQL | 8.0+ |
| Node.js | 18+ |

### 后端启动

```bash
cd 105-backend
# 1. 创建数据库并执行 init.sql
# 2. 修改 application.yml 数据库配置
# spring.datasource.url=jdbc:mysql://localhost:3306/Shanghai
# spring.datasource.username=root
# spring.datasource.password=你的密码
mvn spring-boot:run
```

### 前端启动

```bash
cd 105-frontend
npm install
# 修改后端接口地址（通常在 .env.development 或 src/utils/request.js 中）
npm run dev
```
