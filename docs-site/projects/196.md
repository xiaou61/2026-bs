## 项目简介

本系统面向连锁药店、药师审核人员、门店店员、随访专员和慢病顾客，提供药店门店、顾客档案、药品目录、处方登记、处方审核、风险复核、购药记录、用药指导、慢病方案、续方提醒、随访记录和操作日志的一体化管理能力。

## 角色权限

系统支持 **6** 种角色，各角色权限如下：

### - 系统管理员

- 账号权限
- 基础配置和全部数据维护

### - 药店管理员

- 维护门店
- 药品目录
- 慢病服务规则和人员协作权限

### - 药师审核

- 审核处方
- 处理风险复核并记录用药指导

### - 门店店员

- 登记处方
- 维护购药记录并协助顾客完成取药流程

### - 随访专员

- 维护慢病方案
- 续方提醒和随访记录

### - 慢病顾客

- 查看个人处方
- 购药记录
- 用药指导和续方提醒

## 特色亮点

- 选题贴合医药零售和慢病用药服务方向，覆盖处方登记、药师审核、风险复核、购药记录、用药指导、慢病方案、续方提醒和随访记录
- 形成“顾客建档-处方登记-处方审核-风险复核-购药记录-用药指导-慢病方案-续方提醒-随访闭环”的业务链路
- 按 ADMIN / PHARMACY / PHARMACIST / CLERK / FOLLOWUP / CUSTOMER 六类角色收口菜单、路由和操作权限
- 使用 MyBatis 注解 SQL + PageHelper 分页查询
- 各业务模块支持列表查询、新增编辑、删除、状态处理和状态完成
- 首页看板展示药店门店、顾客档案、药品目录、处方登记核心指标、近7日审核提醒趋势和处方慢病服务状态分布
- Spring Boot + Vue3 前后端分离
- MyBatis 注解 SQL + PageHelper 分页查询
- JWT 登录态拦截与 Redis Token 校验
- 通用业务台账、状态流转和统计看板联动
- ECharts 展示业务趋势和状态分布
- Element Plus 通用数据页复用，统一搜索、表格、分页、弹窗和状态动作交互
- 药店处方审核与慢病续方提醒管理系统类毕设项目
- 医药零售、业务审批、台账管理和统计分析课程设计
- 多角色后台管理系统综合实战
- 需要展示完整业务闭环和可视化看板能力的答辩项目

## 快速启动

### 环境要求

| 工具 | 版本 |
|------|------|
| JDK | 8+ / 11+ |
| Maven | 3.8+ |
| MySQL | 8.0+ |
| Node.js | 18+ |

### 后端启动

```bash
cd 196-backend
# 1. 创建数据库并执行 init.sql
# 2. 修改 application.yml 数据库配置
# spring.datasource.url=jdbc:mysql://localhost:3306/Shanghai
# spring.datasource.username=root
# spring.datasource.password=你的密码
mvn spring-boot:run
```

### 前端启动

```bash
cd 196-frontend
npm install
# 修改后端接口地址（通常在 .env.development 或 src/utils/request.js 中）
npm run dev
```
