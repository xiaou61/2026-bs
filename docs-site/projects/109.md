## 项目简介

数据脱敏与敏感信息识别平台面向企业数据安全治理场景，围绕数据源接入、数据集目录、敏感规则、识别任务、识别结果、脱敏策略、脱敏任务、访问申请、导出审批和风险告警构建完整闭环，帮助企业降低敏感数据泄露风险。

## 角色权限

系统支持 **4** 种角色，各角色权限如下：

### - 管理员

- 维护账号
- 数据源
- 平台配置和操作日志

### - 安全工程师

- 维护敏感规则
- 识别任务
- 脱敏策略和风险告警

### - 数据负责人

- 审批访问申请
- 导出审批和字段血缘

### - 审计员

- 查看识别结果
- 脱敏记录
- 风险告警和审计日志

## 功能模块

| 模块 | 说明 |
|------|------|
| **账号权限** | 管理员/安全工程师/数据负责人/审计员四角色登录、JWT鉴权和Redis登录态 |
| **数据源配置** | 管理数据库、文件和接口数据源、地址、库名、负责人和状态 |
| **数据集目录** | 维护数据集编码、数据源、表名、业务域和安全等级 |
| **敏感规则** | 配置身份证、手机号、邮箱、银行卡、地址等识别规则和风险等级 |
| **识别任务** | 管理敏感信息扫描任务、规则范围、扫描模式并支持启动、完成、失败 |
| **识别结果** | 记录字段命中类型、样例值、置信度、风险等级并支持确认、忽略 |
| **脱敏策略** | 管理脱敏方法、表达式、审批角色并支持启用、停用 |
| **脱敏任务** | 管理数据集脱敏任务、执行模式并支持提交、启动、完成、失败 |
| **脱敏记录** | 记录字段脱敏前后样例、处理数量和执行结果 |
| **字段血缘** | 管理字段来源路径、目标路径、负责人和安全等级 |
| **访问申请** | 管理敏感数据访问申请、用途、有效期并支持提交、通过、驳回、撤销 |
| **导出审批** | 管理导出类型、申请人、文件等级并支持通过、驳回、完成 |
| **风险告警** | 记录原始导出、高危字段、过期访问等告警并支持确认、解决、关闭 |
| **操作日志** | 记录关键操作对象、动作、详情和IP地址 |

## 特色亮点

- 选题贴合数据安全、隐私保护和企业数据治理方向，适合安全类毕设答辩
- 形成“数据源-数据集-敏感规则-识别结果-脱敏策略-访问审批-风险告警”完整闭环
- 使用 MyBatis-Plus BaseMapper + 通用 CRUD Service，与 108 的 MyBatis 路线继续交替
- 识别任务、脱敏任务、访问申请、导出审批、风险告警都有状态流转动作
- 首页看板展示数据源、数据集、识别结果、风险告警、识别趋势和敏感类型分布

## 演示流程

1. 管理员维护账号权限、数据源配置和数据集目录
2. 安全工程师配置敏感规则并启动识别任务
3. 审计员复核识别结果，确认或忽略字段命中
4. 安全工程师配置脱敏策略并发起脱敏任务
5. 数据负责人审批访问申请和导出审批
6. 安全工程师处理风险告警，完成确认、解决和关闭
7. 首页看板查看识别趋势、敏感类型分布和核心统计指标

## 快速启动

### 环境要求

| 工具 | 版本 |
|------|------|
| JDK | 8+ / 11+ |
| Maven | 3.8+ |
| MySQL | 8.0+ |
| Node.js | 18+ |

### 后端启动

```bash
cd 109-backend
# 1. 创建数据库并执行 init.sql
# 2. 修改 application.yml 数据库配置
# spring.datasource.url=jdbc:mysql://localhost:3306/Shanghai
# spring.datasource.username=root
# spring.datasource.password=你的密码
mvn spring-boot:run
```

### 前端启动

```bash
cd 109-frontend
npm install
# 修改后端接口地址（通常在 .env.development 或 src/utils/request.js 中）
npm run dev
```
