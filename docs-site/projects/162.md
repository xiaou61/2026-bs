## 项目简介

本系统面向生鲜零售门店的临期商品治理场景，提供门店档案、供应商档案、生鲜品类、商品批次、保质期规则、临期预警、促销策略、折扣执行、报损记录、库存周转和门店分析的一体化后台管理能力。

## 角色权限

系统支持 **6** 种角色，各角色权限如下：

### - 系统管理员

- 账号权限
- 基础配置和全部数据维护

### - 门店店长

- 门店档案
- 商品批次
- 临期预警和门店分析维护

### - 理货员

- 商品批次
- 折扣执行
- 报损记录和现场处置维护

### - 库存主管

- 生鲜品类
- 保质期规则
- 临期预警
- 报损记录和库存周转维护

### - 促销专员

- 促销策略
- 折扣执行和门店促销复盘维护

### - 供应商协同员

- 供应商档案和供货批次状态协同维护

- 生鲜门店临期商品预警与促销处置系统类毕设项目

- 多角色后台管理系统综合实战
- 需要展示完整业务闭环和可视化看板能力的答辩项目

## 快速启动

### 环境要求

| 工具 | 版本 |
|------|------|
| JDK | 8+ / 11+ |
| Maven | 3.8+ |
| MySQL | 8.0+ |
| Node.js | 18+ |

### 后端启动

```bash
cd 162-backend
# 1. 创建数据库并执行 init.sql
# 2. 修改 application.yml 数据库配置
# spring.datasource.url=jdbc:mysql://localhost:3306/Shanghai
# spring.datasource.username=root
# spring.datasource.password=你的密码
mvn spring-boot:run
```

### 前端启动

```bash
cd 162-frontend
npm install
# 修改后端接口地址（通常在 .env.development 或 src/utils/request.js 中）
npm run dev
```
