## 项目简介

本系统面向高校体育赛事组织与裁判评分场景，提供体育赛事、参赛队伍、运动员档案、赛事报名、报名分组、赛程编排、场地资源、裁判指派、裁判评分、成绩公示和申诉复核的一体化后台管理能力。

## 角色权限

系统支持 **6** 种角色，各角色权限如下：

### - 系统管理员

- 账号权限
- 基础配置和全部数据维护

### - 赛事组织员

- 体育赛事
- 报名分组
- 赛程编排
- 场地资源和成绩公示维护

### - 队伍教练

- 参赛队伍
- 运动员档案
- 赛事报名
- 赛程确认和成绩申诉维护

- 赛事报名
- 赛程查看
- 成绩公示和申诉进度查看

### - 裁判员

- 裁判指派
- 赛事执裁
- 裁判评分和赛果确认维护

### - 仲裁委员

- 评分复核
- 成绩公示
- 申诉复核和争议处置维护

## 快速启动

### 环境要求

| 工具 | 版本 |
|------|------|
| JDK | 8+ / 11+ |
| Maven | 3.8+ |
| MySQL | 8.0+ |
| Node.js | 18+ |

### 后端启动

```bash
cd 164-backend
# 1. 创建数据库并执行 init.sql
# 2. 修改 application.yml 数据库配置
# spring.datasource.url=jdbc:mysql://localhost:3306/Shanghai
# spring.datasource.username=root
# spring.datasource.password=你的密码
mvn spring-boot:run
```

### 前端启动

```bash
cd 164-frontend
npm install
# 修改后端接口地址（通常在 .env.development 或 src/utils/request.js 中）
npm run dev
```
