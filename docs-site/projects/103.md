## 项目简介

本项目面向企业客服中心和在线服务团队，提供客户档案、服务渠道、知识库、客服工单、会话消息、派单记录、质检规则、质检任务、质检结果、知识推荐、坐席绩效和操作审计能力。系统采用 Spring Boot + MyBatis-Plus + MySQL + Redis + Vue3 技术栈，围绕“客户建档-工单受理-会话记录-质检任务-结果复核-知识推荐-绩效统计”形成闭环。

- 账号

### - 客服主管

### - 客服坐席

- 工单处理
- 会话消息和知识采纳

### - 质检员

- 规则
- 任务
- 结果复核和改进建议

## 功能模块

| 模块 | 说明 |
|------|------|
| **账号权限** | 管理员/主管/坐席/质检员四角色登录、JWT鉴权和Redis登录态 |
| **客户档案** | 维护客户等级、来源、手机号、标签和服务备注 |
| **服务渠道** | 管理热线、在线客服、工单系统、企业微信等渠道状态 |
| **知识分类** | 维护知识库分类、排序、启用状态和说明 |
| **知识文章** | 管理文章标题、关键词、适用场景、状态并支持发布下线 |
| **客服工单** | 管理工单编号、客户、渠道、优先级、状态、处理人和处理结果 |
| **会话消息** | 记录客户与坐席对话内容、情绪、敏感词和回复标记 |
| **派单记录** | 跟踪工单分配人、接收人、分配原因、接单状态和响应时间 |
| **质检规则** | 配置礼貌用语、响应时效、敏感词、解决闭环等质检规则 |
| **质检任务** | 创建质检批次，支持启动、完成和驳回 |
| **质检结果** | 记录评分、扣分项、改进建议和复核意见 |
| **知识推荐** | 根据工单和关键词推荐知识文章，支持采纳或拒绝 |
| **坐席绩效** | 维护处理量、平均响应时间、解决率、满意度和质检分 |
| **统计审计** | 首页指标、趋势图表和关键操作日志 |

## 特色亮点

- 选题贴合智能客服、服务质检和企业知识库推荐方向，答辩辨识度高
- 形成“客户建档-工单流转-会话留痕-质检评分-知识推荐-绩效复盘”完整闭环
- 使用 MyBatis-Plus 分页查询和通用 CRUD，与 102 的 MyBatis 注解路线交替
- 质检任务支持运行、完成、驳回和结果复核，便于展示质量管理流程
- 知识推荐支持采纳与拒绝状态沉淀，可体现智能辅助客服处理价值

## 演示流程

1. 管理员登录维护账号权限和操作日志
2. 主管维护服务渠道、知识分类、知识文章、质检规则和绩效数据
3. 坐席创建或接收客服工单，记录会话消息并更新处理结果
4. 质检员创建质检任务并启动评分，生成质检结果和整改建议
5. 质检员复核质检结果，主管查看坐席绩效变化
6. 坐席查看知识推荐并执行采纳或拒绝
7. 首页看板查看工单、质检、推荐、绩效和趋势统计

## 快速启动

### 环境要求

| 工具 | 版本 |
|------|------|
| JDK | 8+ / 11+ |
| Maven | 3.8+ |
| MySQL | 8.0+ |
| Node.js | 18+ |

### 后端启动

```bash
cd 103-backend
# 1. 创建数据库并执行 init.sql
# 2. 修改 application.yml 数据库配置
# spring.datasource.url=jdbc:mysql://localhost:3306/Shanghai
# spring.datasource.username=root
# spring.datasource.password=你的密码
mvn spring-boot:run
```

### 前端启动

```bash
cd 103-frontend
npm install
# 修改后端接口地址（通常在 .env.development 或 src/utils/request.js 中）
npm run dev
```
