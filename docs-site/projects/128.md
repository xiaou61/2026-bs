## 项目简介

ESG 数据填报与可视化报告系统面向毕业设计答辩场景，提供多角色登录、业务台账维护、状态流转、统计看板和操作留痕能力。

## 特色亮点

- 选题贴合 ESG 披露、指标填报、附件审查、评分模型、改进任务和报告导出方向
- 形成“指标库-披露模板-报告周期-企业填报-指标数据-佐证材料-审核任务-评分模型-ESG评分-改进任务-反馈-报告导出”完整闭环
- 使用 MyBatis 注解 SQL + PageHelper 分页查询，与 127 的 MyBatis-Plus 路线继续交替
- 报告周期、企业填报、指标数据、佐证材料、审核任务、改进任务和报告导出都有状态流转动作
- 首页看板展示指标、填报、审核、导出指标，以及 ESG 填报趋势和维度分布

## 演示流程

1. 管理员维护账号权限、指标库、披露模板和评分模型
2. ESG经理创建报告周期并指定填报责任人
3. 填报员提交企业填报、指标数据和佐证材料
4. 审核员处理审核任务并给出审核意见
5. 系统汇总 ESG 评分并生成改进任务
6. 管理层处理利益相关方反馈并导出 ESG 报告
7. 首页看板查看填报趋势、维度分布和审核导出指标

## 快速启动

### 环境要求

| 工具 | 版本 |
|------|------|
| JDK | 8+ / 11+ |
| Maven | 3.8+ |
| MySQL | 8.0+ |
| Node.js | 18+ |

### 后端启动

```bash
cd 128-backend
# 1. 创建数据库并执行 init.sql
# 2. 修改 application.yml 数据库配置
# spring.datasource.url=jdbc:mysql://localhost:3306/Shanghai
# spring.datasource.username=root
# spring.datasource.password=你的密码
mvn spring-boot:run
```

### 前端启动

```bash
cd 128-frontend
npm install
# 修改后端接口地址（通常在 .env.development 或 src/utils/request.js 中）
npm run dev
```
