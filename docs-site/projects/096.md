## 功能模块

| 模块 | 说明 |
|------|------|
| **账号权限** | 管理员/医生/患者三角色登录、患者注册与登录态校验 |
| **科室医生** | 科室档案、医生账号、职称擅长与医生大厅展示 |
| **排班号源** | 医生按日期时段发布排班，维护号源、诊室和挂号费 |
| **在线挂号** | 患者选择医生排班、就诊卡与备注信息提交挂号 |
| **订单支付** | 挂号订单生成、模拟在线支付、订单状态与支付记录维护 |
| **就诊服务** | 就诊卡管理、我的挂号、医生预约患者查看与接诊完成 |
| **评价公告** | 患者完成就诊后评价医生，后台统一维护公告与首页轮播 |
| **统计看板** | 科室预约排行、近七日挂号趋势、热门医生与多角色首页指标 |

## 特色亮点

- 选题贴近真实医院门诊场景，适合做医疗服务类毕设答辩
- 形成“科室医生展示-排班号源-在线挂号-订单支付-接诊完成-评价反馈”完整闭环
- 管理员、医生、患者三角色操作差异明显，答辩演示层次清晰
- MyBatis 原生 XML + PageHelper 便于展示 SQL、分页与联表查询实现
- 首页同时覆盖轮播、公告、热门医生和可视化看板，展示效果直观

## 演示流程

1. 管理员维护科室、医生账号、轮播公告和医生排班
2. 患者登录后进入医生大厅，按科室和擅长方向筛选医生
3. 患者选择排班和就诊卡创建挂号，并在订单页完成支付
4. 医生登录后查看预约患者列表，完成接诊状态更新
5. 患者在我的挂号中查看状态、取消未完成预约并对已完成记录进行评价
6. 管理端通过统计页查看预约趋势、科室排行和热门医生数据

### 后端启动

```bash
cd 096-backend
# 1. 创建数据库并执行 init.sql
# 2. 修改 application.yml 数据库配置
# spring.datasource.url=jdbc:mysql://localhost:3306/Shanghai
# spring.datasource.username=root
# spring.datasource.password=你的密码
mvn spring-boot:run
```

### 前端启动

```bash
cd 096-frontend
npm install
# 修改后端接口地址（通常在 .env.development 或 src/utils/request.js 中）
npm run dev
```
