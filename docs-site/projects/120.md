## 特色亮点

- 选题贴合数字孪生、智慧园区、设备巡检、缺陷整改和能耗监测方向
- 形成“园区楼宇-孪生设备-巡检路线-巡检点位-巡检任务-巡检记录-缺陷报告-维修工单-传感数据-孪生模型-能耗监测-保养计划”完整闭环
- 使用 MyBatis 注解 SQL + PageHelper 分页查询，与 119 的 MyBatis-Plus 路线继续交替
- 巡检任务、巡检记录、缺陷报告、维修工单、传感数据、能耗监测和保养计划都有状态流转动作
- 首页看板展示楼宇、设备、巡检、缺陷、工单指标，以及巡检趋势和缺陷等级分布

## 演示流程

1. 管理员维护账号权限、园区楼宇和孪生设备
2. 园区经理配置巡检路线、巡检点位和孪生模型
3. 巡检员接收巡检任务并提交巡检记录
4. 发现异常后提交缺陷报告并进入维修工单
5. 工程师处理工单并同步保养计划
6. 系统沉淀传感数据和能耗监测记录
7. 首页看板查看巡检趋势、缺陷分布和设备运行指标

## 快速启动

### 环境要求

| 工具 | 版本 |
|------|------|
| JDK | 8+ / 11+ |
| Maven | 3.8+ |
| MySQL | 8.0+ |
| Node.js | 18+ |

### 后端启动

```bash
cd 120-backend
# 1. 创建数据库并执行 init.sql
# 2. 修改 application.yml 数据库配置
# spring.datasource.url=jdbc:mysql://localhost:3306/Shanghai
# spring.datasource.username=root
# spring.datasource.password=你的密码
mvn spring-boot:run
```

### 前端启动

```bash
cd 120-frontend
npm install
# 修改后端接口地址（通常在 .env.development 或 src/utils/request.js 中）
npm run dev
```
