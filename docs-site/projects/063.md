## 项目简介

本项目是进销存管理系统后端服务，配套 `063-frontend` 使用。系统覆盖登录注册、用户管理、供应商、客户、商品分类、商品档案、采购单审核入库、销售单审核出库、库存流水、公告和经营看板等流程。

## 角色权限

系统支持 **2** 种角色，各角色权限如下：

### - ADMIN

- 系统管理员
- 拥有全部权限
- 负责账号配置
- 基础数据维护与运营管理

### - STAFF

- 业务人员
- 负责采购
- 销售
- 库存与日常业务操作

## 功能模块

| 模块 | 说明 |
|------|------|
| **用户与认证** | 登录注册、JWT鉴权、个人中心、密码修改 |
| **用户管理** | 管理员维护账号、角色、启停状态 |
| **供应商管理** | 供应商信息维护与状态控制 |
| **客户管理** | 客户信息维护与状态控制 |

| **商品管理** | 商品档案、成本价/销售价、库存预警配置 |

| **销售管理** | 销售单新增编辑、审核出库、库存校验 |

| **公告中心** | 管理员发布公告，业务人员查看公告 |
| **经营看板** | 核心指标统计、7日采购/销售趋势分析 |

## 演示流程

1. 管理员登录后维护用户、供应商、客户、分类与商品档案
2. 业务员创建采购单并审核，系统自动执行入库并生成库存流水
3. 业务员创建销售单并审核，系统先校验库存再执行出库
4. 管理员在看板查看今日采购额、销售额与库存预警数量
5. 管理员发布公告，业务员在公告中心和首页查看最新通知

## 快速启动

### 环境要求

| 工具 | 版本 |
|------|------|
| JDK | 8+ / 11+ |
| Maven | 3.8+ |
| MySQL | 8.0+ |
| Node.js | 18+ |

### 后端启动

```bash
cd 063-backend
# 1. 创建数据库并执行 init.sql
# 2. 修改 application.yml 数据库配置
# spring.datasource.url=jdbc:mysql://localhost:3306/Shanghai
# spring.datasource.username=root
# spring.datasource.password=你的密码
mvn spring-boot:run
```

### 前端启动

```bash
cd 063-frontend
npm install
# 修改后端接口地址（通常在 .env.development 或 src/utils/request.js 中）
npm run dev
```
