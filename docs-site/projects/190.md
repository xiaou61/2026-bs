## 项目简介

本系统面向写字楼、园区物业、入驻单位、维修技师与巡检人员的楼宇运维场景，提供楼宇档案、设备档案、入驻档案、访修工单、维修派工、保养计划、保养任务、故障预警、巡检记录、备件库存、服务评价和操作日志的一体化管理能力。

## 角色权限

系统支持 **6** 种角色，各角色权限如下：

### - 系统管理员

- 账号权限
- 基础配置和全部数据维护

### - 物业管理员

- 维护楼宇档案
- 设备档案
- 入驻档案和服务规则

### - 访修调度员

- 处理访修工单
- 维修派工
- 服务评价和跨岗位协同

### - 维修技师

- 接收维修派工
- 执行保养任务并处置故障预警

### - 设备巡检员

- 维护巡检记录
- 上报故障预警并跟进设备状态

### - 入驻用户

- 提交访修工单
- 查看维修进度并完成服务评价

- 形成“访修工单-维修派工-维修处置-保养计划-保养任务-巡检预警-服务评价”完整运维闭环

- 各业务模块支持列表查询、新增编辑、删除、状态处理和状态完成
- 首页看板展示核心指标、近7日趋势和状态分布
- Spring Boot + Vue3 前后端分离
- MyBatis 注解 SQL + PageHelper 分页查询
- JWT 登录态拦截与 Redis Token 校验
- 通用业务台账、状态流转和统计看板联动

- Element Plus 通用数据页复用，统一搜索、表格、分页、弹窗和状态动作交互
- 智慧楼宇访修协同与设备保养提醒系统类毕设项目
- 楼宇运维、物业访修、设备保养提醒和统计分析课程设计
- 多角色后台管理系统综合实战
- 需要展示完整业务闭环和可视化看板能力的答辩项目

## 快速启动

### 环境要求

| 工具 | 版本 |
|------|------|
| JDK | 8+ / 11+ |
| Maven | 3.8+ |
| MySQL | 8.0+ |
| Node.js | 18+ |

### 后端启动

```bash
cd 190-backend
# 1. 创建数据库并执行 init.sql
# 2. 修改 application.yml 数据库配置
# spring.datasource.url=jdbc:mysql://localhost:3306/Shanghai
# spring.datasource.username=root
# spring.datasource.password=你的密码
mvn spring-boot:run
```

### 前端启动

```bash
cd 190-frontend
npm install
# 修改后端接口地址（通常在 .env.development 或 src/utils/request.js 中）
npm run dev
```
