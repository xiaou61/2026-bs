## 功能模块

| 模块 | 说明 |
|------|------|
| **运动打卡** | 跑步打卡（GPS轨迹、配速、距离、时长）、健身房打卡、球类运动打卡 |
| **健身计划** | 个人健身计划制定、教练指导计划、计划执行进度跟踪、计划模板库 |

| **健康档案** | BMI指数计算、体脂率记录、饮食记录、健康数据趋势分析 |
| **场馆预约** | 场馆信息展示、时段预约、预约记录管理、签到核销 |
| **积分系统** | 运动打卡获得积分、积分规则、积分历史、等级体系 |

## 特色亮点

- GPS运动轨迹记录：存储经纬度坐标点，轨迹回放展示
- 智能配速计算：距离/时间自动计算平均配速和速度
- 完善积分体系：跑步每公里10分、健身房每30分钟10分、球类每小时15分
- 多维度排行榜：步数、时长、积分三维度，日榜周榜月榜
- 健康数据分析：BMI、体脂率、饮食记录、ECharts趋势图
- 约球活动管理：发起活动、报名限制、水平要求、活动评价
- 场馆预约系统：时段管理、签到核销、超时自动取消
- 定时任务自动化：每日统计、周月榜生成、超时预约取消

## 演示流程

1. 学生注册登录 → 完善身高体重信息
2. 创建跑步打卡 → GPS记录轨迹 → 自动计算配速距离 → 获得积分
3. 浏览健身计划模板 → 选择增肌计划 → 开始执行
4. 发起约球活动 → 选择篮球、场馆、时间 → 其他用户报名
5. 查看排行榜 → 步数/时长/积分排名 → 激励运动
6. 记录健康档案 → 体重、饮食、睡眠 → 查看趋势图
7. 预约场馆 → 选择时间段 → 到场签到 → 运动完成
8. 教练登录 → 为学生制定专业健身计划
9. 管理员后台 → 场馆管理、数据统计

### 后端启动

```bash
cd 019-backend
# 1. 创建数据库并执行 init.sql
# 2. 修改 application.yml 数据库配置
# spring.datasource.url=jdbc:mysql://localhost:3306/Shanghai
# spring.datasource.username=root
# spring.datasource.password=你的密码
mvn spring-boot:run
```

### 前端启动

```bash
cd 019-frontend
npm install
# 修改后端接口地址（通常在 .env.development 或 src/utils/request.js 中）
npm run dev
```
