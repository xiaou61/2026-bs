## 功能模块

| 模块 | 说明 |
|------|------|
| **账号权限** | 管理员/俱乐部经理/球迷三角色登录与权限控制 |
| **联赛赛季** | 联赛信息、赛季周期、轮次与启停状态统一维护 |
| **俱乐部球队** | 俱乐部档案、球队资料、主场球场与赛季归属管理 |
| **教练球员** | 教练与球员档案、号码位置、国籍年龄、进球助攻数据维护 |
| **赛程赛果** | 比赛排期、主客队、裁判、比分录入与完赛状态跟踪 |
| **积分榜** | 依据比赛结果自动重算胜平负、净胜球、积分与排名 |
| **资讯关注** | 联赛资讯发布、球队关注、球迷个人中心维护 |
| **统计看板** | 比赛趋势、积分榜前列、射手榜和核心运营指标可视化 |

## 特色亮点

- 以“联赛运营”而不是单纯赛事展示为核心，适合做后台管理类毕设答辩
- 覆盖联赛、赛季、球队、球员、赛程、积分榜的完整足球业务链路
- 录入赛果后自动刷新积分榜，能直观展示业务联动和统计逻辑
- 管理员、俱乐部经理、球迷三角色界面差异明显，便于答辩演示
- ECharts 看板聚合赛程趋势、积分排行和射手榜，展示效果直观

## 演示流程

1. 管理员维护联赛、赛季、俱乐部和球场基础信息
2. 俱乐部经理维护球队、教练和球员档案，设置赛季目标与配色
3. 管理员创建赛程并安排主客队、开球时间和比赛场地
4. 比赛结束后录入赛果，系统自动重算积分榜排名
5. 球迷登录后浏览赛程、积分榜、球队资讯并关注喜爱的球队
6. 管理端通过统计看板查看联赛整体运营数据

### 后端启动

```bash
cd 095-backend
# 1. 创建数据库并执行 init.sql
# 2. 修改 application.yml 数据库配置
# spring.datasource.url=jdbc:mysql://localhost:3306/Shanghai
# spring.datasource.username=root
# spring.datasource.password=你的密码
mvn spring-boot:run
```

### 前端启动

```bash
cd 095-frontend
npm install
# 修改后端接口地址（通常在 .env.development 或 src/utils/request.js 中）
npm run dev
```
