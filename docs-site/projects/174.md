## 项目简介

本系统面向社区慢病患者建档、随访计划、服药提醒、健康指标采集、风险分层和依从性管理场景，提供社区站点、患者档案、医护团队、慢病档案、随访计划、随访记录、用药方案、服药打卡、健康指标、风险分层和提醒通知的一体化后台管理能力。

## 技术栈

### 后端

- PageHelper 1.4.7

### 前端

- Vue 3.4.0

- Vite 5
- Pinia 2.1.7
- Axios 1.6.2
- ECharts 5.4.3

## 角色权限

系统支持 **6** 种角色，各角色权限如下：

### - 系统管理员

- 账号权限
- 基础配置和全部数据维护

### - 社区卫生中心

- 维护社区站点
- 慢病档案
- 随访计划和统计规则

### - 家庭医生

- 维护患者档案
- 慢病档案
- 随访计划和风险分层

### - 随访护士

- 执行随访记录
- 健康指标采集
- 提醒通知和异常回访

### - 用药药师

- 维护用药方案
- 服药打卡
- 用药依从性和提醒通知

### - 居民患者

- 查看慢病档案
- 随访计划
- 服药提醒和健康指标

## 特色亮点

- 选题贴合慢病管理方向，覆盖社区建档、随访执行、用药依从、健康指标和风险分层
- 形成“社区站点-患者档案-医护团队-慢病档案-随访计划-随访记录-用药方案-服药打卡-健康指标-风险分层-提醒通知”完整业务闭环
- 使用 MyBatis 注解 SQL + PageHelper 分页查询
- 各业务模块支持列表查询、新增编辑、删除、状态处理和状态完成
- 首页看板展示核心指标、近7日趋势和状态分布
- Spring Boot + Vue3 前后端分离
- MyBatis 注解 SQL + PageHelper 分页查询
- JWT 登录态拦截与 Redis Token 校验
- 六类慢病服务角色的菜单、路由和后端接口权限收口
- ECharts 展示业务趋势和状态分布
- Element Plus 通用数据页复用，统一搜索、表格、分页、弹窗和状态动作交互
- 社区慢病档案随访与服药依从性管理系统类毕设项目
- 慢病建档、家庭医生签约、随访管理、服药依从性和风险分层课程设计
- 多角色后台管理系统综合实战
- 需要展示社区慢病服务闭环、依从性追踪和可视化看板能力的答辩项目

## 快速启动

### 环境要求

| 工具 | 版本 |
|------|------|
| JDK | 8+ / 11+ |
| Maven | 3.8+ |
| MySQL | 8.0+ |
| Node.js | 18+ |

### 后端启动

```bash
cd 174-backend
# 1. 创建数据库并执行 init.sql
# 2. 修改 application.yml 数据库配置
# spring.datasource.url=jdbc:mysql://localhost:3306/Shanghai
# spring.datasource.username=root
# spring.datasource.password=你的密码
mvn spring-boot:run
```

### 前端启动

```bash
cd 174-frontend
npm install
# 修改后端接口地址（通常在 .env.development 或 src/utils/request.js 中）
npm run dev
```
