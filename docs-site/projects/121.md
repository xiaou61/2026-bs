## 特色亮点

- 选题贴合无人机巡检、航线规划、缺陷识别、整改闭环和风险预警方向
- 形成“无人机设备-飞手档案-巡检区域-航线规划-巡检任务-飞行记录-缺陷报告-缺陷图片-整改工单-电池站点-维保记录-风险预警”完整闭环
- 使用 MyBatis-Plus 分页查询和通用 CRUD 服务复用，与 120 的 MyBatis 注解 SQL 路线继续交替
- 巡检任务、飞行记录、缺陷报告、缺陷图片、整改工单、维保记录和风险预警都有状态流转动作
- 首页看板展示无人机、任务、缺陷、整改指标，以及巡检任务趋势和缺陷等级分布

## 演示流程

1. 管理员维护账号权限、无人机设备、飞手档案和巡检区域
2. 运营经理配置航线规划并派发巡检任务
3. 飞手执行巡检任务并提交飞行记录和缺陷图片
4. 工程师确认缺陷报告并创建整改工单
5. 责任班组提交整改结果，工程师完成闭环确认
6. 系统结合电池站点、维保记录和缺陷增长生成风险预警
7. 首页看板查看巡检趋势、缺陷等级和整改闭环指标

## 快速启动

### 环境要求

| 工具 | 版本 |
|------|------|
| JDK | 8+ / 11+ |
| Maven | 3.8+ |
| MySQL | 8.0+ |
| Node.js | 18+ |

### 后端启动

```bash
cd 121-backend
# 1. 创建数据库并执行 init.sql
# 2. 修改 application.yml 数据库配置
# spring.datasource.url=jdbc:mysql://localhost:3306/Shanghai
# spring.datasource.username=root
# spring.datasource.password=你的密码
mvn spring-boot:run
```

### 前端启动

```bash
cd 121-frontend
npm install
# 修改后端接口地址（通常在 .env.development 或 src/utils/request.js 中）
npm run dev
```
