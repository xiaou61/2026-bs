## 演示流程

1. 管理员登录系统，查看平台看板与公告概览
2. 管理员维护维修分类，设置分类排序和启停状态
3. 服务方发布维修项目，填写服务名、门店、车型、价格、预约方式
4. 车主进入预约大厅，按关键词和分类筛选维修项目
5. 车主创建预约，填写预约日期/时间、车牌号、车型和故障描述
6. 车主支付预约单，服务方在工单列表确认预约
7. 到店维修完成后，车主确认完成并提交服务评价
8. 若服务争议产生，车主发起申诉，管理员介入回复处理
9. 用户可收藏常用维修项目，便于后续快速预约
10. 管理员持续查看预约趋势、热门项目和分类分布优化运营策略

## 快速启动

### 环境要求

| 工具 | 版本 |
|------|------|
| JDK | 8+ / 11+ |
| Maven | 3.8+ |
| MySQL | 8.0+ |
| Node.js | 18+ |

### 后端启动

```bash
cd 075-backend
# 1. 创建数据库并执行 init.sql
# 2. 修改 application.yml 数据库配置
# spring.datasource.url=jdbc:mysql://localhost:3306/Shanghai
# spring.datasource.username=root
# spring.datasource.password=你的密码
mvn spring-boot:run
```

### 前端启动

```bash
cd 075-frontend
npm install
# 修改后端接口地址（通常在 .env.development 或 src/utils/request.js 中）
npm run dev
```
