## 特色亮点

- 选题贴合智慧工地、安全巡检、隐患整改、班组责任和验收闭环方向
- 形成“工地项目-施工班组-安全员档案-巡检计划-巡检任务-隐患上报-整改工单-验收记录-安全培训-设备检查-防护用品-风险预警”完整闭环
- 使用 MyBatis 注解 SQL + PageHelper 分页查询，与 121 的 MyBatis-Plus 路线继续交替
- 巡检计划、巡检任务、隐患上报、整改工单、验收记录、设备检查和风险预警都有状态流转动作
- 首页看板展示项目、任务、隐患、整改指标，以及隐患整改趋势和隐患等级分布

## 演示流程

1. 管理员维护账号权限、工地项目、施工班组和安全员档案
2. 安全主管配置巡检计划并派发巡检任务
3. 巡检员执行检查并上报现场隐患
4. 班组长接收整改工单并提交整改结果
5. 巡检员完成验收记录，安全主管追踪风险预警
6. 系统同步安全培训、设备检查和防护用品库存
7. 首页看板查看隐患趋势、等级分布和整改闭环指标

## 快速启动

### 环境要求

| 工具 | 版本 |
|------|------|
| JDK | 8+ / 11+ |
| Maven | 3.8+ |
| MySQL | 8.0+ |
| Node.js | 18+ |

### 后端启动

```bash
cd 122-backend
# 1. 创建数据库并执行 init.sql
# 2. 修改 application.yml 数据库配置
# spring.datasource.url=jdbc:mysql://localhost:3306/Shanghai
# spring.datasource.username=root
# spring.datasource.password=你的密码
mvn spring-boot:run
```

### 前端启动

```bash
cd 122-frontend
npm install
# 修改后端接口地址（通常在 .env.development 或 src/utils/request.js 中）
npm run dev
```
