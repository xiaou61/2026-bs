## 特色亮点

- 选题贴合智慧交通、停车诱导、空位预测、预约停车和无人值守运营方向
- 形成“停车场-停车区域-泊位-传感器-车辆-预约-停车记录-支付-预测-诱导屏-导航路线-故障报修”完整闭环
- 使用 MyBatis-Plus 分页查询和通用 CRUD 服务复用，与 124 的 MyBatis 注解 SQL 路线继续交替
- 预约订单、停车记录、支付记录、空位预测、诱导屏、导航路线和故障报修都有状态流转动作
- 首页看板展示停车场、泊位、预约、故障指标，以及空位预测趋势和泊位状态分布

## 演示流程

1. 管理员维护账号权限、停车场档案、停车区域和泊位信息
2. 运营员绑定车位传感器并维护诱导屏展示内容
3. 车主车辆提交预约订单并获取推荐导航路线
4. 岗亭员登记入场、离场和停车记录
5. 系统生成支付记录并沉淀空位预测结果
6. 运维人员处理传感器、诱导屏或泊位故障报修
7. 首页看板查看预约趋势、泊位状态和预测运营指标

## 快速启动

### 环境要求

| 工具 | 版本 |
|------|------|
| JDK | 8+ / 11+ |
| Maven | 3.8+ |
| MySQL | 8.0+ |
| Node.js | 18+ |

### 后端启动

```bash
cd 125-backend
# 1. 创建数据库并执行 init.sql
# 2. 修改 application.yml 数据库配置
# spring.datasource.url=jdbc:mysql://localhost:3306/Shanghai
# spring.datasource.username=root
# spring.datasource.password=你的密码
mvn spring-boot:run
```

### 前端启动

```bash
cd 125-frontend
npm install
# 修改后端接口地址（通常在 .env.development 或 src/utils/request.js 中）
npm run dev
```
