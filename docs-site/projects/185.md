## 项目简介

本系统面向城市公厕点位巡检、保洁任务执行、耗材库存补给、投诉反馈和问题整改场景，提供公厕点位、保洁路线、保洁任务、保洁记录、巡检计划、巡检记录、耗材库存、补给申请、补给调度、投诉反馈和问题整改的一体化后台管理能力。

## 角色权限

系统支持 **6** 种角色，各角色权限如下：

### - 系统管理员

- 账号权限
- 基础配置和全部数据维护

### - 环卫管理员

- 维护公厕点位
- 保洁路线
- 巡检计划和调度规则

### - 保洁员

- 接收保洁任务
- 填写保洁记录并提交补给申请

### - 补给员

- 维护耗材库存
- 处理补给申请并执行补给调度

### - 巡检员

- 执行巡检记录
- 发现问题并跟踪整改闭环

### - 市民监督

- 提交投诉反馈并查看处理进度

## 快速启动

### 环境要求

| 工具 | 版本 |
|------|------|
| JDK | 8+ / 11+ |
| Maven | 3.8+ |
| MySQL | 8.0+ |
| Node.js | 18+ |

### 后端启动

```bash
cd 185-backend
# 1. 创建数据库并执行 init.sql
# 2. 修改 application.yml 数据库配置
# spring.datasource.url=jdbc:mysql://localhost:3306/Shanghai
# spring.datasource.username=root
# spring.datasource.password=你的密码
mvn spring-boot:run
```

### 前端启动

```bash
cd 185-frontend
npm install
# 修改后端接口地址（通常在 .env.development 或 src/utils/request.js 中）
npm run dev
```
