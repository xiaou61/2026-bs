## 项目简介

基于SpringBoot 3.2的中药餐饮推荐交流平台后端系统，提供完整的用户管理、食谱管理、社区交流等核心功能。

**前端：** 中药食疗推荐交流平台是一个专注于中药食疗的社区应用，用户可以浏览、发布食谱，学习食材知识，参与健康讨论。

## 特色亮点

- 个性化推荐算法：基于用户健康需求和浏览行为智能推荐食疗方案
- 完善的食材库：详细的功效、用量、禁忌信息，帮助用户科学搭配
- 活跃社区生态：话题讨论、评论互动、专家问答，建立交流平台
- 创作者激励：认证体系、积分奖励、收益分成，激励内容创作
- 健康档案管理：记录用户健康信息，提供更精准的个性化推荐
- 响应式设计：适配PC和移动端，提供优秀用户体验
- 前后端分离：RESTful API设计，支持快速迭代和功能扩展
- 安全认证：JWT无状态认证、密码加密、角色权限控制

## 演示流程

1. 用户注册登录 → 完善健康档案信息
2. 浏览食谱广场 → 搜索筛选感兴趣的食疗方案
3. 查看食谱详情 → 了解材料、做法、功效、禁忌
4. 收藏喜欢的食谱 → 管理我的收藏列表
5. 浏览食材库 → 查看食材的功效和相克搭配
6. 参与社区讨论 → 发表话题、评论交流、点赞收藏
7. 提问专家 → 获取健康食疗建议
8. 创作者认证 → 申请成为认证用户 → 发布原创内容
9. 查看积分收益 → 参与激励计划 → 申请提现
10. 管理员后台 → 审核内容、管理用户、数据统计

## 快速启动

### 环境要求

| 工具 | 版本 |
|------|------|
| JDK | 17+ |
| Maven | 3.8+ |
| MySQL | 8.0+ |
| Node.js | 18+ |

### 后端启动

```bash
cd 029-backend
# 1. 创建数据库并执行 init.sql
# 2. 修改 application.yml 数据库配置
# spring.datasource.url=jdbc:mysql://localhost:3306/Shanghai
# spring.datasource.username=root
# spring.datasource.password=你的密码
mvn spring-boot:run
```

### 前端启动

```bash
cd 029-frontend
npm install
# 修改后端接口地址（通常在 .env.development 或 src/utils/request.js 中）
npm run dev
```

### 开发注意事项

1. 所有涉及用户的操作都需要JWT认证
2. 某些查看类接口不需要认证（食谱列表、话题列表等）
3. 管理员操作需要验证用户类型为ADMIN
4. 数据库表名和字段名使用下划线命名规范
5. 实体类与数据库字段名通过MyBatis-Plus驼峰转下划线自动映射
