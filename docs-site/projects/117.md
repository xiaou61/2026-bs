## 角色权限

系统支持 **4** 种角色，各角色权限如下：

### 管理员

- 维护账号权限
- 商户资料
- 门店和活动配置。

### 商户

- 查看门店核销
- 结算单和打款记录。

### 收银员

- 执行到店券码核销和撤销操作。

### 财务

- 核算结算单
- 确认明细并执行商户打款。

## 功能模块

| 模块 | 说明 |
|------|------|
| **账号权限** | 管理员/商户/收银员/财务四角色登录、JWT鉴权和Redis登录态 |
| **商户档案** | 管理本地生活商户主体、行业类型、联系人、结算周期和合作状态 |
| **门店网点** | 管理商户门店、城市商圈、地址、联系人和营业状态 |
| **用户档案** | 管理本地生活用户、手机号、等级、注册来源和账号状态 |
| **券模板** | 管理服务券名称、券类型、面额、门槛、有效期和模板状态 |
| **营销活动** | 管理活动名称、适用券、商户、发放总量、日期和活动状态 |
| **用户领券** | 管理用户服务券、领券时间、有效期、来源渠道和可用状态 |
| **核销记录** | 管理券码核销、门店、收银员、消费金额、核销时间和核销状态 |
| **商户结算** | 管理商户结算单、周期、核销金额、平台佣金、应结金额和结算状态 |
| **结算明细** | 管理结算单明细、核销编号、券名称、消费金额、补贴金额和确认状态 |
| **打款记录** | 管理商户打款单、结算单、银行账号、打款金额、打款时间和状态 |
| **申诉工单** | 管理用户或商户申诉、问题类型、处理人、解决方案和工单状态 |
| **活动统计** | 管理活动发放、领取、核销、GMV、转化率和统计状态 |
| **操作日志** | 记录关键模块操作人、动作、对象、详情、IP地址和执行状态 |

## 特色亮点

- 选题贴合本地生活、到店消费、服务券核销、商户结算和平台补贴方向
- 形成“商户档案-门店网点-券模板-营销活动-用户领券-到店核销-结算明细-商户打款-申诉处理”完整闭环
- 使用 MyBatis-Plus 分页查询和通用 CRUD 服务复用，与 116 的 MyBatis 注解 SQL 路线继续交替
- 商户、门店、券模板、营销活动、用户领券、核销记录、结算单、打款记录和申诉工单都有状态流转动作
- 首页看板展示券模板、营销活动、核销记录、结算单指标，以及近7日核销趋势和行业核销分布

## 演示流程

1. 管理员维护账号权限、商户档案和门店网点
2. 管理员创建服务券模板并配置营销活动
3. 用户在活动中领取服务券并到店消费
4. 收银员执行券码核销和撤销处理
5. 平台按核销记录生成结算明细和商户结算单
6. 财务完成结算核算、商户打款和异常申诉处理
7. 首页看板查看核销趋势、行业分布和活动效果统计

## 快速启动

### 环境要求

| 工具 | 版本 |
|------|------|
| JDK | 8+ / 11+ |
| Maven | 3.8+ |
| MySQL | 8.0+ |
| Node.js | 18+ |

### 后端启动

```bash
cd 117-backend
# 1. 创建数据库并执行 init.sql
# 2. 修改 application.yml 数据库配置
# spring.datasource.url=jdbc:mysql://localhost:3306/Shanghai
# spring.datasource.username=root
# spring.datasource.password=你的密码
mvn spring-boot:run
```

### 前端启动

```bash
cd 117-frontend
npm install
# 修改后端接口地址（通常在 .env.development 或 src/utils/request.js 中）
npm run dev
```
