## 项目简介

### 1.1 项目背景
校园快递代领服务平台是一个解决大学生取快递痛点的校园服务平台。很多学生因为课程、社团活动等原因无法及时取快递，而快递站点通常距离宿舍较远，取快递耗时耗力。本平台通过"众包代取"模式，让有时间的学生帮助没时间的学生代领快递，同时赚取跑腿费。

## 特色亮点

- 众包代取模式：让有时间的学生帮没时间的学生代领快递，双向互利
- 智能定价系统：基础价2元+重量加价+距离加价+时效加价，自动推荐合理价格
- 完善信用体系：100分初始、好评+2分、差评-5分、超时-10分、投诉-20分
- 资金冻结机制：发单时冻结跑腿费、确认收货后转账、订单取消自动退款
- 凭证上传功能：取件凭证+送达凭证，拍照留证防止纠纷
- 订单超时管理：30分钟无人接单自动取消、60分钟未取件扣信用分
- 接单限制规则：同时最多接3单、信用分低于60分不可接单
- 实时通知系统：订单状态变更、账户变动及时推送
- 投诉仲裁机制：用户投诉、管理员审核、平台仲裁
- 交易流水清晰：充值、消费、收入、提现、退款全记录

## 演示流程

1. 用户注册 → 填写学号姓名宿舍 → 获得100信用分
2. 发单用户充值余额 → 发布代领需求 → 填写快递信息（菜鸟驿站、取件码）
3. 选择物品类型（文件/服饰/数码等） → 设置跑腿费 → 系统冻结费用
4. 代领员浏览订单广场 → 筛选合适订单 → 点击接单
5. 前往快递站点取件 → 上传取件凭证（快递照片）
6. 送到宿舍楼 → 上传送达凭证（门牌照片）
7. 发单人收到通知 → 确认收货 → 解冻跑腿费转账给代领员
8. 双方互评 → 好评+2信用分 → 订单完成
9. 代领员提现收益 → 查看累计收入
10. 管理后台 → 处理投诉、监控订单、数据统计

### 后端启动

```bash
cd 016-backend
# 1. 创建数据库并执行 init.sql
# 2. 修改 application.yml 数据库配置
# spring.datasource.url=jdbc:mysql://localhost:3306/campus_express
# spring.datasource.username=root
# spring.datasource.password=你的密码
mvn spring-boot:run
```

### 前端启动

```bash
cd 016-frontend
npm install
# 修改后端接口地址（通常在 .env.development 或 src/utils/request.js 中）
npm run dev
```
