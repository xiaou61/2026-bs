### 管理员

- 维护账号权限

### 主播运营

- 维护渠道
- 主播
- 选品
- 排期
- 样品和话术脚本。

### 客服专员

- 处理售后工单
- 退款审核和服务闭环。

### 商家

- 维护供应商品牌
- 配合选品和样品寄送。

## 特色亮点

- 选题贴合直播电商、内容运营、选品排期、售后服务和主播绩效方向
- 形成“供应商品牌-选品池-样品申请-话术脚本-直播排期-直播订单-售后退款-主播绩效”完整闭环
- 使用 MyBatis 注解 SQL + PageHelper，与 115 的 MyBatis-Plus 路线继续交替
- 选品池、直播场次、排期计划、样品申请、话术脚本、直播订单、售后工单、退款记录和主播绩效都有状态流转动作
- 首页看板展示选品、场次、订单、售后、退款、绩效指标，以及GMV趋势和售后类型分布

## 演示流程

1. 管理员维护账号权限、直播渠道和主播档案
2. 商家维护供应商品牌并提交候选商品
3. 主播运营完成选品入围、样品申请和话术审核
4. 主播运营创建直播场次并确认排期计划
5. 直播结束后沉淀直播订单和主播绩效
6. 客服专员处理售后工单和退款记录
7. 首页看板查看GMV趋势、售后类型分布和核心指标

## 快速启动

### 环境要求

| 工具 | 版本 |
|------|------|
| JDK | 8+ / 11+ |
| Maven | 3.8+ |
| MySQL | 8.0+ |
| Node.js | 18+ |

### 后端启动

```bash
cd 116-backend
# 1. 创建数据库并执行 init.sql
# 2. 修改 application.yml 数据库配置
# spring.datasource.url=jdbc:mysql://localhost:3306/Shanghai
# spring.datasource.username=root
# spring.datasource.password=你的密码
mvn spring-boot:run
```

### 前端启动

```bash
cd 116-frontend
npm install
# 修改后端接口地址（通常在 .env.development 或 src/utils/request.js 中）
npm run dev
```
