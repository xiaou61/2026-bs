## 项目简介

城市噪声投诉监测与执法协同平台面向城市噪声治理协同场景，提供多角色登录、投诉受理、执法处置、整改复测、统计看板和操作留痕能力。

## 功能模块

| 模块 | 说明 |
|------|------|
| **账号权限** | 系统管理员/市民投诉人/执法人员/环保监督员四角色登录、JWT鉴权和Redis登录态 |

| **现场巡查** | 巡查编号、投诉标题、巡查人、巡查时间、巡查结论和状态维护 |
| **整改通知** | 通知编号、投诉标题、整改要求、下发时间、责任单位和状态维护 |
| **复测记录** | 复测编号、投诉标题、复测人、分贝值、复测时间和状态维护 |
| **处罚决定** | 处罚编号、投诉标题、处罚类型、处罚对象、处罚金额和状态维护 |
| **公众回访** | 回访编号、投诉标题、回访主题、回访方式、回访时间和状态维护 |
| **预警规则** | 规则编号、投诉标题、预警指标、阈值配置、生效时间和状态维护 |
| **公告公示** | 公告编号、投诉标题、公告类型、公告内容、发布时间和状态维护 |

## 特色亮点

- 选题贴合城市噪声治理、投诉受理、点位监测、执法协同和整改复测方向
- 形成“投诉工单-监测点位-噪声源档案-执法人员档案-处置任务-现场巡查-整改通知-复测记录-处罚决定-公众回访-预警规则-公告公示”完整闭环
- 使用 MyBatis-Plus 分页查询与通用 CRUD 服务复用，与 146 的 MyBatis 注解 SQL 路线继续交替
- 处置任务、现场巡查、整改通知、复测记录、处罚决定和公告公示都有状态流转动作
- 首页看板展示投诉、处置、整改、公示指标，以及噪声处置趋势和工单状态分布
- Spring Boot + Vue3 前后端分离
- MyBatis-Plus 分页查询与通用 CRUD 服务复用
- JWT 登录态拦截与 Redis Token 校验
- 投诉受理、现场巡查、整改复测和公告公示状态机联动
- ECharts 展示噪声处置趋势和工单状态分布
- Element Plus 通用数据页复用，统一搜索、表格、分页、弹窗和状态动作交互
- 城市噪声投诉监测与执法协同平台类毕设项目
- 城市治理、环境监管、执法协同、投诉处置和预警管理课程设计
- 多角色后台管理系统综合实战
- 需要展示投诉闭环、监测联动、执法流程和结果公示能力的答辩项目

## 快速启动

### 环境要求

| 工具 | 版本 |
|------|------|
| JDK | 8+ / 11+ |
| Maven | 3.8+ |
| MySQL | 8.0+ |
| Node.js | 18+ |

### 后端启动

```bash
cd 145-backend
# 1. 创建数据库并执行 init.sql
# 2. 修改 application.yml 数据库配置
# spring.datasource.url=jdbc:mysql://localhost:3306/Shanghai
# spring.datasource.username=root
# spring.datasource.password=你的密码
mvn spring-boot:run
```

### 前端启动

```bash
cd 145-frontend
npm install
# 修改后端接口地址（通常在 .env.development 或 src/utils/request.js 中）
npm run dev
```
