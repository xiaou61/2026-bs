## 功能模块

| 模块 | 说明 |
|------|------|
| **用户管理** | 注册登录、JWT认证、三角色权限（管理员/运维人员/普通用户）、个人信息管理 |
| **站点管理** | 站点CRUD、位置信息、容量管理、当前车辆数量统计 |
| **车辆管理** | 单车信息维护、状态管理（空闲/使用中/维修中/报废）、电量管理、站点分配 |
| **骑行系统** | 扫码开锁、实时骑行状态、行程结束结算、骑行历史查询 |
| **计费规则** | 分段计费（1.5元/30分钟起步+1元/额外30分钟）、车型系数（电动车1.5倍）、每日封顶（普通20元/电动30元） |
| **电子钱包** | 余额充值、消费记录、充值记录、余额查询 |
| **押金管理** | 99元押金缴纳、退还管理、押金状态查询 |
| **信用体系** | 100分初始信用、骑行加分、违规扣分、信用等级展示 |
| **故障上报** | 用户故障上报、运维人员处理、处理状态跟踪（待处理/处理中/已解决） |
| **订单管理** | 订单列表、订单详情、订单状态流转（骑行中→待支付→已完成/已取消） |
| **公告管理** | 系统公告发布、公告列表、公告详情 |
| **意见反馈** | 用户反馈提交、管理员回复、反馈状态管理 |
| **数据统计** | 运营看板、骑行统计、收入统计、ECharts可视化 |

## 演示流程

1. 用户注册登录 → JWT Token认证
2. 缴纳99元押金 → 解锁骑行权限
3. 充值钱包余额 → 准备骑行消费
4. 浏览附近站点 → 查看可用车辆
5. 扫码开锁骑行 → 实时显示骑行状态
6. 到达目的地还车 → 系统自动计费结算
7. 钱包余额扣款 → 查看消费记录
8. 查看骑行历史 → 骑行详情与费用明细
9. 发现故障上报 → 运维人员接单处理
10. 信用分自动更新 → 正常骑行加分
11. 管理员查看看板 → ECharts运营数据可视化
12. 发布系统公告 → 用户查看最新通知

## 快速启动

### 环境要求

| 工具 | 版本 |
|------|------|
| JDK | 8+ / 11+ |
| Maven | 3.8+ |
| MySQL | 8.0+ |
| Node.js | 18+ |

### 后端启动

```bash
cd 071-backend
# 1. 创建数据库并执行 init.sql
# 2. 修改 application.yml 数据库配置
# spring.datasource.url=jdbc:mysql://localhost:3306/Shanghai
# spring.datasource.username=root
# spring.datasource.password=你的密码
mvn spring-boot:run
```

### 前端启动

```bash
cd 071-frontend
npm install
# 修改后端接口地址（通常在 .env.development 或 src/utils/request.js 中）
npm run dev
```
