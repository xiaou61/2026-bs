> 基于SpringBoot+Vue3的养老机构床位分配与护理记录管理系统

## 项目简介

本系统面向养老机构入住评估、床位分配、护理计划、护理记录、家属查询与异常上报场景，提供护理区域、房间床位、老人档案、家属档案、入住评估、床位分配、护理计划、护理记录、家属查询和异常上报的一体化后台管理能力。

## 技术栈

### 后端

- Spring Boot 2.7.18
- MyBatis 2.3.1 注解 SQL
- PageHelper 1.4.7
- MySQL 8.0
- Redis 登录态缓存
- JWT 认证

### 前端

- Vue 3.4.0
- Element Plus 2.4.4
- Vite 5
- Pinia 2.1.7
- Axios 1.6.2
- ECharts 5.4.3

## 角色权限

系统支持 **6** 种角色，各角色权限如下：

### - 系统管理员

- 账号权限
- 基础配置和全部数据维护

### - 入住管理员

- 维护老人档案
- 家属档案
- 入住评估和床位分配

### - 护理主管

- 维护护理区域
- 护理计划
- 护理记录和异常复核

### - 护理人员

- 执行护理计划
- 填写护理记录并上报异常

### - 家属用户

- 查看老人档案
- 护理记录
- 家属查询和异常进度

### - 院长管理员

- 查看全院床位
- 护理
- 异常和运营统计

- 多角色后台管理系统综合实战

## 快速启动

### 环境要求

| 工具 | 版本 |
|------|------|
| JDK | 8+ / 11+ |
| Maven | 3.8+ |
| MySQL | 8.0+ |
| Node.js | 18+ |

### 后端启动

```bash
cd 170-backend
# 1. 创建数据库并执行 init.sql
# 2. 修改 application.yml 数据库配置
# spring.datasource.url=jdbc:mysql://localhost:3306/Shanghai
# spring.datasource.username=root
# spring.datasource.password=你的密码
mvn spring-boot:run
```

### 前端启动

```bash
cd 170-frontend
npm install
# 修改后端接口地址（通常在 .env.development 或 src/utils/request.js 中）
npm run dev
```
