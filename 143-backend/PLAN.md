# 社区公益时间银行互助服务平台 开发计划

## 实施范围
- 后端工程：实体、Mapper、Service、Controller、统一响应、JWT 登录、Redis Token 校验、统计接口
- 前端工程：登录页、布局菜单、通用数据页、模块页面、统计看板、接口封装和路由守卫
- 数据脚本：MySQL 建库建表、初始化账号和演示业务数据

## 模块拆分
- 服务项目：项目编号、项目名称、服务主题、服务对象、发布时间和状态维护
- 服务分类：分类编号、分类名称、服务类型、适用人群、建议时长和状态维护
- 居民档案：居民编号、居民姓名、所属社区、服务偏好、联系电话和状态维护
- 志愿者档案：志愿者编号、志愿者姓名、服务专长、可服务时段、加入时间和状态维护
- 服务预约：预约编号、项目名称、预约人、预约时间、预约人数和状态维护
- 服务签到：签到编号、项目名称、签到人、签到时间、服务结果和状态维护
- 时长账户：账户编号、项目名称、账户类型、开户时间、所属用户和状态维护
- 互助兑换：兑换编号、项目名称、兑换人、兑换时长、兑换时间和状态维护
- 评价反馈：反馈编号、项目名称、反馈类型、评价对象、满意度和状态维护
- 公益活动：活动编号、项目名称、活动主题、活动地点、活动时间和状态维护
- 积分规则：规则编号、项目名称、积分项、积分值、生效时间和状态维护
- 站内通知：通知编号、项目名称、通知类型、通知内容、发布时间和状态维护
- 操作日志：关键模块操作人、动作、对象、详情和结果记录

## 验收标准
- 具备 PRD、PLAN、后端、前端和 SQL 初始化脚本
- 后端端口、前端端口、数据库名、Redis DB 与项目编号一致
- 静态结构达到 14 张表、14 个实体、14 个 Mapper、16 个 Controller、17 个前端页面
- 无旧项目包名、标题、端口、数据库名残留
