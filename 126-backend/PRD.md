# 家庭能源用电分析与节能建议平台 PRD

## 1. 项目背景
居民家庭用电设备增多，用户需要理解账单、设备能耗、异常用电和节能空间，平台用于沉淀家庭能源数据并生成可执行的节能建议。

## 2. 用户角色
系统提供四类默认演示角色：ADMIN（admin / 123456）、RESIDENT（resident / 123456）、ANALYST（analyst / 123456）、MAINTAINER（maintainer / 123456）。

## 3. 功能模块
1. 账号权限：管理员、家庭用户、能耗分析师、维修员四角色账号、JWT 鉴权和 Redis 登录态。
2. 家庭档案：家庭编号、户主、城市、小区、住房面积和家庭人数管理。
3. 家庭成员：成员编号、家庭、姓名、关系、手机号和用电偏好管理。
4. 智能电表：电表编号、家庭、安装位置、通信方式、倍率和在线状态管理。
5. 用电设备：设备编号、家庭、设备名称、设备类型、额定功率和启用状态管理。
6. 用电读数：读数编号、电表、采集时间、总电量、峰电量和谷电量管理。
7. 电费账单：账单编号、家庭、月份、用电量、电费金额和支付状态管理。
8. 设备用电：用电编号、设备、日期、用电量、运行时长和能耗占比管理。
9. 能耗预算：预算编号、家庭、月份、预算电量、预算金额和执行状态管理。
10. 节能建议：建议编号、家庭、设备、建议类型、预计节省和发布状态管理。
11. 异常预警：预警编号、家庭、设备、预警等级、原因和处理人管理。
12. 碳排统计：统计编号、家庭、月份、用电量、碳排量和同比变化管理。
13. 维修工单：工单编号、家庭、设备、问题类型、维修员和预约时间管理。
14. 操作日志：操作人、模块、动作、对象、详情和IP地址审计。

## 4. 核心流程
1. 管理员维护账号、基础档案和业务主数据。
2. 业务人员登记预约、采集、账单或建议记录并推进状态流转。
3. 管理人员查看预测、节能、收益和异常指标。
4. 系统通过首页看板展示趋势、分布和关键数量。
5. 操作日志沉淀关键动作，方便答辩时展示闭环。

## 5. 非功能要求
- 后端采用 Spring Boot，前端采用 Vue3。
- 数据库使用 MySQL，默认账号密码为 root / 1234。
- 登录态使用 JWT 与 Redis 校验。
- 数据页支持分页查询、关键词检索、状态筛选、新增、编辑、删除和状态动作。
