# 智慧停车诱导与空位预测平台 PRD

## 1. 项目背景
城市停车难需要整合停车场、车位、传感器、预约、入离场、收费和空位预测能力，平台用于提升车位利用率和车主停车体验。

## 2. 用户角色
系统提供四类默认演示角色：ADMIN（admin / 123456）、OPERATOR（operator / 123456）、GUARD（guard / 123456）、ANALYST（analyst / 123456）。

## 3. 功能模块
1. 账号权限：管理员、运营员、岗亭员、分析员四角色账号、JWT 鉴权和 Redis 登录态。
2. 停车场：停车场编号、名称、城市、地址、总车位和运营商管理。
3. 停车区域：区域编号、停车场、区域名称、楼层、车位数量和开放状态管理。
4. 车位档案：车位编号、所属区域、车位类型、车位编号牌、收费类型和状态管理。
5. 车位传感器：传感器编号、车位、设备类型、电量、信号强度和在线状态管理。
6. 车主车辆：车辆编号、车主、车牌、车型、手机号和会员等级管理。
7. 预约订单：预约单号、车主、停车场、车位、预约时间和预计时长管理。
8. 停车记录：停车编号、预约单、车牌、入场时间、离场时间和停车时长管理。
9. 支付记录：支付编号、停车编号、车主、金额、支付方式和支付时间管理。
10. 空位预测：预测编号、停车场、预测时段、预测空位、置信度和模型版本管理。
11. 诱导屏幕：屏幕编号、停车场、位置、显示内容、刷新时间和在线状态管理。
12. 导航路线：路线编号、停车场、入口、目标区域、距离和拥堵等级管理。
13. 故障报修：报修编号、对象、故障类型、严重等级、上报人和上报时间管理。
14. 操作日志：操作人、模块、动作、对象、详情和IP地址审计。

## 4. 核心流程
1. 管理员维护账号、基础档案和业务主数据。
2. 业务人员登记预约、采集、账单或建议记录并推进状态流转。
3. 管理人员查看预测、节能、收益和异常指标。
4. 系统通过首页看板展示趋势、分布和关键数量。
5. 操作日志沉淀关键动作，方便答辩时展示闭环。

## 5. 非功能要求
- 后端采用 Spring Boot，前端采用 Vue3。
- 数据库使用 MySQL，默认账号密码为 root / 1234。
- 登录态使用 JWT 与 Redis 校验。
- 数据页支持分页查询、关键词检索、状态筛选、新增、编辑、删除和状态动作。
