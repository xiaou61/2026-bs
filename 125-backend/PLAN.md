# 实现计划

## 问题陈述

构建一个围绕停车诱导、空位预测和收费对账的前后端分离系统，覆盖停车场、停车区域、车位档案、传感监测、预约订单、停车记录、支付记录和导航服务。

## 当前状态

`125-backend` 与 `125-frontend` 已完成智慧停车平台的基础实现，当前进入精修阶段，重点核对预约履约、空位预测、认证安全和文档准确性。

## 实施方案

### 第一阶段：后端开发
1. Spring Boot 2.7.18 + MyBatis-Plus 3.5.5 + Redis + JWT
2. 创建 `smart_parking_125` 数据库和 14 张业务表
3. 实现统一返回、异常处理、JWT 拦截和 Redis Token 校验
4. 实现 Entity、Mapper、Service 和 Controller
5. 实现空位预测、预约导航、停车计费、支付对账和预警统计
6. 实现首页统计和操作日志

### 第二阶段：前端开发
1. Vue3 + Vite + Element Plus + Pinia + Axios + ECharts
2. 登录页、布局页、首页看板
3. 停车场、停车区域、车位档案、车位传感器、车主车辆、预约订单、停车记录、支付记录和日志页面
4. 业务按钮支持预约、入场、离场、支付和查看空位趋势

### 第三阶段：文档更新
1. 更新 `readme.md`
2. 更新 `readme_simple.md`
3. 更新候选清单

## 完成标准

1. 后端结构完整。
2. 前端结构完整。
3. SQL 包含演示数据。
4. 认证返回不暴露敏感字段。
5. 空位预测、预约订单、停车记录和收费对账口径一致。
6. 文档描述与实际实现保持一致。
7. 不执行编译验证。
