# 社区慢病档案随访与服药依从性管理系统 PRD

## 项目概述
本系统面向社区慢病患者建档、随访计划、服药提醒、健康指标采集、风险分层和依从性管理场景，提供社区站点、患者档案、医护团队、慢病档案、随访计划、随访记录、用药方案、服药打卡、健康指标、风险分层和提醒通知的一体化后台管理能力。

## 用户角色
- 系统管理员：账号权限、基础配置和全部数据维护
- 社区卫生中心：维护社区站点、慢病档案、随访计划和统计规则
- 家庭医生：维护患者档案、慢病档案、随访计划和风险分层
- 随访护士：执行随访记录、健康指标采集、提醒通知和异常回访
- 用药药师：维护用药方案、服药打卡、用药依从性和提醒通知
- 居民患者：查看慢病档案、随访计划、服药提醒和健康指标

## 功能模块
- 社区站点：站点编号、站点名称、服务区域、负责人、启用时间和站点状态维护
- 患者档案：患者编号、患者姓名、慢病类型、签约医生、建档时间和档案状态维护
- 医护团队：团队编号、团队名称、服务类型、团队负责人、组建时间和团队状态维护
- 慢病档案：档案编号、患者姓名、疾病类型、管理医生、建档时间和管理状态维护
- 随访计划：计划编号、随访患者、随访类型、责任人员、计划时间和计划状态维护
- 随访记录：记录编号、随访患者、随访方式、随访人员、随访时间和随访状态维护
- 用药方案：方案编号、用药患者、药品类型、指导药师、启用时间和方案状态维护
- 服药打卡：打卡编号、用药患者、打卡类型、核验人员、打卡时间和依从状态维护
- 健康指标：指标编号、监测患者、指标类型、采集人员、采集时间和指标状态维护
- 风险分层：分层编号、评估患者、风险类型、评估医生、评估时间和风险状态维护
- 提醒通知：通知编号、通知患者、通知类型、发送人员、发送时间和通知状态维护
- 操作日志：日志编号、操作模块、操作类型、操作人、操作时间和执行结果维护

## 技术栈
- 后端：Spring Boot 2.7.18、MyBatis 注解 SQL、PageHelper、MySQL、Redis、JWT
- 前端：Vue3、Element Plus、Pinia、Axios、ECharts、Vite
