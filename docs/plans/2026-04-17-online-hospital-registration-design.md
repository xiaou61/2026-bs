# 线上医院挂号设计文档

## 目标
构建一个同时覆盖管理员、医生、患者三类角色的线上医院挂号系统，支持患者完成线上挂号与支付，支持医生完成排班与接诊查看，支持管理员统一维护医院基础资料与运营看板。

## 架构选择
- 后端采用 Spring Boot + MyBatis + PageHelper，沿用仓库已有轻量后端模式，方便快速落地分页查询与 Mapper XML。
- 前端采用单一 Vue3 项目承载管理端与患者端，通过角色菜单和接口数据区分视图，保持 09x 项目现有交付风格。
- 业务闭环以“科室 -> 医生 -> 排班 -> 挂号 -> 订单 -> 评价”为主线，公告与轮播用于首页展示，统计模块用于毕设演示。

## 角色设计
- 管理员：维护用户、科室、医生、排班、公告、轮播、订单和统计数据。
- 医生：查看个人资料、维护个人介绍、查看个人排班和预约患者、更新接诊状态。
- 患者：注册登录、查看医院内容、选择医生排班、创建挂号、支付订单、取消预约、提交评价、维护就诊卡。

## 数据模型
- `sys_user` 负责登录认证与基础身份字段。
- `patient_profile` 与 `medical_card` 承载患者补充档案信息。
- `department_info`、`doctor_info`、`doctor_schedule` 描述医院核心资源。
- `appointment_record` 与 `payment_order` 负责挂号与支付闭环。
- `doctor_review` 用于患者评价医生。
- `news_notice` 与 `banner_info` 负责内容展示。
- `operation_log` 用于记录关键操作，便于统计和补充演示。

## 页面设计
- 登录页支持登录与患者注册。
- 首页根据角色展示不同卡片和内容，管理员看统计，医生看个人业务，患者看轮播和公告。
- 管理端采用左侧菜单布局，患者端也复用同一布局，减少工程分裂。
- 核心业务页面统一采用“筛选栏 + 表格 + 分页 + 弹窗”的仓库常见样式，保证风格一致。

## 关键流程
1. 患者注册登录后完善档案与就诊卡。
2. 患者进入医生大厅，按科室筛选医生。
3. 选择医生后查看排班和剩余号源。
4. 提交挂号生成预约记录与支付订单。
5. 患者支付订单后状态变为已支付，医生端可以查看预约患者。
6. 患者取消未完成预约时释放剩余号源。
7. 已完成的预约允许患者提交医生评价。

## 异常处理
- 未登录或 token 失效统一返回 401。
- 非管理员访问管理接口统一返回 403。
- 挂号时校验剩余号源、患者卡片、排班状态和重复预约。
- 支付、取消、评价等操作校验当前预约状态，避免非法流转。
