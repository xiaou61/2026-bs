# 093 自助售货管理系统设计

## 目标
基于现有 SpringBoot + Vue3 项目骨架，落地一个面向校园、园区、写字楼等场景的自助售货管理系统，覆盖设备管理、商品管理、货道库存、补货维护、顾客下单、支付出货、异常申报和经营看板。

## 设计结论
采用 `091` 电影院会员管理系统作为主骨架，保留其 `Spring Boot 2.7.18 + MyBatis-Plus + Redis + Vue3 + Element Plus + ECharts` 技术组合与登录态方案；再吸收 `063` 进销存管理系统中商品、库存、业务流水和运营看板的结构思路，完成 093 的领域切换。

## 角色设计
- 管理员：维护用户、点位、售货机、商品、公告和统计看板
- 补货员：查看设备货道、提交补货记录、处理故障反馈、跟踪出货异常
- 顾客：浏览设备商品、下单支付、查看订单、反馈故障、维护个人资料

## 核心模块
### 1. 账号与权限
- 管理员、补货员、顾客三角色登录
- JWT + Redis 登录态校验
- 个人资料与密码修改

### 2. 点位与设备
- 点位信息维护
- 售货机基础资料维护
- 设备在线状态、温区、编号、负责人维护

### 3. 商品与货道
- 商品分类管理
- 商品资料、售价、成本价、规格、库存预警维护
- 设备货道维护，记录货道编号、商品、库存、容量、状态

### 4. 补货与出货
- 补货员按设备货道录入补货记录
- 支付成功后自动扣减货道库存
- 出货记录沉淀成功、异常、退款状态

### 5. 顾客消费
- 顾客查看设备与商品
- 创建订单、余额支付
- 查看订单状态和设备取货码

### 6. 反馈与公告
- 顾客提交故障或未出货反馈
- 管理端处理故障与公告

### 7. 经营看板
- 今日销售额、订单数、设备数、商品数
- 近 7 天销售趋势
- 热销商品排行
- 设备销售排行

## 数据模型
系统使用 12 张核心表：
1. `sys_user`
2. `machine_location`
3. `vending_machine`
4. `product_category`
5. `product_info`
6. `machine_slot`
7. `replenish_record`
8. `sale_order`
9. `payment_record`
10. `shipment_record`
11. `fault_report`
12. `system_notice`

## 数据流
1. 管理员创建点位、设备、商品和货道
2. 补货员补货后更新货道库存
3. 顾客浏览设备商品并创建订单
4. 顾客余额支付成功后写入支付记录
5. 系统扣减货道库存并生成出货记录
6. 顾客可查看订单与取货码，异常时提交故障反馈
7. 统计模块基于订单、支付、设备和商品数据生成看板

## 后端改造策略
- 包名从 `com.cinema` 切换为 `com.vending`
- 保留通用返回、异常处理、JWT、Redis、MP 配置
- 删除影院相关实体、DTO、Service、Controller
- 重建自助售货领域实体、Mapper、Service、Controller
- 复用 `091` 的登录和支付骨架，复用 `063` 的商品/库存分页查询思路

## 前端改造策略
- 复制 `091-frontend` 为主骨架
- 路由重建为设备、商品、货道、补货、订单、出货、故障、公告、统计、个人中心
- 菜单按管理员/补货员/顾客分角色显示
- 页面风格延续现有中后台样式，保证与仓库其他项目一致

## 风险与取舍
- 不做编译验证，因此实现以结构一致、命名清晰和文本自检为主
- 093 重点展示完整业务链路，支付采用余额支付模拟，不接真实支付通道
- Redis 用于登录态和下单短时订单/设备流程辅助，不额外扩展复杂分布式锁
