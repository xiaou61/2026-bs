# 095 足球联赛管理系统设计

## 目标
基于现有 SpringBoot + Vue3 项目骨架，落地一个面向校园联赛、城市业余联赛和协会赛事场景的足球联赛管理系统，覆盖联赛赛季、俱乐部球队、教练球员、比赛赛程、赛果录入、积分榜、资讯公告和统计看板。

## 设计结论
采用仓库最近项目的 `Spring Boot 2.7.18 + MyBatis-Plus + Redis + Vue3 + Element Plus + ECharts` 技术组合与统一登录态方案，沿用现有中后台分页查询、保存、删除和看板展示模式，将领域切换为足球联赛、球队与比赛管理。

## 角色设计
- 管理员：维护账号、联赛、赛季、公告和总览统计
- 俱乐部经理：维护俱乐部、球队、教练、球员、球场和比赛结果
- 球迷用户：查看赛程、积分榜、球队资料、公告资讯并关注球队

## 核心模块
### 1. 账号与权限
- 管理员、俱乐部经理、球迷三角色登录
- JWT + Redis 登录态校验
- 个人资料与密码修改

### 2. 联赛与赛季
- 联赛基础资料维护
- 赛季配置与状态切换
- 联赛与赛季下拉联动

### 3. 俱乐部与球队
- 俱乐部资料维护
- 球队信息、主场、赛季归属维护
- 球迷查看球队资料

### 4. 教练与球员
- 教练档案维护
- 球员编号、位置、号码、年龄、国籍、进球数、助攻数维护
- 球员与球队关联展示

### 5. 球场与赛程
- 球场资料维护
- 比赛轮次、开赛时间、主客队、比赛状态维护
- 比分录入与完赛状态流转

### 6. 积分榜与关注
- 根据赛果自动刷新积分榜
- 统计胜平负、进失球、净胜球和积分
- 球迷关注球队并查看关注列表

### 7. 资讯与看板
- 公告资讯发布
- 总比赛数、已完赛数、总进球数、球队数、球迷数统计
- 近 7 天赛程趋势和积分榜排行展示

## 数据模型
系统使用 12 张核心表：
1. `sys_user`
2. `league_info`
3. `season_info`
4. `club_info`
5. `team_info`
6. `coach_info`
7. `player_info`
8. `venue_info`
9. `match_schedule`
10. `standing_record`
11. `news_notice`
12. `fan_follow`

## 数据流
1. 管理员创建联赛和赛季
2. 俱乐部经理维护俱乐部、球队、教练、球员和主场
3. 管理端维护赛程，配置主客队、轮次和比赛时间
4. 俱乐部经理录入比赛比分并将比赛状态更新为完赛
5. 系统根据赛果重算当前赛季积分榜
6. 球迷查看赛程、积分榜、球队资料与公告，并关注球队
7. 统计模块基于比赛、球队、球员和关注数据生成看板

## 后端改造策略
- 包名使用 `com.football`
- 保留统一返回、异常处理、JWT、Redis、MP 配置
- 重建联赛、赛季、俱乐部、球队、教练、球员、球场、赛程、积分榜、公告和关注领域实体
- 赛果录入后触发积分榜重算
- 公共查询接口优先服务球迷端查看赛程、积分榜和资讯

## 前端改造策略
- 保留 Vue3 + Element Plus 管理台风格
- 路由重建为用户、联赛、赛季、俱乐部、球队、教练、球员、球场、赛程、积分榜、资讯、关注、统计、个人中心
- 菜单按管理员、俱乐部经理、球迷角色显示
- 首页区分管理端总览与球迷欢迎页

## 风险与取舍
- 不做编译验证，实现以结构一致和文本自检为主
- 不接入真实赛程编排算法，赛程以人工维护为主
- 不实现复杂转会、伤病和票务，只保留毕设最核心的联赛管理闭环
