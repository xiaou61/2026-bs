# 085 数学课程评价系统设计文档

- 日期：2026-03-10
- 项目目录：`085-backend` / `085-frontend`

## 1. 设计目标
构建一个面向高校数学课程教学质量改进的评教系统，支持课程维度任务发布、学生多指标评分、评价结果统计分析与公告通知。

## 2. 系统架构

### 2.1 后端
- 技术：Spring Boot + MyBatis(XML) + PageHelper + Redis + JWT
- 模块：认证、课程分类、课程、评价指标、评价任务、课程评价、公告、统计
- 鉴权：JWT解析用户ID/角色，Redis校验会话有效性

### 2.2 前端
- 技术：Vue3 + Vite + Pinia + Element Plus + ECharts
- 页面：登录、布局、看板、课程分类、课程管理、指标管理、任务管理、课程评价、公告管理
- 通信：Axios统一封装，响应拦截处理401

## 3. 数据模型
核心表：
1. `sys_user`
2. `course_category`
3. `math_course`
4. `eval_indicator`
5. `eval_task`
6. `eval_record`
7. `eval_record_item`
8. `system_notice`
9. `operation_log`

关键约束：
- `eval_record` 以 `(task_id, student_id)` 唯一约束防重复提交
- 指标权重在业务层限制启用总和不超过100

## 4. 关键业务流程

### 4.1 任务发布流程
管理员创建任务：关联课程、设置学期与评价时间区间、启用任务。

### 4.2 学生评价流程
学生选择有效任务，按启用指标评分并填写建议；系统校验任务状态、时间范围、是否重复提交后落库。

### 4.3 统计看板流程
系统聚合用户数、课程数、任务数、评价数、平均分、月度趋势、课程均分TOP5用于首页展示。

## 5. 接口分层
- Controller：参数接收与结果返回
- Service：业务校验与流程编排
- Mapper：SQL读写与聚合统计

## 6. 错误处理与边界
- 统一异常：`BusinessException` + 全局异常处理
- 登录失效：返回401，前端拦截后跳转登录
- 业务边界：
  - 非学生角色不可提交评价
  - 任务未启用/超时不可提交
  - 指标权重配置异常时阻断提交

## 7. 测试与验证策略
- 后端：`mvn -DskipTests package` 通过
- 前端：`npm run build` 通过
- 联调重点：登录鉴权、任务提交防重复、看板统计字段映射
