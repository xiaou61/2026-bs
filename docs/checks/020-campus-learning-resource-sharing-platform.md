### 3.2 与 PRD 对照后的结论

- PRD 中描述的核心业务骨架基本都已存在，不是空壳项目
- 但在巡检前，存在多处“文档写了，代码没闭环”的问题，主要集中在：
  - 前端构建配置缺失导致无法正常构建
  - 首页统计未完整接线
  - 积分系统只实现了下载扣分和悬赏提问，未完整覆盖上传资源与回答问题奖励
  - 私密笔记访问控制未收口
  - 登录/用户信息存在密码哈希泄露风险

## 4. 本轮已修复问题

### 4.1 前端

1. 修复 `020-frontend/vite.config.js` 未配置 `@` alias 导致 `npm run build` 失败的问题
2. 调整 `020-frontend/src/router/index.js` 的静态/动态路由顺序，消除 `/resource/upload`、`/qa/ask`、`/note/edit` 被动态路由吞掉的风险
3. 修复 `020-frontend/src/views/Home.vue` 首页统计只加载资源数的问题，补齐学习小组总数、题库题目总数和热门问答列表加载

### 4.2 后端

1. 修复 `UserService.login()` 与 `getUserInfo()` 直接返回密码哈希的问题，统一返回脱敏用户对象
2. 修复 `ResourceService.uploadResource()` 未给上传者加积分、未写积分记录的问题
3. 修复 `AnswerService.addAnswer()` 未给回答者加积分的问题，并补充：
   - 禁止回答已解决问题
   - 禁止自问自答
   - 禁止同一用户重复回答同一问题
4. 为 `ResourceService.downloadResource()` 补充上传者收益积分记录，形成积分流水闭环
5. 为 `QuestionAnswerService.askQuestion()`、`ResourceService`、`AnswerService` 的关键多表写操作补充事务边界
6. 修复 `NoteService` 私密笔记权限边界：
   - 公开列表默认仅返回公开笔记
   - 私密笔记详情仅允许本人访问
7. 修复 `NoteController` 对私密笔记访问仍返回成功包装的问题，改为明确返回 `code=404`

### 4.3 测试

1. 将 `PasswordTest` 从打印 + Java `assert` 改为 JUnit `Assertions`
2. 新增 `ServiceBehaviorTest`，覆盖以下核心规则：
   - 登录返回密码脱敏
   - 上传资源奖励积分
   - 自问自答拦截
   - 有效回答奖励积分
   - 私密笔记禁止越权访问
