## 5. 本次修复

### 5.1 工程与环境修复

1. 补齐后端 `pom.xml` 的 Spring Boot、MyBatis、H2、MySQL、Lombok、测试和插件依赖
2. 将默认数据源从强依赖本地 MySQL 改为 H2 文件库自举，默认端口调整为后端 `8026`
3. 新增 `schema-h2.sql` 和 `data-h2.sql`，每次启动自动重建并写入演示数据
4. 修复原始库表设计问题：订单状态字段由 `VARCHAR(20)` 扩为 `VARCHAR(32)`，避免 `PENDING_FINAL_PAYMENT` 写入失败
5. 前端默认请求地址改为 `http://127.0.0.1:8026`，Vite 开发端口固定为 `3026`
6. 新增后端冒烟测试，覆盖登录、已审核画师列表和订单详情链路

### 5.2 业务与接口修复

1. 修复 `GlobalExceptionHandler` 在底层异常 message 为空时返回 `msg=null` 的问题
2. 修复用户登录、注册、查询接口直接回传明文密码的问题，统一做返回脱敏
3. 修复画师审核通过后用户角色没有真正写回数据库的问题
4. 修复尾款支付后把收益错误加回客户账户而不是画师账户的问题
5. 修复 `DemandMapper` 目标画师名称字段映射错误
6. 修复前端个人中心角色文案拼写错误，避免画师角色显示异常

## 6. 验证结果

### 6.1 构建与测试

1. 后端执行 `mvn test -DskipTests=false`：`通过`
2. 前端执行 `npm run build`：`通过`

### 6.2 启动验证

1. 后端 `mvn spring-boot:run`：`通过`，监听 `8026`
2. 前端 `npm run dev -- --host 0.0.0.0 --port 3026`：`通过`，首页响应 `200`

### 6.3 接口抽测

已实际验证以下链路：

1. `POST /user/login`
   - 账号 `buyer01 / 123456`
   - 返回 `code=200`
   - 返回用户 `buyer01`
   - `password=null`，已脱敏
2. `GET /artist/approved`
   - 返回已审核画师列表
   - 首条画师昵称为 `星川绘`
3. `GET /demand/open`
   - 返回公开需求列表
   - 当前开放需求数量为 `1`
4. `POST /order/pay-final/3`
   - 支付后订单 `3` 状态更新为 `COMPLETED`
   - 画师 `artist01` 余额更新为 `3520.0`
5. `PUT /artist/approve/2`
   - 审核后用户 `4` 角色更新为 `ARTIST`
