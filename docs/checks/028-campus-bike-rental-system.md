## 4. 已修复问题

### 4.1 工程与环境问题

1. `mybatis-plus-boot-starter` 与 `Spring Boot 3.2` 不兼容，启动时报 `factoryBeanObjectType` 异常
   - 已改为 `mybatis-plus-spring-boot3-starter`
2. 默认数据源依赖本机 `MySQL root/root`，实际环境下登录接口触发 `Access denied for user 'root'@'localhost'`
   - 已切换为默认 `H2` 文件库自举
   - 已补充 `schema-h2.sql` 与 `data-h2.sql`
3. 分页插件原来固定为 `DbType.MYSQL`
   - 已改为 `DbType.H2`
4. 后端默认端口与已有进程冲突
   - 已调整后端端口为 `8028`
   - 前端开发端口同步调整为 `3028`
   - 前端代理目标同步改为 `http://localhost:8028`
5. 启动成功提示里的文档地址原来写死为 `8028`
   - 已改为按当前实际端口动态输出

### 4.2 默认演示数据修复

1. 已补齐默认 H2 初始化数据，保证项目开箱可登录、可联调
2. 默认账号已校正为可用 bcrypt 哈希
3. 已内置停车点、车辆、钱包、历史订单、充值记录、故障记录等演示数据

## 7. 与 PRD 的一致性评估

## 7.1 已有闭环

以下 PRD 模块在当前实现中已具备基础闭环：

1. 用户注册、登录、个人信息查看
2. 实名认证提交、后台审核
3. 停车点列表、车辆列表
4. 扫码/编号租车、还车、当前订单、历史订单
5. 钱包余额、充值记录、押金状态
6. 故障上报、故障列表、后台处理
7. 信用分基础加减逻辑
8. 后台用户、车辆、停车点、订单、故障与统计

## 7.2 明显缺口

以下内容与 PRD 仍有明显差距：

1. `订单申诉`
   - 数据表已存在
   - 后端控制器、服务、前端页面均未实现
2. `临时锁车`
   - PRD 提到骑行过程中的临时锁车
   - 当前无对应接口与状态流转
3. `地图找车`
   - 当前是停车点列表和车辆列表
   - 未实现真实地图组件与附近车辆展示
4. `图片上传闭环`
   - 还车照片、故障图片当前只是字符串字段
   - 未实现真实上传能力
5. `信用体系展示`
   - 当前只有信用分字段和加减记录落库逻辑
   - 缺少独立的信用记录、信用等级、规则说明页面/接口闭环
