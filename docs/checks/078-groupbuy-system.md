# 078 网上团购系统检查报告

## 1. 检查结论

- 项目编号：`078`
- 项目名称：`网上团购系统`
- 检查日期：`2026-05-02`
- 当前状态：`已完成`
- 综合结论：`本轮已修复默认环境强依赖 MySQL/Redis/8080、H2 自举缺失、MySQL 驱动旧坐标、后端无自动化测试、登录响应 password 字段泄露、JWT Bearer 不兼容、未登录/越权 HTTP 状态不准确、登出后 token 不失效、普通用户可访问管理接口、购物车/地址/订单/商品/团购活动归属校验不足、前端依赖缺失、代理仍指向 8080、前端请求头缺少 Bearer token、前端 API 导出断链等问题。当前后端可在默认 H2 环境下启动并通过自动化冒烟测试，前端可完成生产构建，真实 HTTP 团购订单主链路和前端代理登录复测通过。`

## 2. 项目形态

- 后端目录：`078-backend`
- 前端目录：`078-frontend`
- 后端技术栈：`Spring Boot 2.7.18 + MyBatis-Plus 3.5.2 + JWT + H2/MySQL + Redis 依赖保留`
- 前端技术栈：`Vue 3 + Vite + Element Plus + Pinia + Axios + ECharts`
- 默认后端端口：`8078`
- 前端开发端口：`3078`

## 3. 本轮修复

1. 后端默认端口从 `8080` 调整为 `8078`。
2. 默认 `application.yml` 改为 H2 内存库自举，新增 `db/schema-h2.sql` 与 `db/data-h2.sql`。
3. 新增 `application-mysql.yml`，保留原 MySQL 部署入口。
4. 修复 MySQL 驱动坐标为 `com.mysql:mysql-connector-j`。
5. 项目编译目标改为 JDK 17，并增加 H2 与 Spring Boot Test 依赖。
6. MyBatis-Plus 分页方言改为配置化，默认使用 `H2`，MySQL profile 使用 `MYSQL`。
7. `User.password` 改为 Jackson `WRITE_ONLY`，登录响应、用户信息响应和前端代理登录均不再回传密码字段。
8. JWT 拦截器兼容 `Authorization: Bearer <token>`。
9. 新增本地运行态 token 失效表，默认演示环境不再依赖 Redis 才能完成登录/登出闭环。
10. 全局异常处理改为真实 HTTP 状态：未登录/失效 `401`，越权 `403`，参数错误 `400`。
11. 用户、商家、分类、公告、评价和统计等管理型接口补充服务端管理员校验。
12. 商品、团购活动、地址、购物车、普通订单和拼团订单补充本人/商家归属校验。
13. 新增评价删除和公告状态切换接口，补齐前端管理页 API 断链。
14. 前端 Vite 开发端口改为 `3078`，代理目标改为 `http://localhost:8078`。
15. 前端请求拦截器自动补齐 `Bearer` token，并在 HTTP `401` 时清理登录态。
16. 修复首页公开团购列表调用，使其使用公开的 `/api/activity/home`。
17. 新增后端冒烟测试 `GroupBuyApplicationSmokeTest`。

## 4. 已具备能力

1. 后端默认 H2 自举，可直接执行 `mvn spring-boot:run` 启动。
2. 管理员、商家和普通用户账号可登录。
3. 登录响应不回传 `password` 字段。
4. 未登录访问受保护接口返回 HTTP `401`。
5. 普通用户访问管理统计返回 HTTP `403`。
6. 管理员可访问统计和评价管理接口。
7. 游客可访问分类、首页团购、商品详情和前台公告。
8. 普通用户可加入购物车、创建订单、支付订单、确认收货和评价。
9. 非订单本人无法支付或确认收货，返回 HTTP `403`。
10. 非订单所属商家无法发货，返回 HTTP `403`。
11. 登出后旧 token 失效，重新访问用户信息返回 HTTP `401`。
12. 前端构建、首页公开数据调用和代理登录已与后端主链路对齐。

1. 密码仍为明文存储，生产环境建议改为 BCrypt。
2. 默认演示模式使用 H2 内存库，MySQL profile 需要按目标机器单独导入 `sql/init.sql`。
3. Redis 依赖保留但当前默认演示链路未使用分布式会话。
4. 支付、退款和成团失败退款为模拟状态流转，未接入真实支付。
5. 拼团成团、库存扣减和订单状态流转未实现高并发锁库存策略。
6. 图片上传和对象存储仍使用 URL 字段，未接入真实文件服务。
7. 前端构建存在 chunk 体积提示，当前不影响构建产物生成。
