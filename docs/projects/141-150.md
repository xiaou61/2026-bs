**后端**
- Spring Boot 2.7.18
- MyBatis-Plus 3.5.5
- MySQL 8.0
- Redis 登录态缓存
- JWT 认证

**前端**

- Element Plus 2.4.4
- Vite 5
- Pinia 2.1.7
- Axios 1.6.2
- ECharts 5.4.3

7. **盘点明细** - 明细编号、任务名称、资产名称、扫描时间、盘点结果和状态维护
8. **借用申请** - 申请编号、资产名称、借用用途、借用日期、借用人和状态维护
9. **借用归还** - 归还编号、资产名称、归还人、归还数量、归还时间和状态维护

11. **折旧记录** - 折旧编号、资产名称、资产原值、累计折旧、计提日期和状态维护
12. **闲置处置** - 处置编号、资产名称、处置方式、处置原因、处置时间和状态维护
13. **预警通知** - 通知编号、资产名称、预警类型、预警内容、发送时间和状态维护

#### 🎓 适合场景

### 150 - 医院门诊检查预约与报告回传管理系统

#### 🏷️ 项目名称
基于SpringBoot+Vue3的医院门诊检查预约与报告回传管理系统

#### 💻 技术栈
**后端**
- Spring Boot 2.7.18
- MyBatis 2.3.1 注解 SQL
- PageHelper 1.4.7
- MySQL 8.0
- Redis 登录态缓存
- JWT 认证

**前端**
- Vue 3.4.0
- Element Plus 2.4.4
- Vite 5
- Pinia 2.1.7
- Axios 1.6.2
- ECharts 5.4.3

#### 🎯 功能模块
1. **账号权限** - 系统管理员/门诊医生/检查技师/患者本人四角色登录、JWT鉴权和Redis登录态
2. **检查项目** - 项目编号、项目名称、检查类别、适用科室、检查时长和状态维护
3. **患者档案** - 患者编号、患者姓名、身份证号、联系电话、就诊卡号和状态维护
4. **医生档案** - 医生编号、医生姓名、联系电话、所属科室、职称等级和状态维护
5. **检查预约** - 预约编号、检查项目、预约日期、检查时段、开单医生和状态维护
6. **检查科室** - 科室编号、科室名称、检查设备、开放时间、候诊区域和状态维护
7. **签到记录** - 签到编号、患者姓名、签到方式、签到窗口、签到时间和状态维护
8. **检查报告** - 报告编号、患者姓名、检查结论、完成时间、检查技师和状态维护
9. **异常预警** - 预警编号、患者姓名、预警类型、预警时间、处理人和状态维护
10. **报告回传** - 回传编号、患者姓名、回传方式、回传时间、接收人和状态维护
11. **复诊建议** - 建议编号、患者姓名、建议主题、建议类型、建议时间和状态维护
12. **队列叫号** - 叫号编号、患者姓名、叫号时间、候诊状态、诊室编号和状态维护
13. **通知公告** - 通知编号、患者姓名、通知类型、通知内容、接收人和状态维护
14. **操作日志** - 关键模块操作人、动作、对象、详情和结果记录

#### ✨ 特色亮点
- 选题贴合门诊检查管理、预约登记、队列叫号、报告回传和异常提醒方向
- 形成“检查项目-患者档案-医生档案-检查预约-检查科室-签到记录-检查报告-异常预警-报告回传-复诊建议-队列叫号-通知公告”完整闭环
- 使用 MyBatis 注解 SQL + PageHelper 分页查询，与 149 的 MyBatis-Plus 路线继续交替
- 检查预约、签到记录、检查报告、异常预警、报告回传和队列叫号都有状态流转动作
- 首页看板展示项目、科室、报告、叫号指标，以及检查预约趋势和检查状态分布
