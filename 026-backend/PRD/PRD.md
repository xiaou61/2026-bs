# 基于SpringBoot的画师接稿平台的设计与实现 - 产品需求文档

## 1. 项目概述

### 1.1 项目背景
随着数字艺术和自媒体行业的蓬勃发展，越来越多的画师通过网络接单创作。然而，传统的接稿方式存在信息不对称、交易不规范、纠纷处理困难等问题。本项目旨在打造一个专业的画师接稿平台，为画师和客户搭建一个安全、便捷、透明的交易桥梁。

### 1.2 项目目标
- 为画师提供展示作品、接收委托、管理订单的一站式服务平台
- 为客户提供浏览画师作品、发布需求、安全交易的便捷渠道
- 建立完善的订单流程和评价体系，保障双方权益
- 通过平台担保交易，降低交易风险

## 2. 用户角色

### 2.1 普通用户/客户
- 浏览画师作品集
- 发布绘画需求/委托
- 下单并支付订金
- 查看订单进度
- 确认作品并支付尾款
- 评价画师服务

### 2.2 画师
- 注册认证画师身份
- 创建和管理个人作品集
- 设置接稿信息（价格、类型、时间等）
- 接受/拒绝委托
- 提交作品草图和成品
- 查看收益和提现

### 2.3 管理员
- 审核画师认证申请
- 管理用户和画师信息
- 处理投诉和纠纷
- 审核提现申请
- 数据统计和平台监控

## 3. 核心功能需求

### 3.1 用户中心

#### 3.1.1 注册与登录
- 手机号/邮箱注册
- 密码登录、找回密码
- 个人信息管理（头像、昵称、简介）

#### 3.1.2 画师认证
- 提交认证申请（上传作品、实名信息）
- 管理员审核
- 认证通过后获得画师权限

### 3.2 画师作品集

#### 3.2.1 作品管理
- 上传作品（图片、标题、描述、标签）
- 作品分类（原创/同人、插画/漫画/设计等）
- 编辑、删除作品
- 设置代表作/精选作品

#### 3.2.2 画师主页
- 展示个人信息和风格介绍
- 作品集展示（瀑布流/网格布局）
- 接稿信息（价格区间、擅长类型、交稿周期）
- 历史评价和好评率
- 关注/收藏功能

### 3.3 需求发布与接稿

#### 3.3.1 需求发布（客户端）
- 填写需求详情
  - 作品类型（人物/场景/logo等）
  - 尺寸规格
  - 风格要求
  - 参考图上传
  - 预算范围
  - 截止日期
- 选择指定画师或公开招标

#### 3.3.2 需求广场
- 展示公开招标的需求
- 按类型/预算/时间筛选
- 画师浏览并报价

#### 3.3.3 接稿流程
- 画师查看需求详情
- 提交报价和预计交稿时间
- 客户选择心仪的画师
- 双方确认订单

### 3.4 订单管理

#### 3.4.1 订单状态流转
1. **待支付** - 客户确认订单后支付定金（30%）
2. **制作中** - 画师创作中，可提交草图
3. **待确认** - 画师提交成品，客户确认
4. **修改中** - 客户提出修改意见（限定次数）
5. **待付尾款** - 客户确认满意，支付尾款（70%）
6. **已完成** - 双方互评，订单结束
7. **已取消** - 特殊情况下取消订单
8. **申诉中** - 存在纠纷，平台介入

#### 3.4.2 订单详情
- 需求信息
- 报价和支付记录
- 草图/成品文件
- 沟通记录
- 修改历史

#### 3.4.3 文件交付
- 画师上传草图（低分辨率预览）
- 客户确认后上传高清成品
- 支持多格式（JPG/PNG/PSD等）
- 文件加密存储，付款后可下载

### 3.5 支付系统

#### 3.5.1 钱包功能
- 账户余额充值
- 订金/尾款支付
- 资金冻结与解冻
- 提现申请（画师）

#### 3.5.2 交易流程
- 客户支付定金，平台冻结资金
- 草图确认后继续创作
- 成品确认后支付尾款
- 平台扣除手续费（5-10%）后转账给画师
- 超时未确认自动确认

### 3.6 评价系统

#### 3.6.1 双向评价
- 客户评价画师（质量/态度/速度）
- 画师评价客户（沟通/配合度）
- 5星评分 + 文字评价
- 评价展示在主页

#### 3.6.2 信用体系
- 好评率统计
- 完成订单数量
- 平均响应时间
- 擅长标签（由评价生成）

### 3.7 沟通系统

#### 3.7.1 站内消息
- 订单相关消息通知
- 系统公告推送

#### 3.7.2 订单聊天（可选）
- 订单内嵌聊天功能
- 文字 + 图片消息
- 沟通记录保存

### 3.8 搜索与推荐

#### 3.8.1 搜索功能
- 按画师名称/标签搜索
- 按作品风格/类型搜索
- 价格区间筛选
- 排序（好评率/订单量/价格）

#### 3.8.2 推荐系统
- 首页推荐热门画师
- 根据浏览历史推荐相似画师
- 新人画师扶持展示

### 3.9 管理后台

#### 3.9.1 用户管理
- 用户列表查看
- 禁用违规账号

#### 3.9.2 画师审核
- 审核认证申请
- 查看作品和资质

#### 3.9.3 订单监控
- 查看所有订单
- 处理申诉和纠纷
- 强制退款/确认

#### 3.9.4 提现管理
- 审核提现申请
- 财务流水记录

#### 3.9.5 数据统计
- 交易额统计
- 活跃用户数
- 订单完成率
- 可视化图表

## 4. 非功能需求

### 4.1 性能要求
- 页面加载时间 < 2秒
- 图片资源CDN加速
- 支持千级并发访问

### 4.2 安全性
- 密码MD5/BCrypt加密
- JWT Token认证
- 支付信息加密传输
- 防SQL注入/XSS攻击
- 敏感操作二次验证

### 4.3 易用性
- 简洁美观的UI设计
- 响应式布局（PC+移动端）
- 操作流程清晰易懂

### 4.4 可扩展性
- 预留社区功能接口
- 支持多种支付方式
- 支持国际化

## 5. 技术栈

### 5.1 后端
- **框架**: Spring Boot 3.x
- **ORM**: MyBatis
- **数据库**: MySQL 8.0
- **缓存**: Redis（可选）
- **认证**: JWT
- **文件存储**: 本地/OSS

### 5.2 前端
- **框架**: Vue 3 + Vite
- **UI库**: Element Plus
- **状态管理**: Pinia
- **HTTP**: Axios
- **路由**: Vue Router

## 6. 数据库设计要点

### 6.1 核心表
- **user** - 用户表（客户+画师）
- **artist** - 画师信息表
- **portfolio** - 作品集表
- **demand** - 需求/委托表
- **order** - 订单表
- **payment** - 支付记录表
- **review** - 评价表
- **message** - 消息表
- **withdraw** - 提现记录表

## 7. 项目亮点

1. **完整的交易闭环**: 从需求发布、报价、订单创建、作品交付到评价，全流程管理
2. **资金安全保障**: 平台担保交易，定金+尾款分阶段支付
3. **双向评价体系**: 建立画师和客户的信用档案
4. **作品展示**: 精美的作品集展示，支持多维度搜索
5. **灵活接稿模式**: 支持指定画师和公开招标两种方式
6. **修改机制**: 允许客户提出有限次数的修改，平衡双方利益
7. **纠纷处理**: 申诉流程和管理员仲裁机制

## 9. 风险与挑战

### 9.1 技术风险
- 文件上传和存储的性能优化
- 支付流程的安全性保障
- 并发订单的数据一致性

### 9.2 业务风险
- 修改次数的合理限定
- 纠纷仲裁的公正性
- 画师质量参差不齐

### 9.3 解决方案
- 使用异步上传和CDN
- 采用第三方支付SDK
- 订单状态使用乐观锁
- 制定明确的交易规则
- 画师分级和推荐机制
