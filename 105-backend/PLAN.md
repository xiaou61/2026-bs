# 实现计划

## 问题陈述

构建一个围绕接口文档、Mock 测试和用例执行管理的前后端分离系统，覆盖接口项目、分组、定义、参数字段、Mock 规则、测试用例、执行记录和文档快照。

## 当前状态

`105-backend` 与 `105-frontend` 已完成接口测试管理平台的基础实现，当前进入精修阶段，重点核对认证安全、执行流程、页面一致性和文档准确性。

## 实施方案

### 第一阶段：后端开发
1. Spring Boot 2.7.18 + MyBatis-Plus 3.5.5 + Redis + JWT
2. 创建 `api_testcase_105` 数据库和 14 张业务表
3. 实现统一返回、异常处理、JWT 拦截和 Redis Token 校验
4. 实现 Entity、Mapper、Service 和 Controller
5. 实现用例执行、结果回写、Mock 规则启停和文档快照管理
6. 实现首页统计和操作日志

### 第二阶段：前端开发
1. Vue3 + Vite + Element Plus + Pinia + Axios + ECharts
2. 登录页、布局页、首页看板
3. 接口项目、接口定义、参数字段、Mock 规则、测试用例、执行记录、结果明细、文档快照和日志页面
4. 业务按钮支持执行、复制、发布、启停和查看结果

### 第三阶段：文档更新
1. 更新 `readme.md`
2. 更新 `readme_simple.md`
3. 更新候选清单

## 完成标准

- 后端结构完整
- 前端结构完整
- SQL 包含演示数据
- 认证返回不暴露敏感字段
- 用例执行、结果展示和文档快照保持一致
- 文档描述与实际实现保持一致
- 不执行编译验证
