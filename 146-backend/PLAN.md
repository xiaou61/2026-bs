# 食品安全抽检任务与结果公示平台 开发计划

## 实施范围
- 后端工程：实体、Mapper、Service、Controller、统一响应、JWT 登录、Redis Token 校验、统计接口
- 前端工程：登录页、布局菜单、通用数据页、模块页面、统计看板、接口封装和路由守卫
- 数据脚本：MySQL 建库建表、初始化账号和演示业务数据

## 模块拆分
- 抽检计划：计划编号、计划名称、抽检类别、负责区域、计划周期和状态维护
- 食品档案：食品编号、食品名称、食品类型、生产批次、抽检状态和状态维护
- 商户档案：商户编号、商户名称、联系人、联系电话、经营地址和状态维护
- 抽样任务：任务编号、任务主题、抽样地点、抽样时间、负责人员和状态维护
- 检测机构：机构编号、机构名称、检测资质、入驻时间、负责领域和状态维护
- 样品信息：样品编号、食品名称、样品类型、抽样说明、录入时间和状态维护
- 检测结果：结果编号、食品名称、检测结论、检测时间、检测员和状态维护
- 复检申请：申请编号、食品名称、复检原因、申请时间、申请人和状态维护
- 处置记录：处置编号、食品名称、处置方式、处置时间、责任单位和状态维护
- 结果公示：公示编号、食品名称、公示标题、公示渠道、公示时间和状态维护
- 风险预警：预警编号、食品名称、预警时间、风险类型、处置建议和状态维护
- 通知公告：通知编号、食品名称、通知类型、通知内容、接收人和状态维护
- 操作日志：关键模块操作人、动作、对象、详情和结果记录

## 验收标准
- 具备 PRD、PLAN、后端、前端和 SQL 初始化脚本
- 后端端口、前端端口、数据库名、Redis DB 与项目编号一致
- 静态结构达到 14 张表、14 个实体、14 个 Mapper、16 个 Controller、17 个前端页面
- 无旧项目包名、标题、端口、数据库名残留
