# 医疗器械借用消毒与追踪管理系统 开发计划

## 实施范围
- 后端工程：实体、Mapper、Service、Controller、统一响应、JWT 登录、Redis Token 校验、统计接口
- 前端工程：登录页、布局菜单、通用数据页、模块页面、统计看板、接口封装和路由守卫
- 数据脚本：MySQL 建库建表、初始化账号和演示业务数据

## 模块拆分
- 器械档案：器械编号、名称、型号、科室、责任人和状态维护
- 器械分类：分类编号、分类名称、风险等级、消毒方式和状态维护
- 科室信息：科室编号、科室名称、楼层、联系人和联系电话维护
- 借用申请：申请编号、器械、申请科室、申请人、申请时间和状态维护
- 借用记录：借用编号、申请编号、器械、借出时间、借用人和状态维护
- 归还记录：归还编号、借用编号、归还时间、检查结果和接收人维护
- 消毒批次：批次编号、消毒方式、设备、负责人、开始时间和状态维护
- 消毒记录：记录编号、批次、器械、消毒结果、操作人和状态维护
- 二维码追踪：追踪编号、二维码、器械、当前位置和流转状态维护
- 维护记录：维护编号、器械、维护类型、维护时间、维护人和状态维护
- 巡检任务：巡检编号、器械、巡检日期、巡检人和任务状态维护
- 风险预警：预警编号、器械、预警等级、触发原因、处理人和状态维护
- 操作日志：关键模块操作人、动作、对象、详情和结果记录

## 验收标准
- 具备 PRD、PLAN、后端、前端和 SQL 初始化脚本
- 后端端口、前端端口、数据库名、Redis DB 与项目编号一致
- 静态结构达到 14 张表、14 个实体、14 个 Mapper、16 个 Controller、17 个前端页面
- 无旧项目包名、标题、端口、数据库名残留
