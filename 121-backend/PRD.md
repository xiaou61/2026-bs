# 无人机巡检任务调度与缺陷上报平台 PRD

## 1. 项目背景
无人机巡检已广泛用于电力、园区、水利和交通场景，平台需要统一管理无人机、飞手、航线、任务、缺陷和整改闭环。

## 2. 用户角色
系统提供四类默认演示角色：ADMIN（admin / 123456）、PILOT（pilot / 123456）、ENGINEER（engineer / 123456）、MANAGER（manager / 123456）。

## 3. 功能模块
1. 账号权限：管理员、飞手、工程师、运营经理四角色账号、JWT 鉴权和 Redis 登录态。
2. 无人机设备：设备编号、名称、型号、电量、所属队伍和运行状态管理。
3. 飞手档案：飞手编号、姓名、执照等级、所属队伍和累计飞行小时管理。
4. 巡检区域：区域编号、区域名称、城市、风险等级和负责人管理。
5. 航线规划：航线编号、名称、区域、里程、航点数量和风险等级管理。
6. 巡检任务：任务单号、航线、飞手、计划时间、优先级和执行状态管理。
7. 飞行记录：飞行记录编号、任务单号、无人机、起飞时间、飞行时长和巡检结果管理。
8. 缺陷报告：缺陷编号、区域、类型、等级、上报人和照片数量管理。
9. 缺陷图片：图片编号、缺陷编号、文件名、AI标签、置信度和复核结果管理。
10. 整改工单：工单编号、缺陷编号、责任班组、截止时间、整改措施和验收结果管理。
11. 电池站点：站点编号、站点名称、区域、端口数量、可用端口和管理员管理。
12. 维保记录：维保编号、无人机、维保类型、技术员、维保时间和费用管理。
13. 风险预警：预警编号、风险对象、等级、原因、处理人和触发时间管理。
14. 操作日志：操作人、模块、动作、对象、详情和IP地址审计。

## 4. 核心流程
1. 管理员维护账号、基础档案和业务主数据。
2. 业务人员创建任务、登记过程记录并推进状态流转。
3. 管理人员查看风险、整改、计划和运营指标。
4. 系统通过首页看板展示趋势、分布和关键数量。
5. 操作日志沉淀关键动作，方便答辩时展示闭环。

## 5. 非功能要求
- 后端采用 Spring Boot，前端采用 Vue3。
- 数据库使用 MySQL，默认账号密码为 root / 1234。
- 登录态使用 JWT 与 Redis 校验。
- 数据页支持分页查询、关键词检索、状态筛选、新增、编辑、删除和状态动作。
