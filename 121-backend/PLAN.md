# 实现计划

## 问题陈述

构建一个围绕无人机巡检、航线规划、缺陷上报和整改闭环的前后端分离系统，覆盖无人机设备、飞手档案、巡检区域、飞行记录、缺陷报告、整改工单和风险预警。

## 当前状态

`121-backend` 与 `121-frontend` 已完成无人机巡检平台的基础实现，当前进入精修阶段，重点核对认证安全、任务调度、缺陷闭环和文档准确性。

## 实施方案

### 第一阶段：后端开发
1. Spring Boot 2.7.18 + MyBatis-Plus 3.5.5 + Redis + JWT
2. 创建 `drone_inspection_121` 数据库和 14 张业务表
3. 实现统一返回、异常处理、JWT 拦截和 Redis Token 校验
4. 实现 Entity、Mapper、Service 和 Controller
5. 实现任务调度、航线规划、缺陷登记、整改跟踪和风险预警
6. 实现首页统计和操作日志

### 第二阶段：前端开发
1. Vue3 + Vite + Element Plus + Pinia + Axios + ECharts
2. 登录页、布局页、首页看板
3. 无人机设备、飞手档案、巡检区域、航线规划、巡检任务、飞行记录、缺陷报告、整改工单和日志页面
4. 业务按钮支持派发、执行、上报、整改和查看统计

### 第三阶段：文档更新
1. 更新 `readme.md`
2. 更新 `readme_simple.md`
3. 更新候选清单

## 完成标准

1. 后端结构完整。
2. 前端结构完整。
3. SQL 包含演示数据。
4. 认证返回不暴露敏感字段。
5. 巡检任务、飞行记录、缺陷上报和整改闭环口径一致。
6. 文档描述与实际实现保持一致。
7. 不执行编译验证。
