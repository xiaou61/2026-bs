# 药品不良反应上报与随访管理系统 开发计划

## 实施范围
- 后端工程：实体、Mapper、Service、Controller、统一响应、JWT 登录、Redis Token 校验、统计接口
- 前端工程：登录页、布局菜单、通用数据页、模块页面、统计看板、接口封装和路由守卫
- 数据脚本：MySQL 建库建表、初始化账号和演示业务数据

## 模块拆分
- 患者档案：患者编号、姓名、性别、年龄、联系电话和状态维护
- 药品目录：药品编号、名称、剂型、生产企业和批准文号维护
- 上报人档案：上报人编号、姓名、机构、联系电话和角色维护
- 不良反应上报：上报编号、患者、药品、上报时间、严重等级和处理状态维护
- 反应症状：症状编号、上报编号、症状名称、出现时间和症状等级维护
- 风险评估：评估编号、上报编号、风险等级、评分和评估人维护
- 随访计划：计划编号、上报编号、随访日期、随访方式和负责人维护
- 随访记录：记录编号、计划编号、随访时间、患者情况和记录人维护
- 病例复核：复核编号、上报编号、复核人、复核意见和复核时间维护
- 处置建议：建议编号、上报编号、建议类型、建议内容和医生维护
- 科室信息：科室编号、科室名称、医院名称、联系人和联系电话维护
- 统计报表：统计编号、科室、统计月份、上报数量和严重例数维护
- 操作日志：关键模块操作人、动作、对象、详情和结果记录

## 验收标准
- 具备 PRD、PLAN、后端、前端和 SQL 初始化脚本
- 后端端口、前端端口、数据库名、Redis DB 与项目编号一致
- 静态结构达到 14 张表、14 个实体、14 个 Mapper、16 个 Controller、17 个前端页面
- 无旧项目包名、标题、端口、数据库名残留
