# 温室大棚物联网控制与病害预警系统 开发计划

## 实施范围
- 后端工程：实体、Mapper、Service、Controller、统一响应、JWT 登录、Redis Token 校验、统计接口
- 前端工程：登录页、布局菜单、通用数据页、模块页面、统计看板、接口封装和路由守卫
- 数据脚本：MySQL 建库建表、初始化账号和演示业务数据

## 模块拆分
- 温室档案：温室编号、名称、基地、面积、负责人和状态维护
- 作物批次：作物批次、温室、作物名称、定植日期和负责人维护
- 环境传感器：传感器编号、温室、设备类型、安装位置和在线状态维护
- 环境读数：读数编号、温室、采集时间、温度、湿度和状态维护
- 灌溉任务：灌溉任务、温室、任务时间、水量和执行人维护
- 施肥计划：施肥计划、温室、肥料类型、计划用量和负责人维护
- 虫害预警：虫害预警、温室、虫害类型、等级和处理人维护
- 病害诊断：诊断编号、作物批次、病害名称、建议方案和诊断状态维护
- 控制设备：控制设备、温室、设备名称、设备类型和控制模式维护
- 远程指令：指令编号、设备编号、指令类型、参数、下发人和执行状态维护
- 采收记录：采收编号、作物批次、采收日期、采收重量和质检状态维护
- 维护工单：工单编号、设备、问题类型、负责人、截止时间和处理状态维护
- 操作日志：关键模块操作人、动作、对象、详情和结果记录

## 验收标准
- 具备 PRD、PLAN、后端、前端和 SQL 初始化脚本
- 后端端口、前端端口、数据库名、Redis DB 与项目编号一致
- 静态结构达到 14 张表、14 个实体、14 个 Mapper、16 个 Controller、17 个前端页面
- 无旧项目包名、标题、端口、数据库名残留
