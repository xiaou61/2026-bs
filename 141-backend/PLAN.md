# 固定资产 RFID 盘点与借用归还系统 开发计划

## 实施范围
- 后端工程：实体、Mapper、Service、Controller、统一响应、JWT 登录、Redis Token 校验、统计接口
- 前端工程：登录页、布局菜单、通用数据页、模块页面、统计看板、接口封装和路由守卫
- 数据脚本：MySQL 建库建表、初始化账号和演示业务数据

## 模块拆分
- 资产档案：资产编号、资产名称、资产规格、所属部门、购置日期和状态维护
- 资产分类：分类编号、分类名称、资产类型、管理部门、折旧年限和状态维护
- RFID标签：标签编号、资产名称、绑定编码、标签状态、盘点区域和状态维护
- 存放位置：位置编号、位置名称、责任部门、位置容量、启用时间和状态维护
- 盘点任务：任务编号、任务名称、执行人员、盘点日期、差异数量和状态维护
- 盘点明细：明细编号、任务名称、资产名称、扫描时间、盘点结果和状态维护
- 借用申请：申请编号、资产名称、借用用途、借用日期、借用人和状态维护
- 借用归还：归还编号、资产名称、归还人、归还数量、归还时间和状态维护
- 维修记录：维修编号、资产名称、维修类型、责任部门、维修费用和状态维护
- 折旧记录：折旧编号、资产名称、资产原值、累计折旧、计提日期和状态维护
- 闲置处置：处置编号、资产名称、处置方式、处置原因、处置时间和状态维护
- 预警通知：通知编号、资产名称、预警类型、预警内容、发送时间和状态维护
- 操作日志：关键模块操作人、动作、对象、详情和结果记录

## 验收标准
- 具备 PRD、PLAN、后端、前端和 SQL 初始化脚本
- 后端端口、前端端口、数据库名、Redis DB 与资产编号一致
- 静态结构达到 14 张表、14 个实体、14 个 Mapper、16 个 Controller、17 个前端页面
- 无旧项目包名、标题、端口、数据库名残留
