# 实现计划

## 问题陈述

构建一个围绕开源许可证合规扫描、风险治理和项目台账管理的前后端分离系统，覆盖组织团队、项目仓库、依赖清单、策略基线、扫描任务、风险问题、整改任务、审计报告和审批流程。

## 当前状态

`104-backend` 与 `104-frontend` 已完成许可证合规系统的基础实现，当前进入精修阶段，重点核对认证安全、状态流转、页面一致性和文档准确性。

## 实施方案

### 第一阶段：后端开发
1. Spring Boot 2.7.18 + MyBatis 2.3.1 + PageHelper + Redis + JWT
2. 创建 `license_compliance_104` 数据库和 14 张业务表
3. 实现统一返回、异常处理、JWT 拦截、Redis 配置和分页查询
4. 实现 Entity、Mapper、Service 和 Controller
5. 实现扫描执行、问题流转、整改跟进、报告归档和审批处理
6. 实现首页统计和操作日志

### 第二阶段：前端开发
1. Vue3 + Vite + Element Plus + Pinia + Axios + ECharts
2. 登录页、布局页、首页看板
3. 组织团队、项目台账、依赖台账、扫描任务、风险问题、整改任务、审计报告、审批记录和日志页面
4. 业务按钮支持扫描、指派、整改、审核和归档

### 第三阶段：文档更新
1. 更新 `readme.md`
2. 更新 `readme_simple.md`
3. 更新候选清单

## 完成标准

- 后端结构完整
- 前端结构完整
- SQL 包含演示数据
- 认证返回不暴露敏感字段
- 页面路由、业务状态和审批动作保持一致
- 文档描述与实际实现保持一致
- 不执行编译验证
