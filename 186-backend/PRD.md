# 校园餐厅后厨留样与卫生巡检台账系统 PRD

## 项目概述
本系统面向校园餐厅后厨留样、食品安全巡检、食材验收、消毒记录、问题整改和风险预警场景，提供餐厅档案、后厨区域、菜品档案、留样登记、留样存储、食材验收、消毒记录、卫生巡检、问题整改、风险提醒和厨余处置的一体化后台管理能力。

## 用户角色
- 系统管理员：账号权限、基础配置和全部数据维护
- 食堂管理员：维护餐厅档案、后厨区域、菜品档案和台账规则
- 后厨负责人：登记菜品留样、留样存储、消毒记录和厨余处置
- 卫生巡检员：执行卫生巡检、发现问题并跟踪整改闭环
- 仓储管理员：维护食材验收、库存流转和风险提醒
- 校方监管：查看留样、巡检、整改和风险统计

## 功能模块
- 餐厅档案：餐厅编号、餐厅名称、经营类型、负责人员、启用时间和餐厅状态维护
- 后厨区域：区域编号、区域名称、区域类型、负责人、启用时间和区域状态维护
- 菜品档案：菜品编号、菜品名称、菜品类型、厨师人员、上架时间和菜品状态维护
- 留样登记：留样编号、关联菜品、留样类型、留样人员、留样时间和留样状态维护
- 留样存储：存储编号、留样登记、存储类型、管理人员、存储时间和存储状态维护
- 食材验收：验收编号、食材名称、验收类型、仓储人员、验收时间和验收状态维护
- 消毒记录：消毒编号、后厨区域、消毒类型、执行人员、消毒时间和消毒状态维护
- 卫生巡检：巡检编号、后厨区域、巡检类型、巡检人员、巡检时间和巡检状态维护
- 问题整改：整改编号、巡检问题、整改类型、责任人员、整改时间和整改状态维护
- 风险提醒：提醒编号、风险对象、风险类型、提醒人员、提醒时间和提醒状态维护
- 厨余处置：处置编号、后厨区域、处置类型、处置人员、处置时间和处置状态维护
- 操作日志：日志编号、操作模块、操作类型、操作人、操作时间和执行结果维护
