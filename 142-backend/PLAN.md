# 车辆保险理赔材料审核与进度跟踪系统 开发计划

## 实施范围
- 后端工程：实体、Mapper、Service、Controller、统一响应、JWT 登录、Redis Token 校验、统计接口
- 前端工程：登录页、布局菜单、通用数据页、模块页面、统计看板、接口封装和路由守卫
- 数据脚本：MySQL 建库建表、初始化账号和演示业务数据

## 模块拆分
- 保险保单：保单编号、保单名称、险种类型、承保公司、保障期间和状态维护
- 车辆档案：车辆编号、车牌号码、车架号码、品牌型号、使用性质和状态维护
- 客户档案：客户编号、客户姓名、证件号码、联系电话、客户类型和状态维护
- 理赔申请：申请编号、案件名称、报案渠道、申请时间、事故地点和状态维护
- 事故报案：报案编号、案件名称、事故类型、报案时间、报案人和状态维护
- 材料清单：清单编号、报案编号、材料类型、材料说明、提交时间和状态维护
- 材料审核：审核编号、报案编号、审核结论、审核时间、审核人和状态维护
- 定损记录：定损编号、报案编号、定损金额、定损时间、定损员和状态维护
- 赔付记录：赔付编号、报案编号、赔付金额、赔付时间、收款人和状态维护
- 进度跟踪：跟踪编号、报案编号、当前节点、处理部门、预计完成和状态维护
- 回访记录：回访编号、报案编号、回访方式、回访内容、客户反馈和状态维护
- 消息通知：通知编号、报案编号、通知类型、通知内容、接收人和状态维护
- 操作日志：关键模块操作人、动作、对象、详情和结果记录

## 验收标准
- 具备 PRD、PLAN、后端、前端和 SQL 初始化脚本
- 后端端口、前端端口、数据库名、Redis DB 与项目编号一致
- 静态结构达到 14 张表、14 个实体、14 个 Mapper、16 个 Controller、17 个前端页面
- 无旧项目包名、标题、端口、数据库名残留
