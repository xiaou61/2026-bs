# 实现计划

## 问题陈述

构建一个围绕工地安全巡检、隐患上报和整改验收的前后端分离系统，覆盖工地项目、施工班组、安全员、巡检计划、隐患工单、整改处理和验收记录。

## 当前状态

`122-backend` 与 `122-frontend` 已完成智慧工地巡检系统的基础实现，当前进入精修阶段，重点核对认证安全、隐患整改、责任闭环和文档准确性。

## 实施方案

### 第一阶段：后端开发
1. Spring Boot 2.7.18 + MyBatis 2.3.1 + PageHelper + Redis + JWT
2. 创建 `smart_worksite_safety_122` 数据库和 14 张业务表
3. 实现统一返回、异常处理、JWT 拦截、Redis 配置和分页查询
4. 实现 Entity、Mapper、Service 和 Controller
5. 实现巡检计划、隐患上报、整改工单、验收记录和风险统计
6. 实现首页统计和操作日志

### 第二阶段：前端开发
1. Vue3 + Vite + Element Plus + Pinia + Axios + ECharts
2. 登录页、布局页、首页看板
3. 工地项目、施工班组、安全员、巡检计划、巡检任务、隐患上报、整改工单、验收记录和日志页面
4. 业务按钮支持巡检、上报、整改、验收和查看统计

### 第三阶段：文档更新
1. 更新 `readme.md`
2. 更新 `readme_simple.md`
3. 更新候选清单

## 完成标准

1. 后端结构完整。
2. 前端结构完整。
3. SQL 包含演示数据。
4. 认证返回不暴露敏感字段。
5. 巡检、隐患、整改、验收和责任追踪口径一致。
6. 文档描述与实际实现保持一致。
7. 不执行编译验证。
