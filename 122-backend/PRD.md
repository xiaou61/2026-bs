# 智慧工地安全巡检与隐患整改系统 PRD

## 1. 项目背景
建筑工地安全管理需要覆盖项目、班组、巡检、隐患、整改、验收和培训全流程，系统用于提升隐患闭环效率和安全责任追踪能力。

## 2. 用户角色
系统提供四类默认演示角色：ADMIN（admin / 123456）、INSPECTOR（inspector / 123456）、TEAM_LEADER（teamleader / 123456）、SUPERVISOR（supervisor / 123456）。

## 3. 功能模块
1. 账号权限：管理员、巡检员、班组长、安全主管四角色账号、JWT 鉴权和 Redis 登录态。
2. 工地项目：项目编号、名称、地址、承建单位、开工日期和风险等级管理。
3. 施工班组：班组编号、名称、班组长、人数、工种和联系电话管理。
4. 安全员档案：安全员编号、姓名、证书、所属组、电话和专长管理。
5. 巡检计划：计划编号、项目、日期、安全员、区域和重点风险管理。
6. 巡检任务：任务单号、计划、巡检员、检查区域、优先级和截止时间管理。
7. 隐患上报：隐患编号、项目、类型、等级、上报人和上报时间管理。
8. 整改工单：工单编号、隐患编号、责任班组、截止时间、整改要求和验收状态管理。
9. 验收记录：验收编号、工单编号、验收人、验收时间、结果和评分管理。
10. 安全培训：培训编号、课程、讲师、班组、人数和培训日期管理。
11. 设备检查：检查编号、设备名称、设备类型、项目、检查结果和检查人管理。
12. 防护用品：用品编号、名称、类型、库存、安全库存和仓库管理。
13. 风险预警：预警编号、风险对象、等级、原因、处理人和触发时间管理。
14. 操作日志：操作人、模块、动作、对象、详情和IP地址审计。

## 4. 核心流程
1. 管理员维护账号、基础档案和业务主数据。
2. 业务人员创建任务、登记过程记录并推进状态流转。
3. 管理人员查看风险、整改、计划和运营指标。
4. 系统通过首页看板展示趋势、分布和关键数量。
5. 操作日志沉淀关键动作，方便答辩时展示闭环。

## 5. 非功能要求
- 后端采用 Spring Boot，前端采用 Vue3。
- 数据库使用 MySQL，默认账号密码为 root / 1234。
- 登录态使用 JWT 与 Redis 校验。
- 数据页支持分页查询、关键词检索、状态筛选、新增、编辑、删除和状态动作。
