# 高校实验设备共享预约与违规使用追踪系统 开发计划

## 实施范围
- 后端工程：实体、Mapper、Service、Controller、统一响应、JWT 登录、Redis Token 校验、统计接口
- 前端工程：登录页、布局菜单、通用数据页、模块页面、统计看板、接口封装和路由守卫
- 数据脚本：MySQL 建库建表、初始化账号和演示业务数据

## 模块拆分
- 设备档案：设备编号、设备名称、设备型号、所属实验室、入库时间和状态维护
- 实验室档案：实验室编号、实验室名称、实验室类型、所属校区、容纳数量和状态维护
- 设备分类：分类编号、分类名称、设备类型、存放区域、状态标签和状态维护
- 设备借用人：借用人编号、借用人姓名、所属班级、联系电话、注册时间和状态维护
- 预约申请：预约编号、设备名称、申请人、预约时间、预约状态和状态维护
- 借用记录：借用编号、设备名称、借用人、借出时间、借用用途和状态维护
- 使用登记：登记编号、设备名称、使用场景、登记时间、指导教师和状态维护
- 违规记录：违规编号、设备名称、处理人、违规类型、违规时间和状态维护
- 维修工单：工单编号、设备名称、故障类型、报修人、处理结果和状态维护
- 归还确认：确认编号、设备名称、归还人、归还状态、确认时间和状态维护
- 巡检计划：巡检编号、设备名称、巡检主题、巡检内容、巡检时间和状态维护
- 通知公告：公告编号、设备名称、公告类型、公告内容、发布时间和状态维护
- 操作日志：关键模块操作人、动作、对象、详情和结果记录

## 验收标准
- 具备 PRD、PLAN、后端、前端和 SQL 初始化脚本
- 后端端口、前端端口、数据库名、Redis DB 与项目编号一致
- 静态结构达到 14 张表、14 个实体、14 个 Mapper、16 个 Controller、17 个前端页面
- 无旧项目包名、标题、端口、数据库名残留
