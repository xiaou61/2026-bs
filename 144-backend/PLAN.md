# 无障碍出行路线规划与志愿协助平台 开发计划

## 实施范围
- 后端工程：实体、Mapper、Service、Controller、统一响应、JWT 登录、Redis Token 校验、统计接口
- 前端工程：登录页、布局菜单、通用数据页、模块页面、统计看板、接口封装和路由守卫
- 数据脚本：MySQL 建库建表、初始化账号和演示业务数据

## 模块拆分
- 无障碍路线：路线编号、路线名称、路线类型、适用场景、建议时长和状态维护
- 设施点位：点位编号、点位名称、设施类型、详细地址、开放状态和状态维护
- 出行用户：用户编号、用户姓名、联系电话、辅助需求、出行偏好和状态维护
- 协助预约：预约编号、预约主题、出发地点、预约时间、目的地点和状态维护
- 志愿者档案：志愿者编号、志愿者姓名、服务专长、加入时间、服务区域和状态维护
- 路线方案：方案编号、志愿者编号、路径类型、路线说明、生成时间和状态维护
- 协助任务：任务编号、志愿者编号、任务类型、执行时间、执行人和状态维护
- 服务签到：签到编号、志愿者编号、签到地点、签到时间、签到人和状态维护
- 评价反馈：评价编号、志愿者编号、满意度、评价时间、评价对象和状态维护
- 应急联系人：联系人编号、用户编号、联系人姓名、联系电话、关系说明和状态维护
- 行程记录：行程编号、用户编号、出行日期、出行路线、完成情况和状态维护
- 消息通知：通知编号、用户编号、通知类型、通知内容、接收人和状态维护
- 操作日志：关键模块操作人、动作、对象、详情和结果记录

## 验收标准
- 具备 PRD、PLAN、后端、前端和 SQL 初始化脚本
- 后端端口、前端端口、数据库名、Redis DB 与项目编号一致
- 静态结构达到 14 张表、14 个实体、14 个 Mapper、16 个 Controller、17 个前端页面
- 无旧项目包名、标题、端口、数据库名残留
