# 学术会议投稿评审与日程管理系统 开发计划

## 实施范围
- 后端工程：实体、Mapper、Service、Controller、统一响应、JWT 登录、Redis Token 校验、统计接口
- 前端工程：登录页、布局菜单、通用数据页、模块页面、统计看板、接口封装和路由守卫
- 数据脚本：MySQL 建库建表、初始化账号和演示业务数据

## 模块拆分
- 会议信息：会议编号、会议名称、会议主题、主办单位、会场数量和状态维护
- 征稿通知：通知编号、通知标题、发布人、截稿日期、投稿指引和状态维护
- 作者档案：作者编号、作者姓名、所属单位、联系邮箱、联系电话和状态维护
- 论文投稿：投稿编号、论文题目、作者姓名、提交版本、附件数量和状态维护
- 审稿专家：专家编号、专家姓名、研究方向、所在单位、邀请时间和状态维护
- 审稿分配：分配编号、投稿编号、审稿专家、分配意见、分配时间和状态维护
- 盲审记录：评审编号、稿件编号、论文题目、评审得分、提交日期和状态维护
- 录用通知：通知编号、稿件编号、论文题目、录用结果、通知人和状态维护
- 参会报名：报名编号、参会主题、参会单位、报名人数、联系人和状态维护
- 会场安排：会场编号、会场名称、对应议题、座位数量、签到数量和状态维护
- 会议日程：日程编号、日程主题、开始顺序、日程类型、主持人和状态维护
- 签到记录：签到编号、参会人、签到次数、签到方式、会务人员和状态维护
- 操作日志：关键模块操作人、动作、对象、详情和结果记录

## 验收标准
- 具备 PRD、PLAN、后端、前端和 SQL 初始化脚本
- 后端端口、前端端口、数据库名、Redis DB 与项目编号一致
- 静态结构达到 14 张表、14 个实体、14 个 Mapper、16 个 Controller、17 个前端页面
- 无旧项目包名、标题、端口、数据库名残留
