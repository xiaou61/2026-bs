# 实现计划

## 问题陈述

构建一个围绕招聘材料解析和岗位匹配的前后端分离系统，覆盖候选人档案、简历证书、岗位要求、解析任务、匹配任务、面试排期、反馈和统计审计。

## 当前状态

`101-backend` 与 `101-frontend` 已完成招聘匹配系统的基础实现，当前进入精修阶段，重点核对认证安全、业务一致性、界面细节和文档准确性。当前以 `101` 作为逐项目精修样板，要求实现与 PRD 的角色边界完全对齐。

## 实施方案

### 第一阶段：后端开发
1. Spring Boot 2.7.18 + MyBatis-Plus 3.5.5 + Redis + JWT
2. 创建 `recruit_match_101` 数据库和 13 张表
3. 实现统一返回、异常处理、JWT 拦截、Redis 配置
4. 实现 13 个 Entity、Mapper、Service
5. 实现解析任务、匹配任务、证书核验、面试排期和统计服务
6. 实现 REST Controller

### 第二阶段：前端开发
1. Vue3 + Vite + Element Plus + Pinia + Axios
2. 登录页、布局页、首页看板
3. 用户、候选人、简历、证书、岗位、要求、解析、匹配、面试、反馈、日志页面
4. 动作按钮支持解析、匹配、复核、核验、面试状态流转

### 第三阶段：文档更新
1. 更新 `readme.md`
2. 更新 `readme_simple.md`
3. 更新候选清单
4. 校正文档中的角色范围、看板权限和数据归属说明

## 完成标准

- 后端结构完整
- 前端结构完整
- SQL 包含演示数据
- 认证返回不暴露敏感字段
- 页面路由、菜单和关键业务动作保持一致
- 管理员仅访问账号、日志和看板
- HR 仅访问招聘业务管理模块
- 候选人仅维护自己的档案、简历和证书
- 面试官仅处理分配给自己的排期和反馈
- 解析任务、匹配任务、面试排期的状态通过动作接口流转，不允许通过普通编辑直接改状态
- 文档描述与实际实现保持一致
- 后端至少通过 `mvn test`，前端至少通过 `npm run build`
