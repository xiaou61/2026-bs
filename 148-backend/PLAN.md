# 社区养老服务派单与健康随访管理系统 开发计划

## 实施范围
- 后端工程：实体、Mapper、Service、Controller、统一响应、JWT 登录、Redis Token 校验、统计接口
- 前端工程：登录页、布局菜单、通用数据页、模块页面、统计看板、接口封装和路由守卫
- 数据脚本：MySQL 建库建表、初始化账号和演示业务数据

## 模块拆分
- 服务套餐：套餐编号、套餐名称、服务类别、适用人群、服务周期和状态维护
- 老人档案：老人编号、老人姓名、年龄分层、居住地址、照护等级和状态维护
- 护理档案：护理编号、护理姓名、联系电话、护理等级、所属机构和状态维护
- 服务工单：工单编号、服务主题、服务地址、上门时间、派单人和状态维护
- 服务团队：团队编号、团队名称、擅长服务、组建时间、服务片区和状态维护
- 上门签到：签到编号、老人姓名、签到类型、签到说明、签到时间和状态维护
- 服务记录：记录编号、老人姓名、服务结论、服务时间、服务人员和状态维护
- 健康评估：评估编号、老人姓名、评估项目、评估时间、评估人和状态维护
- 用药提醒：提醒编号、老人姓名、提醒类型、提醒时间、接收对象和状态维护
- 家属回访：回访编号、老人姓名、回访主题、回访方式、回访时间和状态维护
- 预警事件：事件编号、老人姓名、上报时间、事件类型、处理建议和状态维护
- 通知公告：通知编号、老人姓名、通知类型、通知内容、接收人和状态维护
- 操作日志：关键模块操作人、动作、对象、详情和结果记录

## 验收标准
- 具备 PRD、PLAN、后端、前端和 SQL 初始化脚本
- 后端端口、前端端口、数据库名、Redis DB 与项目编号一致
- 静态结构达到 14 张表、14 个实体、14 个 Mapper、16 个 Controller、17 个前端页面
- 无旧项目包名、标题、端口、数据库名残留
