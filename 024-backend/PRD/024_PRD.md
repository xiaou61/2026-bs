# 流浪动物领养与救助平台 - 产品需求文档 (PRD)

## 1. 项目概述

### 1.1 项目背景
随着社会发展，人们对宠物的关注度日益提高，但流浪动物数量也在不断增加。传统的流浪动物救助方式信息传递滞后、领养渠道不畅通。为了更好地帮助流浪动物找到温暖的家，同时规范领养流程，开发本“流浪动物领养与救助平台”。

### 1.2 项目目标
- 构建一个集流浪动物展示、领养申请、救助信息发布、科普宣传于一体的综合性平台。
- 提高流浪动物的领养率，降低流浪动物数量。
- 建立规范的领养审核机制，确保动物被负责任地领养。
- 增强用户互动，形成关爱流浪动物的良好社区氛围。

## 2. 用户角色

本系统主要包含以下三种角色：

1.  **普通用户 (前台)**
    *   有领养意向的爱心人士。
    *   发现流浪动物需要发布救助信息的路人。
    *   关注流浪动物的志愿者。
2.  **管理员 (后台)**
    *   负责平台运营、审核、数据管理的工作人员。

## 3. 功能需求

### 3.1 普通用户功能 (前台)

#### 3.1.1 用户中心
- **注册/登录**：支持手机号或用户名注册登录。
- **个人信息管理**：修改头像、昵称、联系方式、收货地址等。
- **我的申请**：查看领养申请的进度和结果。
- **我的发布**：查看自己发布的救助信息或帖子。

#### 3.1.2 首页与展示
- **轮播图**：展示紧急救助、热门领养或公益活动。
- **动物列表**：展示待领养的流浪动物，支持按品种、性别、年龄、体型筛选。
- **动物详情**：查看动物的详细信息（照片、名字、故事、健康状况、疫苗情况）。

#### 3.1.3 领养服务
- **在线申请**：填写领养申请表（包含养宠经验、家庭环境、经济状况等）。
- **签署协议**：在线阅读并同意领养协议（模拟）。
- **回访反馈**：领养成功后，上传宠物生活照进行反馈。

#### 3.1.4 救助中心
- **发布救助**：用户发现流浪动物可发布线索（地点、照片、描述）。
- **救助地图**：(可选) 在地图上标记流浪动物出没地点。

#### 3.1.5 爱心社区 (论坛)
- **发帖交流**：分享养宠经验、救助故事。
- **评论互动**：对帖子进行评论、点赞。

#### 3.1.6 科普公告
- **养宠知识**：阅读科学养宠文章。
- **平台公告**：查看系统通知和活动公告。

### 3.2 管理员功能 (后台)

#### 3.2.1 综合管理
- **控制台**：查看今日新增用户、待审核申请、新增流浪动物数量等数据概览。

#### 3.2.2 动物管理
- **动物录入**：添加新的流浪动物信息（图片上传、基本信息录入）。
- **动物维护**：编辑、删除动物信息，更新领养状态（待领养、申请中、已领养）。

#### 3.2.3 领养管理
- **申请审核**：查看用户的领养申请，进行通过或驳回操作。
- **回访记录**：记录对领养人的回访情况。

#### 3.2.4 内容管理
- **文章管理**：发布、编辑、删除科普文章和公告。
- **评论管理**：审核社区评论，删除违规内容。
- **救助信息审核**：审核用户发布的救助线索。

#### 3.2.5 用户管理
- **用户列表**：查看平台注册用户，支持禁用违规账号。
- **管理员管理**：添加或删除管理员账号。

## 4. 非功能需求

1.  **性能要求**：页面加载速度快，图片资源需优化。
2.  **安全性**：
    *   用户密码加密存储 (MD5/BCrypt)。
    *   防止SQL注入和XSS攻击。
    *   敏感操作（如删除）需二次确认。
3.  **易用性**：界面温馨、简洁，操作流程清晰，符合用户习惯。
4.  **兼容性**：前端页面需适配主流浏览器。

## 5. 技术栈

-   **后端**：Spring Boot 3.x
-   **数据库**：MySQL 8.0
-   **ORM框架**：MyBatis (XML配置方式，不使用Plus)
-   **前端**：Vue 3 + Vite + Element Plus (或类似UI库)
-   **开发工具**：IDEA, Navicat, Postman
